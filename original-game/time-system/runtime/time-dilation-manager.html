    TimeDilationManager¶

    本页合并多个源文档以去除同名 TimeDilation/Tick 类页的重复导航，同时保留每个源文件的原始信息。

    TimeDilationManager¶

    字段¶

    字段 Offset 说明
    static s_timeDilationInstId: uint static 全局 TimeDilation instance id 递增源。
    m_poiseBreakTimeStopInstance: PoiseBreakTimeStopInstance 0x80 韧性击破时停辅助实例。
    m_entityTimeDilationInsts: List<EntityTimeDilationInst> 0x88 Entity 层实例列表。
    m_globalTimeDilationInsts: List<GlobalTimeDilationInst> 0x90 Global 层实例列表。

    方法¶

    方法 确认状态 已确认行为
    OnInit() 已确认 创建/绑定 PoiseBreakTimeStopInstance。
    OnRelease() 已确认 转发 _Clear()。
    OnMapUnloaded(string) 已确认 转发/触发清理路径，细节待单独展开。
    GetNextInstId() 已确认 从静态 id 源分配 instance uid。
    Tick(float) 已确认 倒序 tick entity/global instance；无效时 Stop、从列表移除并回收到 pool；最后 tick PoiseBreakTimeStopInstance。未见在本方法中直接应用 TimeManager time-scale cache，cache apply 点仍以 TimeManager.TailLateTick 为准。
    StartEntityTimeDilation(...) 已确认 pool allocate EntityTimeDilationInst，Reset 后 _AddNewEntityTimeDilationInst，返回 instance uid handle。
    StartGlobalTimeDilation(...) 已确认 pool allocate GlobalTimeDilationInst，Reset 后 _AddNewGlobalTimeDilationInst，返回 instance uid handle。
    StartHitStop(...) 已确认 先按 HitStopAction.AffectType 创建临时 AbilitySystem list：OnlyAttacker(0) 只加 attacker、OnlyTarget(1) 加 targets、Both(2) 加 attacker + targets；每个 entry 读取预定义 tag 80 作为 TimeDilation/Layer/Entity/HitStop slot。useDirectCurve=true 时使用 action direct curve，否则读取 DataManager.hitStopConfig.hitSopSettings[key].timeScaleCurve，再调用 StartEntityTimeDilation(entity, priorityTag, duration, hitStopSlot, curve, null, null)。
    StartPoiseBreakTimeStop(AbilitySystem) 已确认 从 source ability system 取 entity/template，按敌人模板类型选择 PoiseBreakConfig.normal/boss/elite，懒创建 PoiseBreakTimeStopInstance 并委托 TryStart。唯一确认 caller 为 PoiseController.OnPoiseZero 状态 已确认，xref 已确认。
    StopAllEntityTimeDilation(Entity) 已确认 遍历 entity instances，匹配 owner 后 Stop、RemoveAt、Recycle。
    StopTimeDilation(uint) 已确认 按 instance id 找 entity/global instance 并 Stop；列表移除在后续 tick 清理。
    _AddNewEntityTimeDilationInst(EntityTimeDilationInst) 已确认 同 owner 且同 slot 竞争；旧 priority 大于新 priority 时拒绝新实例，否则 Stop/remove 旧实例，add 新实例并立即 tick/start 一次建立 handle。
    _AddNewGlobalTimeDilationInst(GlobalTimeDilationInst) 已确认 同 slot 竞争；旧 priority 大于新 priority 时拒绝新实例，否则 Stop/remove 旧实例，add 新实例并调用 GlobalTimeDilationInst.Start()。
    _Clear() 已确认 Stop 并回收所有 entity/global instance，清理列表与 poise break state。
    get_globalGameplaySlot() 已确认 从 TimeDilationConfig.predefinedTags 读取 TimeDilation/Layer/Global/GamePlay。
    get_influenceSkillCd() 已确认 只检查当前 TimeManager.timeDilationInst 对应的 active global instance，并返回该 instance 的 influenceSkillCd。

    调用入口¶

    • TimeDilationAction.ExecuteInternal：Global 分支调用 StartGlobalTimeDilation，Entity 分支调用 StartEntityTimeDilation。
    • HitStopAction：调用 StartHitStop。
    • UltimateTimeAction：调用 StartUltimateTime。
    • PoiseBreak 相关流程：AbilitySystem.PoiseController.OnPoiseZero 调用 StartPoiseBreakTimeStop。
    • TimeDilationHandle.Stop()：调用 StopTimeDilation(instId)。

    生命周期¶

    1. 外部 action 入口请求 start。
    2. Manager 分配 instance 并 Reset。
    3. _AddNew* 执行 slot/priority 竞争。
    4. 新实例加入列表并立即 start/tick，建立 TimeManager 或 entity tick-component handle。
    5. Manager 通过 BaseTickManager 的 Frame tick function 每帧驱动 Tick(float)；因为 ignoreGlobalTimeScale=true，全局时停期间 manager 仍按未缩放 tick 推进。
    6. isValid == false 时 Stop、RemoveAt、Recycle。
    7. OnRelease / OnMapUnloaded 调 _Clear() 全部清理。

    已确认行为¶

    • Global 竞争只看 slot。
    • Entity 竞争看 owner + slot；不同 owner 的相同 slot 可共存。
    • TimeDilationManager 是 BaseWorldManager 派生 world manager，但 tick 接入来自其继承链上的 BaseTickManager；构造参数为 Frame、ignore global time scale、1/30s interval。
    • priority 来源是 TimeDilationConfig.priorityMap；旧 priority 严格大于新 priority 时，新实例被拒绝。
    • StopTimeDilation 是软停止：Stop 清理底层 handle，下一轮 tick 因 isValid == false 被回收。
    • influenceSkillCd 只由当前 active global TimeDilation instance 决定。
    • StartUltimateTime 是原游戏专门的 Global TimeDilation helper：终结技全局时停在 Global/UltiSkill slot 上创建近似无限时长实例，并通过 action lifetime / handle stop 结束。
    • StartPoiseBreakTimeStop 上游 caller 已确认是 AbilitySystem.PoiseController.OnPoiseZero；该方法随后处理 poiseBreakEffects 的视觉 effect 创建。
    • StartHitStop 的 keyed curve owner 是 HitStopConfig.hitSopSettings，不是 TimeDilationConfig.curveMap；TimeDilationConfig 只提供 priority/slot 等通用配置。

    TickManager 属性¶

    构造函数初始化 Global/Entity instance list 后调用：

    BaseTickManager(TickType=8, ignoreGlobalTimeScale=true, tickInterval=0.033333335)

    含义：

    • TimeDilationManager 自身不受全局 timeScale 影响。
    • 大招或全局时停期间，manager 仍然会 tick。
    • instance 的 elapsed / duration 清理不会因为全局 scale=0 而永久停住。

    Slot / Priority 竞争¶

    Global 层¶

    Global instance 只按 slot 竞争：

    sameSlot(old.slot, new.slot)

    竞争处理：

    if oldPriority > newPriority:
 reject new instance
else:
 old.Stop()
 remove old from list
 add new
 new.Start()

    Entity 层¶

    Entity instance 按 owner/target + slot 竞争。不同目标即使 slot 相同也可以共存。
