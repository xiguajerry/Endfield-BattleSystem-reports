    EntityTimeDilationInst¶

    本页合并多个源文档以去除同名 TimeDilation/Tick 类页的重复导航，同时保留每个源文件的原始信息。

    EntityTimeDilationInst¶

    字段¶

    字段 Offset 说明
    m_instanceId: uint 0x18 运行时 id/handle 相关字段，具体用途待补。
    m_owner: ObjectPtr<Entity> 0x20 被影响的 Entity。
    m_timeScaleHandle: uint 0x38 Entity tick component additional time scale handle。
    m_duration: float 0x3C 持续时间，单位秒；小于 0 表示无限期。
    m_scaleCurve: AnimationCurve 0x40 scale 曲线。
    m_slotTag: GameplayTag 0x48 Entity 层 slot。
    m_priorityTag: GameplayTag 0x58 priority tag。
    m_elapsedTime: float 0x68 已经过时间。
    m_influencedTimeScaleByGlobal: bool 0x6C elapsed 是否受当前 global scale 影响，由 TimeDilationConfig.slotSpecialConfig 决定。

    方法¶

    方法 确认状态 已确认行为
    Reset(Entity, float, GameplayTag, GameplayTag, AnimationCurve, List<Type>) 已确认 写 owner/duration/curve/slot/priority/componentFilter，读取 slotSpecialConfig 设置 m_influencedTimeScaleByGlobal。
    OnTick(float) 已确认 先 _ClearTimeScaleHandle()，按 elapsed/duration evaluate curve，向 owner tick component 添加 additional scale，保存 handle；若 m_influencedTimeScaleByGlobal 为 true，elapsed 增量乘当前 TimeManager.timeScale。
    Stop() 已确认 调用 _ClearTimeScaleHandle()；不直接从 manager list 移除。
    _ClearTimeScaleHandle() 已确认 handle 非 0 时从 owner tick component 移除 additional scale，并清零 handle。
    get_isValid() 已确认 owner valid、handle 非 0，且 duration + epsilon >= elapsed 或 duration < 0。
    get_owner() 已确认 返回 m_owner 指向的 Entity。

    调用入口¶

    • TimeDilationManager.StartEntityTimeDilation 创建。
    • TimeDilationManager._AddNewEntityTimeDilationInst 负责竞争与立即 start/tick。
    • TimeDilationManager.Tick 每帧调用 OnTick 或回收。
    • HitStopAction / Entity 层 TimeDilationAction 经 manager 创建本类。

    生命周期¶

    1. Reset 写入目标、曲线和 slot。
    2. manager add 后立即调用一次 OnTick 建立 tick component handle。
    3. 每帧先移除上一帧 handle，再添加当前曲线 scale handle。
    4. Stop 或过期后 handle 被清理。
    5. 下一轮 manager tick 看到 isValid == false 后移除/回收。

    已确认行为¶

    • Entity TimeDilation 使用 entity 的 tick component additional scale source，不是直接覆盖全局 scale。
    • 多个 source 的合成委托 tick component；该 component 删除 source 后会重算剩余 source 乘积。
    • slotSpecialConfig 可让 Entity Frozen slot 的 duration 受 Global slot 影响。

    OnTick 行为¶

    每帧执行：

    1. _ClearTimeScaleHandle()
2. normalizedTime = elapsed / duration
3. transformed = time transform(normalizedTime)
4. timeScale = scaleCurve.Evaluate(transformed)
5. timeScale = max(0.0, timeScale)
6. owner.TimeScaleComponent.ApplyTimeScale(timeScale)
7. 保存 TimeScaleComponent 返回的 handleId

    关键点：

    • Entity instance 每帧先移除上一帧 handle，再 apply 新 handle。
    • 这说明 TimeScaleComponent 是多 source 合成，而不是覆盖一个字段。

    _ClearTimeScaleHandle¶

    if timeScaleHandleId != 0:
 owner.TimeScaleComponent.RemoveTimeScale(timeScaleHandleId)
 timeScaleHandleId = 0

    Slot 竞争¶

    Entity instance 只和同 target 且同 slot 的旧 instance 竞争；不同 target 的同 slot instance 可以共存。
