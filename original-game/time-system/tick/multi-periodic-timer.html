    MultiPeriodicTimer¶

    Fields¶

    字段 Offset 说明
    m_timers 0x10 List<PeriodicTimer>，通常表示多充能 CD。
    m_periodTime 0x18 当前周期秒数。

    Call Entrypoints¶

    • Skill.m_cooldownTimer 字段 offset 0x48。
    • Skill._OnCreate 状态 已确认 读取 castData.maxChargeTime 并构造 MultiPeriodicTimer(timerCnt=maxChargeTime, periodTime=Skill.get_cooldown(), waitFirstPeriod=false)。
    • Skill.CheckCd 状态 已确认 使用 oneReady / maxPassedTime < castData.startCdFrame / 30。
    • Skill.OnTick 状态 已确认 通过 cooldown recovery scalar 推进 MultiPeriodicTimer.Update(delta)。
    • Skill.DoCast 状态 已确认 调用 ResetOne(cooldown or overrideNextCd, true)，cast start 即启动一次 CD charge。
    • Skill.CastEnd 状态 已确认 在 cast 未经过 startCdFrame / 30 - epsilon 时调用 ResetOne(cooldown, false) 释放当前未 ready charge。

    Lifecycle¶

    MultiPeriodicTimer 是 Skill CD / 多充能 timer，不属于 TimeManager.TimerGroup。它由 Skill owner tick 手动推进。多 timer 并非全部同时减秒，而是每 tick 只推进当前 remaining 最小的未 ready timer。 - ResetOne(cd,true) 消耗一个 ready charge，使它进入完整 CD。 - ResetOne(cd,false) 会把一个未 ready charge 重置为 ready，适合 startCdFrame 前取消 cast 的 CD 释放。 - 空 timer 的 allReady=true，但 oneReady=false；原游戏 Skill._OnCreate 会按 maxChargeTime 构造 m_cooldownTimer，常规 skill 不依赖空 timer 作为 ready 状态。 - Skill.CheckCd 不是单纯 CD ready gate；在没有 ready charge 时，如果 running charge 的最大已过时间还小于 startCdFrame / 30，仍返回 true。
