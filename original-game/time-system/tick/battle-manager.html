    BattleManager¶

    BattleManager 是战斗运行时的聚合管理器，负责消费 TickOwner 链路传入的 delta，并推进 ATB、GlobalBuff、连携窗口、战斗记录等子系统。它不是 TimeDilation 状态的拥有者；时间缩放由 Tick/TimeManager 链路在进入 BattleManager 前完成。

    字段¶

    核心字段分组如下：

    分组 字段
    ATB m_atb, m_returnedAtb, atbMax, m_atbRecoverPauseTime, m_atbRecoverValue, m_atbRecoverRecordTime, s_atbRecoverModifierList
    战斗模式/敌人 m_curBattleMode, m_curBattleGame, m_enemies, m_inFightBossNum, m_pausedEnemies
    global modifiers m_globalModifiers, m_onAddBuffActionMap
    recorder/reporter BattleRecorder battleRecorder, ServerReporter serverReporter

    调用入口¶

    • TimeManager / tick root 调用 BattleManager.Tick(float deltaTime)；构造器确认 BattleManager 通过 BaseWorldManager 注册 Frame 与 TailLate 两条 tick function。
    • TimeManager.TailLateTick / tail-late root 调用 BattleManager.TailLateTick(float deltaTime)。
    • 技能、Action、资源消耗路径调用 CostAtb / GetAtbRecover 相关 API。
    • AbilitySystem.TacticalItem._Cast/_OnCastFinish 调用 CheckTargetIsNotInTacticalItemCastCd、StartTacticalItemCastTargetCdTimer、UseTacticalItem。
    • ability event 进入 TriggerComboSkillEvent；满足条件时由 PendingComboSkill 打开 pending combo window，或 immediate 分支直接调用 AbilitySystem.TryCastComboSkill。
    • 前端/UI 查询连携窗口时走 GetRemainComboSkillPendingTime，该查询同时计算 canCast，但 pending window 是否存在由 m_pendingComboSkill 决定。

    生命周期¶

    • 初始化/释放：OnInit()、OnRelease()。
    • 重置：Reset(bool resetEnemy = true)、ResetAtb()、_ResetComboSkill()。
    • 运行期：Tick(float deltaTime)、TailLateTick(float deltaTime)。

    Tick 是 battle 侧运行期聚合点。连携 pending record 的生命周期在 PendingComboSkill 创建/追加，在 _UpdatePendingComboSkill 按 delta 过期，在 CastPendingComboSkill cast 后清空。

    类型职责¶

    BattleManager 是 battle runtime 的 delta consumer，不是 TimeDilation 状态管理者。它消费 TickOwner 链路传入的 delta，然后推进战斗子系统。

    Tick 相关调用¶

    已确认 BattleManager.Tick 会继续分发 delta 到：

    • _UpdateAtb
    • _TickGlobalBuff
    • _UpdatePendingComboSkill

    • 其他 battle runtime 子系统 tick

    • 不应保留“大招专用暂停 BattleManager 子系统”的硬编码。

    • 大招动画期间，其他角色技能和 buff 应通过 effectiveTimeScale=0 空转。
    • TimeDilationManager 自身仍要 tick，以便大招动画结束后清理 Global UltimateTime instance。
