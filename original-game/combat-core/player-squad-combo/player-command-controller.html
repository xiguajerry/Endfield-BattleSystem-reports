    PlayerController.PlayerCommandController¶

    更新时间：2026-06-13

    归属¶

    PlayerCommandController 是 PlayerController 的嵌套类。它维护玩家命令对象、缓存、激活、消费和 blocker 检查。它不负责把 command 解析为具体 skillId。具体技能映射和 next skill request 在原游戏中由 AbilitySystem.ComboController 负责。

    字段¶

    字段 offset 说明
    m_controller 0x10 所属 PlayerController。
    m_cmdDic 0x18 PlayerCommandConfig.CmdType -> CommandBase 字典。

    相关嵌套类型¶

    类型 说明
    IPlayerCommandBlocker 提供 CheckBlockPlayerCommand、CheckConsumePlayerCommand、OnExecutePlayerCommand。
    CommandBase command 状态机基类。
    CommandBase.State Cached=0, Active=1, Invalid=2。
    MoveAxisCommand 有 m_moveAxis、UpdateAxis、OnActivate、OnConsume；已确认 delayed consume 中不覆盖 axis，activate 同步当前 player input，consume 清 axis。

    Release() 已确认 已定位 释放命令资源，待拆。
    IsValid(cmdType) 已确认 调用边界确认 检查 command 字典和 command state，细节待拆。

    CenterSkillState.CheckConsumePlayerCommand 已确认 已确认 只对 Dash(4) 特判；dash window 命中 consume；当前技能为 spec code 7 且 Skill.canDash == false 时 consume。
    CenterDashState.CheckConsumePlayerCommand 已确认 已确认 只对 Attack(2) 特判；dash attack window 命中 consume。
    CenterDeadState.CheckBlockPlayerCommand / CheckConsumePlayerCommand 已确认 / 已确认 已确认 默认均返回 true。
    CenterDummyState.CheckBlockPlayerCommand / CheckConsumePlayerCommand 已确认 / 已确认 已确认 默认均返回 true。
    ConsumeAllCommand(force) 已确认 已定位 批量 consume 所有 command。
    CommandBase.Activate() 已确认 已确认 仅 Cached(0) command 转为 Active(1)，随后触发 OnActivate 虚回调；Active/Invalid 不重复激活。
    CommandBase.Tick(deltaTime) 已确认 已确认 Invalid 直接返回；Active+Instant 立即 force consume；Instant cached 累计 m_time 到 cacheTime 后 consume；Continuous 累计 m_time，若 m_delayToConsumeTime 有值则递减，到期 force consume。
    CommandBase.Init(data) 已确认 已确认 只写 _config，不 reset；CreateCommand<T> 随后显式调用 Reset()。
    CommandBase.Reset() 已确认 已确认 清 m_time、m_delayToConsumeTime、isBlocked，state 回到 Cached(0)，记录当前主控角色并触发 OnReset。
    CommandBase.Consume(force) 已确认 已确认 force 或非 Continuous 直接 Invalid(2) 并触发 OnConsume；Continuous 非 force 时，m_time <= enableDelayConsumeTime 立即 consume，否则写 m_delayToConsumeTime=delayConsumeTime。
    PlayerCommandController.CreateCommand<T> 已确认 已确认 查 m_cmdDic；已有 command 则 Reset() 并按 immediatelyActivate 可选 Activate() 后返回；不存在则 Activator.CreateInstance<T>()，从 PlayerCommandConfig.config 取 PlayerCommandData，Init(data)、Reset()、可选 Activate()，最后 Add 到 m_cmdDic。
    PlayerCommandController.GetCommand<T> 已确认 已确认 查 m_cmdDic；只有 command 存在且 m_state == Active(1) 时返回，否则返回 null。
    PlayerCommandController.ConsumeAllCommand(force) 已确认 已确认 遍历 m_cmdDic.Values；只对 m_state != Invalid(2) 的 command 调 CommandBase.Consume(force)。
    PlayerCommandController.Release() 已确认 已确认 遍历 m_cmdDic.Values 调 CommandBase.Reset()，随后清 m_controller。未看到清空 m_cmdDic。
    MoveAxisCommand.UpdateAxis 已确认 已确认 m_delayToConsumeTime.HasValue == false 时写入 m_moveAxis；delay consume 中忽略新 axis。
    MoveAxisCommand.OnActivate 已确认 已确认 读取 GameInstance.playerController 当前 input axis 并调用 UpdateAxis。
    MoveAxisCommand.OnConsume 已确认 已确认 m_moveAxis = Vector2.zero。
    SkillCommand.AssignData 已确认 已确认 owner 非空；若 dictionary 已有 owner data，先 SkillCastInputData.Dispose()；随后 SkillCastInputData.CopyFrom(inputView) 并 set dictionary item。
    SkillCommand.TryGetSkillInputData 已确认 已确认 owner 非空且 dictionary 命中时，将 SkillCastInputData 构造成 SkillCastInputView 输出并返回 true；否则清零输出并返回 false。
    SkillCommand.OnReset 已确认 已确认 遍历 dictionary values 调 SkillCastInputData.Dispose()，然后 Clear()。

    调用入口¶

    • PlayerController tick 调用 PlayerCommandController.OnTick。
    • input 层调用 Execute(cmdType) 激活/执行某个 command。
    • command state 更新中会调用 _PreprocessCommand、CommandBase.Consume 和 blocker 检查。

    生命周期¶

    1. Init 绑定 PlayerController 并准备 command 字典。
    2. 输入到来时对应 CommandBase 进入 cached/active 状态。
    3. OnTick 推进 command 状态和消费逻辑。
    4. Execute 在 command active 时通知主控 AI 与 bark 系统。
    5. Consume / ConsumeAllCommand 将命令消费或失效。

    _UpdateCmdState 已确认顺序¶

    1. controller 可用时先调用 _PreprocessCommand()。
    2. 遍历 m_cmdDic 的 PlayerCommandConfig.CmdType -> CommandBase entries。
    3. 对每个 command，先经过当前 center state 的 command consume 检查；命中时调用 CommandBase.Consume(true) 并继续下一项。
    4. 调用 PlayerController.get_blockPlayerInput()；全局输入 block 命中时设置 command.isBlocked = true。
    5. 未全局 block 时继续执行 command blocker / consume blocker 检查；PlayerCommandController 自身只转发到 CharacterMainCtrlSystem，实际规则由当前 CenterStateBase 虚方法决定。
    6. 未被 block/consume 的 command 调用 CommandBase.Activate()，写 isBlocked = false，随后调用 PlayerCommandController.Execute(cmdType)。
    7. block path 写 isBlocked = true。
    8. activate/block path 最后都会调用 CommandBase.Tick(deltaTime)。

    这些结论确认了 PlayerCommandController 的职责是 command object 状态推进、consume/block/activate 和 Execute 通知；具体 skillId 映射仍不在该类。

    已确认行为¶

    • Execute 不解析 skillId，也不直接 cast skill。
    • command active state 为 1 时才触发 CharacterMainCtrlSystem.OnExecutePlayerCommand 与 BarkSystem.TriggerMainCharCmd。
    • command 的缓存和消费是 PlayerCommandController 职责，技能映射是 AbilitySystem.ComboController 职责。
    • _PreprocessCommand 的已确认主逻辑是 Jump 消费 Dash/Attack：Jump(1) 为 Cached/Active 时，Dash(4) 与 Attack(2) 为 Cached/Active 会被 Consume(true)。
    • PlayerCommandConfig.CmdType 数值：StartMove=0、Jump=1、Attack=2、Skill=3、Dash=4、MoveAxis=5、Enum=6。
    • _UpdateCmdState 每个 command 的 tick 是 activate/block 后的统一尾步骤，不是只在成功执行时 tick。
    • CommandBase.Reset 会保存当时的 main character；Instant cached command 的 tick 会在主控变化或缓存时间过期后 consume。
    • CommandBase.Consume(false) 对 Continuous command 支持延迟消费；这是 _UpdateCmdState 不能只用 queue frame 过期替代的原因。
    • CommandBase.Init 不 reset；CreateCommand<T> 的创建路径是 Init -> Reset -> optional Activate -> Add。
    • CreateCommand<T> 重复调用同一个 cmdType 会复用 m_cmdDic 中已有 command，并先 reset 其状态。
    • GetCommand<T> 是 active-only 查询，不是 raw dictionary lookup。
    • ConsumeAllCommand 跳过已经 Invalid 的 command，避免重复触发 consume。
    • Release reset command objects 并清 controller，不清 dictionary。
    • _CheckBlockPlayerCommand / _CheckConsumePlayerCommand 不在 PlayerCommandController 内部实现具体规则，只桥接到 GameWorld.aiManager.characterMainCtrlSystem。
    • CharacterMainCtrlSystem.CheckBlockPlayerCommand / CheckConsumePlayerCommand 要求 center state machine running 且 current state 是 CenterStateBase；否则返回 false。slot 8 对应 block，slot 9 对应 consume。
    • CenterDeadState / CenterDummyState 对 block/consume 均返回 true。
    • CenterSkillState.CheckBlockPlayerCommand 只在 Jump 且当前技能不可 interrupt 时 block；CheckConsumePlayerCommand 只在 Dash 且 dash window / current skill canDash=false 时 consume。
    • CenterDashState.CheckConsumePlayerCommand 只在 Attack 且 dash attack window 命中时 consume。
    • MoveAxisCommand.UpdateAxis 在 delayed consume 中不覆盖 axis；OnActivate 同步当前 player input axis；OnConsume 清零 axis。
    • SkillCommand.AssignData 替换已有 owner input 前会 dispose old data；OnReset dispose all + clear。
