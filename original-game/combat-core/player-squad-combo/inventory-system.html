    InventorySystem¶

    更新时间：2026-06-11

    归属¶

    InventorySystem 是玩家道具背包和物品使用 owner。对 tactical item 来说，它负责：

    • 查询背包或试用角色 tactical item 总数。
    • 在 tactical item cast 期间 hold / release 当前 item，避免可用数量被重复占用。
    • UseItem 成功路径发送使用协议，或对特定 useType 直接本地 _UseItemInternal。

    战斗内 tactical item 使用次数仍归 BattleManager.EquippedTacticalItemRuntimeData；item buff 应用归 AbilitySystem.TacticalItemController / InventorySystem._UseItemInternal 的具体 action path。

    字段¶

    字段 offset 说明
    itemBag 0x70 scope-aware item bag。
    trialCharItemDepot 0x148 trial char item depot。

    UseItem 闭包字段：

    字段 offset 说明
    __4__this 0x10 InventorySystem。
    scope 0x18 使用 scope。
    itemId 0x20 item id。
    targetCharInstList 0x28 目标角色 inst id 列表。
    userInstId 0x30 使用者角色 inst id。
    userChar 0x38 使用者 AbilitySystem。
    targetCharInstId 0x50 revive path 的目标 inst id。

    方法¶

    方法 状态 状态 结论
    CanUseItem(Scope,string,int,int) 已确认 已确认 UseItem 的前置 gate；确认 table/slot/count、system/game-mode/UI toast、MainCharCastSkill -> AbilitySystem.CanCastItemSkill、uiType HP/USP/dead/alive 和 throw current-skill gate。
    SendUseItemMsg(Scope,string,int,List<ulong>,ulong,ulong) 已确认 已定位 普通物品使用协议发送入口；UseItem 默认成功路径调用它。
    _HandleUseItem(SC_ITEM_BAG_USE_ITEM) 已确认 已定位 服务器使用回包处理；尚未展开。
    _UseItemInternal(string,List<ulong>,ulong) 已确认 已确认 本地应用 item use action；被 UseItem(useType==2) 调用。确认 persistent buff 注册、target AbilitySystem SetAsItemUseTarget、MainCharCastSkill -> main character TryCastSkillOnce(skillBBData) 分支。
    HoldTacticalItem(Scope,string,int) 已确认 已确认 增加 tactical item held count；TacticalItem._Cast 调用。
    ReleaseTacticalItem(Scope,string,int) 已确认 已确认 减少 held count，减到 <= 0 移除 item key；TacticalItem.StopUse / _OnCastFinish 调用。
    GetAvailableTacticalItemCount(Scope,string,ulong) 已确认 已确认 返回 tactical item total count 减 held count；试用角色直接返回 trial count。
    GetTacticalItemCount(Scope,string,ulong) 已确认 已确认 非试用角色走 GetItemCountInBag(scope,itemId)；试用角色返回 trialData 内 count。
    GetItemCountInBag(Scope,string) 已确认 已定位 普通背包数量查询；被 GetTacticalItemCount 调用。

    调用入口¶

    • AbilitySystem.TacticalItem._OnEventTrigger 调 GetAvailableTacticalItemCount(scope,itemId,userInstId) 判断库存可用。
    • AbilitySystem.TacticalItem._Cast 调 HoldTacticalItem(scope,itemId,1)。
    • AbilitySystem.TacticalItem.StopUse 和 _OnCastFinish 调 ReleaseTacticalItem(scope,itemId,1)。
    • BattleManager.UseTacticalItem 调 InventorySystem.UseItem(scope,itemId,targetCharIndex,userCharIndex)。
    • UseItem 的普通路径通过 SendUseItemMsg 等服务器回包；useType == 2 直接 _UseItemInternal。

    生命周期¶

    1. tactical item 条件满足并开始 cast 时，TacticalItem._Cast hold 一份 item。
    2. cast 被停止或结束时 release held item。
    3. _OnEventTrigger 的可用数量读取总数并扣除 held count，避免同一 item 在 cast 中被重复判定可用。
    4. cast 完成后 BattleManager.UseTacticalItem 统一调用 InventorySystem.UseItem。
    5. UseItem 成功后返回 true，BattleManager 扣 battle runtime remaining count 并记录 recorder。

    已确认行为¶

    • HoldTacticalItem：
    • m_holdItemDict 没有 scope key 时创建 Dictionary<string,int> 并写入。
    • scope 字典没有 item key 时先写 0。
    • 最后把 held count 加上 count。
    • ReleaseTacticalItem：
    • scope key 不存在或 item key 不存在时直接返回。
    • 读取当前 held count，减去 count 后写回。
    • 若新 held count <= 0，移除该 item key。
    • GetTacticalItemCount：
    • 通过 GameInstance.player.CharBagSystem.GetCharInfo(equippedCharInstId) 判断装备角色。
    • 若 CharInfo.charType == 1 且 trialData 存在，返回 trial data offset 0x1C 的 tactical item count。
    • 否则调用 GetItemCountInBag(scope,itemId)。
    • GetAvailableTacticalItemCount：
    • 先调用 GetTacticalItemCount(scope,itemId,userInstId) 得总数。
    • 若装备角色是 trial char，直接返回总数，不扣 held count。
    • 非 trial char 时，如果 m_holdItemDict[scope][itemId] 存在，返回 total - heldCount。
    • 没有 held count 时返回 total。
    • UseItem：
    • 创建 display class 保存 this/scope/itemId。
    • 先调用 CanUseItem(scope,itemId,targetCharIndex,userCharIndex)；false 时直接返回 false。
    • targetCharIndex >= 0 时，从当前 squad slots 取目标 slot，把 slot char inst id 加入 targetCharInstList。
    • userCharIndex >= 0 时，从当前 squad slots 取 user slot，保存 userInstId。
    • 读取 UseItemTable[itemId]。
    • useType == 4 且 target/user index 都有效时，会取 user AbilitySystem、target inst id、revive position，并调用 BattleManager.PushRevivePosition，同时把 target inst id 加入 BattleManager.inReviveCharInstList。
    • 默认成功路径调用 SendUseItemMsg(scope,itemId,count=1,targetCharInstList,userInst,bombInstId=0)。
    • useType == 2 路径调用 _UseItemInternal(itemId,targetCharInstList,userInstId)，随后通过 InteractiveInstigatorControlComponent.CacheUseItemInfo(scope,itemId) 缓存交互使用信息。
    • 成功路径返回 true；CanUseItem false 返回 false。
    • CanUseItem：
    • 先读取 UseItemTable[itemId]；缺表失败。
    • targetIndex >= 0 时解析 squad slot / equipped char inst id，并调用 GetTacticalItemCount(scope,itemId,equippedCharInstId)；数量为 0 时失败。
    • 读取 ItemUseData.useActions；存在 useType == MainCharCastSkill(1) 时调用 main character AbilitySystem.CanCastItemSkill()，失败则拒绝使用。
    • uiType == Throw(2) 分支检查当前技能 interrupt gate；失败则拒绝使用。
    • uiType == SingleUsp(5) 要求目标存在、目标存活且当前 USP 小于最大 USP；否则按 no-target/max-usp 路径失败。
    • uiType == AllUsp(6) 遍历 squad，任一存活成员 USP 小于最大 USP 即允许，否则 max-usp 失败。
    • uiType == AllHeal(1) 遍历 squad，任一存活成员 HP 小于最大 HP 即允许，否则 max-hp 失败。
    • uiType == SingleHeal(0) 目标分支比较目标 HP / MaxHP；已满血失败。
    • uiType == Revive(4) 分支涉及 source slot CharInfo 私有状态字节；当前只确认与目标/死亡状态相关，字段名待补。
    • _UseItemInternal：
    • 先查 UseItemTable[itemTemplateId]；没有数据时直接结束。
    • 若 itemData.isPersistentBuff == true 且 itemData.effectType == 2 (Buff)，调用 BattleManager.RegisterPersistentTacticalItem(itemData,targetCharInsts,-1.0)，随后 flush ServerReporter。
    • 非 persistent 且 effectType != 1 (Revive) 时分配 List<AbilitySystem>，把 targetCharInsts 中能从 SquadManager 找到的角色 AbilitySystem 加入列表。
    • 从 sourceCharInst 解析 source AbilitySystem；若目标列表为空，则取 PlayerController.mainCharacter.abilityCom 调 AbilitySystem.SetAsItemUseTarget(itemData,sourceAbilitySystem)。
    • 遍历目标 AbilitySystem 列表，对每个目标调 SetAsItemUseTarget(itemData,sourceAbilitySystem)。
    • 再遍历 ItemUseData.useActions。当 UseActionBundle.useType == 1 (MainCharCastSkill) 时：
    • 取 PlayerController.mainCharacter.abilityCom。
    • 如果当前 skill 为空，或 currentSkill.canInterrupt == true，读取 UseActionBundle.skillBBData。
    • 调用 AbilitySystem.TryCastSkillOnce(skillBBData,target=0,callback=0,options=null,afterCast=0)。
    • TryCastSkillOnce 返回值没有被 _UseItemInternal 用来回滚或中止后续流程。
    • 末尾调用 BattleManager.ServerReporter.Flush()。
