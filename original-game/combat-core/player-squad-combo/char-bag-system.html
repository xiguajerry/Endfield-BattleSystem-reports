    CharBagSystem¶

    更新时间：2026-06-11

    归属¶

    CharBagSystem 是玩家角色包/队伍数据 owner。它保存 CharInfo，其中包含角色等级、技能等级、天赋、潜能、武器、装备、tactical item、server battle info 和战斗 skill data cache 等数据。对 tactical item 来说，它保存每个 CharInfo.tacticalItemId，并处理客户端装备请求和服务器装备变更回包。 这不是 battle runtime owner。战斗内的使用次数、recover timer、target cast cd 和实际使用记录归 BattleManager；战术物品 cast/runtime 归 AbilitySystem.TacticalItem。

    字段¶

    字段 offset 说明
    m_scopedTemplateId2Char 0x70 按 scope/char type/template id 索引角色。
    m_scopedCharLists 0x78 scope 内角色列表。
    m_scopedTeams 0x80 scope 内队伍列表。
    _charInfos_k__BackingField 0xB0 Dictionary<ulong, CharInfo>。
    m_clientCharInfos 0xC8 client-side char info cache。
    m_clientItemBundles 0xD0 client-side item bundle cache。

    CharInfo 已确认字段：

    字段 offset 说明
    instId 0x10 角色实例 id。
    templateId 0x18 角色模板 id。
    charType 0x24 角色类型；trial path 使用该字段区分。
    trialData 0x30 试用角色数据。
    level 0x44 角色等级。
    skillGroupLevelInfoList 0x50 技能组等级列表。
    talentInfo 0x58 天赋节点状态。
    potentialLevel 0x60 潜能等级。
    weaponInstId 0x70 武器实例 id。
    equipCol 0x78 装备槽位到装备实例 id。
    equipSuit 0x80 套装统计。
    tacticalItemId 0x88 普通角色装备的 tactical item string id。
    isDead 0x90 AbilitySystem.TacticalItem.ConditionChecker.Check 的 OnDie 分支读取该字段，要求目标 CharInfo.isDead == true。
    serverEntityInfo 0x98 server entity hp / ultimateSp mirror。
    serverBattleInfo 0xA0 server battle info。
    serverGeneration 0xA8 server generation。
    battleInstId 0xB0 battle instance id。
    attributesPack 0xB8 attribute pack。
    talentAttributeModifierList 0xD0 talent attribute modifiers。
    potentialAttributeModifier 0xD8 potential attribute modifier。
    weaponAttributeModifier 0xE0 weapon attribute modifier。
    equipAttributeModifierList 0xE8 equip attribute modifiers。
    cardSkills 0xF0 card skill list/cache。
    serverCardSkillDict 0xF8 server card skill map。
    activeSkillDataList 0x108 active skill data list。
    serverSkillDataDict 0x110 server skill data map。

    方法¶

    方法 状态 状态 结论
    GetCharInfo(ulong) 已确认 已确认 BattleManager / InventorySystem / CharBag 回包路径都用它按 char inst id 读取 CharInfo。
    PutOnTacticalItem(ulong serverId, string itemStrId) 已确认 已确认 客户端装备入口；把 item string id 通过 Tables.itemIdToNum 转成数值 id，构造 CS_EQUIP_MEDICINE_MODIFY 并 SendMsg。
    _Handle_ChangeTacticalItem(SC_EQUIP_MEDICINE_MODIFY) 已确认 已确认 服务器装备变更回包；按 char id 取 CharInfo，把数值 item id 通过 Tables.itemIdToStr 转回 string，写入 CharInfo.tacticalItemId。
    _Handle_TrialCharTacticalItemModify(SC_TRIAL_CHARACTER_EQUIP_MEDICINE_MODIFY) 已确认 已确认 试用角色 tactical item 修改；只在 CharInfo.charType == 1 且 trialData 存在时写 trial data 内 tactical item 数值字段。
    CharInfo.isDead read path 已确认 已确认 by tactical item caller AbilitySystem.TacticalItem.ConditionChecker.Check 的 condType == OnDie 分支读取 CharBagSystem.GetCharInfo(charInstId).isDead。

    调用入口¶

    • 玩家或 UI 装备 tactical item 时调用 PutOnTacticalItem。
    • 服务器回包调用 _Handle_ChangeTacticalItem 或 _Handle_TrialCharTacticalItemModify。
    • BattleManager._ResetEquippedTacticalItemRuntimeData 通过 GetCharInfo(slot.charId).tacticalItemId 重建 battle runtime tactical item data。
    • InventorySystem.GetTacticalItemCount / GetAvailableTacticalItemCount 也通过 GetCharInfo(equippedCharInstId) 处理试用角色 count。
    • AbilitySystem.TacticalItem.ConditionChecker.Check 的 OnDie 分支通过 GetCharInfo(targetInstId).isDead 判定死亡目标。
    • AbilitySystem._CreateSkill / SkillUtil.CreateSkillDataWithCharInfo 角色分支通过 CharInfo 获取角色 skill data / server skill data。
    • 装备、武器、天赋、潜能 attribute modifier attach/detach 都以 CharInfo 为 owner。

    生命周期¶

    1. 非战斗层 CharBagSystem 保存角色装备状态。
    2. 装备请求发送后，由服务器回包同步到 CharInfo.tacticalItemId 或 trial data。
    3. 进入/重建战斗 tactical runtime 时，BattleManager 读取当前 squad slot 的 CharInfo.tacticalItemId。
    4. 战斗中数量、cooldown、target cast cd、item buff lifetime 不再由 CharBagSystem 维护。
    5. 角色装备/武器/天赋/潜能 modifier attach 后保存在 CharInfo，供 attribute / SkillData 计算路径使用。

    已确认行为¶

    • PutOnTacticalItem：
    • 查询 Tables.itemIdToNum.TryGetValue(itemStrId, out value)。
    • item 不存在时数值 id 被置为 0。
    • 构造 CS_EQUIP_MEDICINE_MODIFY，写入 char serverId 和数值 item id 后 BasePlayerManager.SendMsg(msg, null)。
    • _Handle_ChangeTacticalItem：
    • 创建 ScopeName=1 对应 scope。
    • 从 msg offset 0x18 读 char id，调用 GetCharInfo。
    • 从 msg offset 0x20 读数值 item id，走 Tables.itemIdToStr.TryGetValue。
    • 找不到时写空字符串。
    • 写入 CharInfo.tacticalItemId offset 0x88。
    • 调 CharInfo.OnReplicatedDataChange(changeType=7)。
    • 遍历当前 squad slot；若 slot char id 匹配回包 char id，发送一个带 slot index 和 item id 的 global event。
    • 最后发送另一个按 char inst id + item id 的 global event，并 _WriteEventLog()。
    • _Handle_TrialCharTacticalItemModify：
    • 从 msg 取 char id 后 GetCharInfo。
    • 仅当 CharInfo.charType == 1 才处理。
    • 从 CharInfo.trialData 读取 trial data，写入 msg 中的 tactical item 数值 id 到 trial data offset 0x1C。
    • CharInfo.isDead：
    • TacticalItem OnDie condition 使用该字段，而不是只看 Entity runtime dead flag。
