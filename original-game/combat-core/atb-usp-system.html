    ATB 与 USP 资源管理系统¶

    概述¶

    ATB (Active Time Battle / 技力) 和 USP (Ultimate Skill Points / 终结技能量) 是 Endfield 战斗系统的两项核心资源:

    • ATB — 角色释放技能所需的技力值，随时间自动恢复，消耗后按比例转化为 USP
    • USP — 终结技 (大招) 的能量，主要通过消耗 ATB 获得

    SkillSetting 配置¶

    SkillSetting 是 Unity ScriptableObject 序列化资产 (Misc/assets/.../SkillSetting.json)，包含全局战斗配置。

    ATB 相关字段¶

    字段 值 说明
    atbMax 300.0 ATB 上限 (3格 × 100)
    atbPerCell 100 每格 ATB 值 (UI 3格显示)
    atbRecover 8.0 每秒 ATB 恢复量
    atbRecoverInterval 0.5 恢复间隔(秒)
    atbGainEfficiency 1.0 获得效率倍率
    atbConsumedDefaultUspGainSelf 0.065 消耗 ATB 时自身 USP 转化率 (6.5%)
    atbConsumedDefaultUspGainOther 0.065 消耗 ATB 时他人 USP 转化率

    USP 相关字段¶

    字段 值 说明
    ultimateSpRecover 0.0 终结技能量基础恢复 (为0，通过消耗ATB获得)
    skillPressingUltimateTime 0.25 大招按键确认时间(秒)
    skillPressingCancelTime 0.75 大招取消时间(秒)

    连携技时序¶

    字段 值 说明
    comboSkillPendingInterval 6.0 连携技触发等待时间(秒)
    comboSkillTriggerUIInterval 1.0 UI 触发间隔(秒)
    comboSkillUseWaitTime 1.3 连携技使用等待时间(秒)

    CostAtb 消耗流程¶

    消耗逻辑:

    1. 先从 m_returnedAtb (可返还池) 中扣除
    2. 不足部分从 m_atb 中扣除
    3. 返回 nonReturnedAtbCost (未能从返还池覆盖的部分)
    4. 根据 atbConsumedDefaultUspGainSelf 将消耗的 ATB 转化为施法者 USP，并根据 atbConsumedDefaultUspGainOther 转化为其他小队成员 USP

    ATB 逐帧恢复¶

    每帧恢复 atbRecover / 60 = 8.0 / 60 ≈ 0.133 ATB。

    技能消耗执行¶

    此前 CheckCost() 永远返回 true，_ApplyCost() 为空操作，冷却时间也未启动。现已全部补齐。

    关键修复: 技能冷却启动¶

    此前 MultiPeriodicTimer 只在 DoEnable()/DoDisable() 中被 reset() (清零)，从未在技能施放时用实际冷却值 start()。 现在在 _ApplyCost() 中:

    _ApplyCost() 完成资源消耗后会派发 AbilitySystem.Event.OnAfterSkillApplyCost，并携带当前 SkillCastInfo。该事件用于 passive/buff 在“技能已经应用消耗”之后继续判断技能 id、技能类型和消耗上下文。

    公式汇总¶

    计算项 公式
    每帧 ATB 恢复 atbRecover / 60
    ATB → USP 转化（自身） actualCost × atbConsumedDefaultUspGainSelf (0.065)
    ATB → USP 转化（其他小队成员） actualCost × atbConsumedDefaultUspGainOther (0.065)
    ATB 上限 atbMax (300.0)
    USP 上限 AttributeTable[MaxUltimateSp]
