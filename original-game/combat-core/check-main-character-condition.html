    Beyond.Gameplay.Core.Conditions.CheckMainCharacterCondition¶

    类定位¶

    • MemoryPack union tag: 89

    已确认 实际是 Data.get_actionType，真正执行逻辑在 已确认。

    数据结构¶

    Data 只有一个业务字段:

    字段 偏移 类型 作用
    checkTarget 0x20 TargetSettings 要检查的目标来源/选择器

    没有 compare type、反向开关或额外参数。需要取反时，配置通常通过外层条件动作 例如 NotNextCheckAction 实现。

    空值与多目标行为¶

    关键行为:

    • GetTargetsView 返回的目标组只取 firstTarget。
    • firstTarget == null 时返回 false。

    • m_data == null、TargetHandle 为空、GameInstance.playerController 为空等路径走

    • 多目标场景下不会遍历所有目标；只有第一个目标参与判断。

    因此如果 checkTarget 来自一个 target group，配置语义应理解为“该组第一个目标是否为 当前主控角色”，不是“组内任意目标是否为当前主控角色”。

    PlayerController.mainCharacter 来源¶

    GameInstance.get_playerController() 返回 GameInstance.instance.m_playerController。 CheckMainCharacterCondition 直接读取 PlayerController.mainCharacter，没有额外缓存层。 PlayerController.mainCharacter 的字段写入由 PlayerController.SetMainCharacter 完成； 但真正决定主控实体的是 SquadManager/Squad 的 leader 系统，入口最终统一回到:

    SquadManager._SetMainCharacter(newLeader, reason)
 -> PlayerController.SetMainCharacter(newLeader, reason)

    类级细节已经按类拆分:

    • PlayerController mainCharacter 维护
    • SquadManager / Squad 主控维护

    语义总结¶

    CheckMainCharacterCondition 是一个纯条件 action:

    • 输入: checkTarget。
    • 解析: 通过 AbilityActionUtils.GetTargetsView 得到目标组。
    • 判定: 只取 firstTarget，比较 firstTarget.entity 和 PlayerController.mainCharacter。
    • 输出: 相同返回 true，第一个目标为空返回 false。
    • 不负责查表、改状态、同步网络或缓存。

    使用时最需要注意的是两点:

    • 它判断的是“当前主控角色”，不是玩家阵营、队员或本地玩家拥有者。

    • 目标为空或当前没有主控角色时返回 false。

    • NotNextCheckAction + CheckMainCharacterCondition 已按前缀取反语义支持，可表达“目标不是当前主控角色”。
