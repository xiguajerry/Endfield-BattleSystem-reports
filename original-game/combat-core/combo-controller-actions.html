    ComboController / AllowNextSkillAction / ComboCacheAction¶

    结论¶

    ComboController 是 AbilitySystem 里负责“输入命令到技能”的运行时路由层。它不执行技能本身，而是维护三类状态:

    • BattleCommandType -> skillId 的基础映射、运行时临时映射、最终生效映射。
    • 玩家输入后生成的 NextSkillRequest，包含下一技能、缓存时间、是否已经允许释放。
    • 当前技能/BUFF 打开的 AllowedNextSkillPack 白名单，用于放宽“当前技能能否被新技能打断”的判断。

    ComboCacheAction 往 ComboController 注册临时 command mapping。例如普攻 1 段在时间轴上注册:

    {
 "$type": "Beyond.Gameplay.Core.ComboCacheAction+Data, Gameplay.Beyond",
 "mappingDataList": [
 {
 "cmdType": "Attack",
 "skillId": "chr_0012_avywen_attack2",
 "cacheEndByAction": true,
 "overrideCacheTime": false
 }
 ]
}

    这表示这段 Action 生效期间，Attack 命令优先映射到 attack2。 AllowNextSkillAction 往 ComboController 注册允许接续列表。例如同一技能稍晚的窗口注册:

    {
 "$type": "Beyond.Gameplay.Core.AllowNextSkillAction+Data, Gameplay.Beyond",
 "allowedSkillIdList": [
 "chr_0012_avywen_attack2"
 ]
}

    这表示当前技能还没自然结束时，attack2 也可以通过 CheckCanInterruptCurSkill() 的白名单检查。 所以连段接续通常不是一个单点开关，而是两个窗口叠在一起:

    ComboCacheAction: 这个按键现在要缓存/映射到哪个 skillId
AllowNextSkillAction: 这个 skillId 此时能不能打断当前技能

    Handle.RemoveModifier¶

    MappingModifier.Handle.RemoveModifier() 不只是移除 mapping:

    ComboCacheAction.Data.BattleCmdMappingModifierData.cacheEndByAction 会传给 clearSkillRequestOnRemove。因此当这个字段为 true 时，Action 结束不仅撤销映射，还会清掉由该 modifier 生成的 NextSkillRequest。

    • clearOffsetTargetSkillIdOnEnd 现在会把当前 mapping 的 skillId 记录为轻量 offset target。
    • MappingModifier.Handle.remove() 会在 clearOffsetTargetSkillIdOnRemove=true 时调用轻量 ClearOffsetModifier(true)，移除 offset pack 并清空 offset target。
    • RefreshOffsetTargetSkillId 会跳过 dodge、plunging start 和 plunging end；normal attack list 为空时直接返回。
    • RefreshOffsetMappingModifier 会为 Attack 创建或续期 Priority.Offset mapping；若此前未记录 offset target，会回退普通攻击链解析结果。
    • Skill._ResetOffsetSkillId 会在技能开始时把 m_refreshedOffsetSkillId=false；只有 SkillType.Attack 且不是 plunging start/end 时才通知 AbilitySystem.ComboControllerOnSkillCastStart(skillId)。
    • OnSkillCastStart 会无条件把 m_offsetTargetSkillId 写成当前 skillId；如果存在 m_offsetModifierPack，会把 pack 的 skillCasted 改成 true。native 没有检查当前 skill 是否等于旧 offset target 或 trigger skill。

    对普攻打断规则的意义¶

    这套实现解释了“普攻被下落攻击、冲刺、战技或连携技打断后，下次普攻不一定从第一段开始”的机制:

    1. 段数不是 PlayerController 的计数器。
    2. 当前/后续动作的时间轴会用 ComboCacheAction 注册 Attack -> 下一段 skillId。
    3. AllowNextSkillAction 决定该 skillId 在当前技能期间是否能打断释放。
    4. 玩家输入只刷新 NextSkillRequest，真正是否可放由每帧 _AllowNextSkill() 判断。
    5. cacheEndByAction == true 时，Action 结束会清理由该 mapping 产生的 request；所以“不中断连段”依赖的是后续动作/BUFF/offset 是否还在注册正确 mapping，而不是旧 request 永久保存。

    更短地说:

    ComboCacheAction 决定按键缓存到哪一段。
AllowNextSkillAction 决定那一段是否能插入当前动作。
ComboController.Tick 把两者合成可消费的 nextSkillId。
