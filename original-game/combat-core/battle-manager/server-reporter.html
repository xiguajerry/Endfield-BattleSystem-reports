    BattleManager.ServerReporter¶

    状态：documented（core lifecycle / upload gate / generic CreateServerOp<T> / CreateServerActionOp / CreateModifyEntityValueOp / CreateUpdateAtbInfoOp / RefreshModifyEntityValueOp 已确认; op builders partial）

    原游戏来源¶

    • 原游戏类：Beyond.Gameplay.Core.BattleManager.ServerReporter

    字段¶

    字段 类型 说明
    debugMode bool offset 0x10; ctor 置 false。
    m_manager BattleManager offset 0x18; owning BattleManager。
    m_cachedOp List<BATTLE_CLIENT_OP_DATA> offset 0x20; 缓存的客户端 battle op。
    m_lastUploadTimestamp float offset 0x28; 上次上传时间。
    m_msgList List<CS_BATTLE_OP> offset 0x30; 待上传消息。
    m_triggerMaxCountFrameLastFrame bool offset 0x38; 上传数量/帧相关状态。

    调用入口¶

    • BattleManager.Tick 尾部调用 ServerReporter.TryUpLoad(false)。
    • InventorySystem._UseItemInternal 末尾调用 ServerReporter.Flush()。

    生命周期¶

    • 随 BattleManager 创建；ctor 创建 m_cachedOp / m_msgList 并保存 manager。
    • battle op 产生时缓存到 m_cachedOp。
    • tick 时 TryUpLoad(false) 受 _CheckCanUpload gate 控制。
    • item local use 或强制上传调用 Flush()，等价 TryUpLoad(true)。
    • Release() 清空 cached op 和 msg list。

    已确认行为¶

    Constructor¶

    确认状态：已确认 已确认：

    • 创建 List<BATTLE_CLIENT_OP_DATA> 写入 this + 0x20。
    • 创建 List<CS_BATTLE_OP> 写入 this + 0x30。
    • 写入 owning BattleManager 到 this + 0x18。
    • debugMode 字节 this + 0x10 置 false。
    • 为 NetWorkMsgPoolSend<CS_BATTLE_OP> 设置两个 static callback。

    CreateServerOp<T>¶

    确认状态：已确认（CreateServerOp<System.Object> specialization） 参数：

    • opType: BATTLE_ACTION_OPERATE_TYPE，Windows x64 edx，写入 BATTLE_CLIENT_OP_DATA.OpType。
    • opData: typed proto payload，Windows x64 r8。
    • ownerInfo: BattleManager.ServerReporter.OwnerInfo*，Windows x64 r9。

    native 主路径：

    1. 通过 ownerInfo.serverId 调 GameWorld.ObjectContainer.GetEntityById(...)。
    2. 若 owner entity 存在、entity.charCtrl 存在且 entity.charCtrl.isScriptControlled == true（Entity.charCtrl offset 0x168，CharacterController.isScriptControlled offset 0x1A2），直接 return，不创建 op。
    3. BattleProtoAllocator.Allocate<BATTLE_CLIENT_OP_DATA>()。
    4. 写 header：
    5. clientOp.OpType = opType（offset +0x20）。
    6. clientOp.OwnerId = ownerInfo.battleInstId（ownerInfo+0x08 -> op+0x18）。注意 proto 字段名是 OwnerId，但 native 写入的是 battle inst id。
    7. clientOp.Generation = ownerInfo.generation（ownerInfo+0x10 -> op+0x24）。
    8. clientOp.ChapterMapId = current GameWorld map/chapter state（observed GameWorld + 0x1E8 -> +0xB0 -> op+0x28）。
    9. clientOp.SeqId = GameInstance.gameplayNetwork.GetNextBattleSeqId()（op+0x2C）。
    10. clientOp.ClientTickTms = TimeManager.curClientTms（op+0x30）。
    BATTLE_ACTION_OPERATE_TYPE proto oneof setter field number
    BattleOpAddBuff(1) AddBuffOpData 101
    BattleOpFinishBuff(2) FinishBuffOpData 102
    BattleOpSkillAttach(3) SkillAttachOpData 103
    BattleOpSkillDetach(4) SkillDetachOpData 104
    BattleOpSkillStartCast(5) SkillStartCastOpData 106
    BattleOpSkillEndCast(6) SkillEndCastOpData 107
    BattleOpSwitchMode(7) no setter in observed specialization none
    BattleOpTriggerAction(8) TriggerActionOpData 109
    BattleOpEntityDie(10) EntityDieOpData 110
    BattleOpEntityValueModify(11) EntityValueModifyData 111
    BattleOpResetBattleData(12) ResetBattleOpData 112
    BattleOpUpdateDataVersion(13) UpdateDataVersionOpData 113
    BattleOpEnablePassiveSkill(14) SkillEnableOpData 105
    BattleOpTriggerKeyword(15) TriggerKeywordOpData 114
    BattleOpShieldValueModify(16) ShieldValueModifyOpData 115
    BattleOpUpdateAtbInfo(17) UpdateAtbInfoOpData 116
    BattleOpModifyBattleState(18) ModifyBattleStateOpData 117
    BattleOpModifyPoiseValue(19) ModifyPoiseValueOpData 118
    BattleOpSetBuffEnableState(20) SetBuffEnableStateOpData 119
    1. 无论 oneof 是否被设置，正常路径最后调用 m_cachedOp.Add(clientOp)。

    Flush¶

    确认状态：已确认 native 主路径：

    1. tail call TryUpLoad(true)。

    _CheckCanUpload¶

    确认状态：已确认 native 主路径：

    1. m_cachedOp == null 进入 assert/trap；m_cachedOp.Count == 0 返回 false。
    2. 若 m_triggerMaxCountFrameLastFrame == true，返回 true。
    3. 否则读取 TimeManager.time 和 m_lastUploadTimestamp。
    4. 从 GameInstance battle data config 读取 upload interval 字段（observed offset 0x8C），加 epsilon 后与 elapsed time 比较；elapsed 超过 interval 时返回 true，否则 false。

    TryUpLoad¶

    确认状态：已确认 native 主路径：

    1. 若 force == false，先调用 _CheckCanUpload()；false 时直接 return。
    2. 清空 m_msgList。
    3. 从 NetWorkMsgPoolSend<CS_BATTLE_OP>.Allocate() 获取 message，加入 m_msgList。
    4. 清 m_triggerMaxCountFrameLastFrame。
    5. 遍历 m_cachedOp，把 op 添加到当前 CS_BATTLE_OP 的 repeated field。
    6. 上传分包受 battle data config 控制：
    7. observed +0x94：最大 op count。
    8. observed +0x90：message size/limit。
    9. 达到限制时通过分支设置 max-count/flush 状态并切换 batch。
    10. 遍历 m_msgList，只发送 repeated field 非空的 CS_BATTLE_OP，调用 BattleManager.SendMsg(IMessage, Action<int>)。
    11. 若没有触发 max-count frame flag，清空 m_cachedOp 和 m_msgList。
    12. 更新 m_lastUploadTimestamp = TimeManager.time。

    Release¶

    确认状态：已确认 native 主路径清空：

    • m_cachedOp
    • m_msgList

    AbilityAction.ServerActionParams.CreateServerOp¶

    确认状态：已确认

    字段 offset 类型
    ownerInfo 0x00 BattleManager.ServerReporter.OwnerInfo
    ownerId 0x18 ulong
    instId 0x20 ulong
    templateId 0x28 TEMPLATE_ID
    triggerType 0x30 BATTLE_TRIGGER_TYPE
    envType 0x34 BATTLE_ACTION_OWNER_TYPE
    actionInst 0x38 BATTLE_ACTION_INST
    serverRuntime 0x40 object

    native 主路径：

    1. 从当前全局 BattleManager 路径取 serverReporter。
    2. 若 manager/reporter 缺失，直接 return。
    3. 否则以 this 作为 actionParams 跳转到 ServerReporter.CreateServerActionOp。

    CreateServerActionOp¶

    确认状态：已确认 native 主路径：

    1. BattleProtoAllocator.Allocate<BATTLE_TRIGGER_ACTION_OP_DATA>()。
    2. 字段拷贝：
    3. op.OwnerId = actionParams.ownerId（+0x18 -> +0x18）。
    4. op.InstId = actionParams.instId（+0x20 -> +0x20）。
    5. op.TemplateId = actionParams.templateId（+0x28 -> +0x28）。
    6. op.Action = actionParams.actionInst（+0x38 -> +0x30）。
    7. op.TriggerType = actionParams.triggerType（+0x30 -> +0x38）。
    8. op.OwnerType = actionParams.envType（+0x34 -> +0x3C）。
    9. 调 CreateServerOp<BATTLE_TRIGGER_ACTION_OP_DATA>：
    10. opType = BattleOpTriggerAction(8)。
    11. opData = BATTLE_TRIGGER_ACTION_OP_DATA。

    12. ownerInfo = actionParams.ownerInfo。 确认差异点：

    13. ServerActionParams.serverRuntime 不在该方法中直接拷贝；runtime oneof 应已经挂在 actionInst 内。

    CreateModifyEntityValueOp¶

    确认状态：已确认 参数：

    • entity: Entity*，Windows x64 rdx，保存到 rdi。
    • hp: double，保存到 xmm6。
    • usp: float，保存到 xmm7。
    • type: ModifyValueType，栈参数，写入 proto Source。

    native 主路径：

    1. BattleProtoAllocator.Allocate<BATTLE_ENTITY_确认状态LUE_MODIFY_DATA>()。
    2. 写 op.Source = type（[rsp+0xB0] -> op+0x30）。
    3. 若 entity == null，走 assert/trap；正常路径继续。
    4. 写 entity id 字段：
    5. op.EntityInstId = entity.battleInstId（observed entity+0x40 -> op+0x18）。
    6. op.SceneNumId = entity.sceneNumId（observed entity+0x48 -> op+0x20）。
    7. BattleProtoAllocator.Allocate<BATTLE_INFO>()，写入 op.Value。
    8. 写 nested battle info：
    9. op.Value.Hp = hp（xmm6 -> value+0x18）。
    10. op.Value.Ultimatesp = usp（xmm7 -> value+0x20）。
    11. 构造栈上 OwnerInfo：
    12. ownerInfo.serverId = entity.serverId（observed entity+0x30）。
    13. ownerInfo.battleInstId = entity.battleInstId（observed entity+0x40）。
    14. ownerInfo.generation = entity.generation（observed entity+0x38）。
    15. 调 CreateServerOp<BATTLE_ENTITY_确认状态LUE_MODIFY_DATA>：
    16. opType = BattleOpEntityValueModify(11)。
    17. opData = BATTLE_ENTITY_确认状态LUE_MODIFY_DATA。
    18. ownerInfo = stack OwnerInfo。
    19. 返回分配的 BATTLE_ENTITY_确认状态LUE_MODIFY_DATA。

    CreateUpdateAtbInfoOp¶

    确认状态：已确认 参数：

    • updateType: BATTLE_UPDATE_ATB_INFO_REASON，保存到 r14d。
    • templateId: TEMPLATE_ID*，保存到 rbx，null 时分配空 TEMPLATE_ID。
    • originId: TEMPLATE_ID*，保存到 rsi，null 时分配空 TEMPLATE_ID。
    • defaultValue: double，栈参数，写入 DeltaValue。

    native 主路径：

    1. 静态 MethodInfo 初始化。
    2. 分配 BATTLE_UPDATE_ATB_INFO_OP_DATA。
    3. 分配 nested BATTLE_ATB_DATA，写入 op.AtbData。
    4. 从当前 BattleManager 读取 ATB 当前值，写 AtbData.AtbValue。
    5. 调 BattleManager.GetAtbRecover()，写 AtbData.AtbRecoverySpeed。
    6. 调 TimeManager.curClientTms，写 AtbData.ClientTickTs。
    7. 写 AtbData.Reason = updateType。
    8. 写 op.TemplateId = templateId ?: new TEMPLATE_ID()。
    9. 写 op.OriginalSourceTemplateId = originId ?: new TEMPLATE_ID()。
    10. 写 op.DeltaValue = defaultValue。
    11. 从 GameInstance.playerController 当前主控对象取 AbilitySystem，调用 BattleServerDataConverter.GetOwnerInfo 构造 ownerInfo。
    12. 从当前 BattleManager 取 serverReporter，调 CreateServerOp<BATTLE_UPDATE_ATB_INFO_OP_DATA>：
    13. opType = BattleOpUpdateAtbInfo(17)。
    14. opData = BATTLE_UPDATE_ATB_INFO_OP_DATA。

    15. ownerInfo = current main-control AbilitySystem ownerInfo。 确认差异点：

    16. ownerInfo 暂由 world.playerController.mainCharacter.serverId 派生，等待原 Entity identity 字段还原。

    RefreshModifyEntityValueOp¶

    确认状态：已确认 参数：

    • entity: Entity*，Windows x64 rdx，保存到 rbx。

    native 主路径：

    1. 静态 MethodInfo 初始化，准备 List<BATTLE_CLIENT_OP_DATA>.get_Item 等方法引用。
    2. 读取 this.m_cachedOp（offset 0x20）和 list _size。
    3. 遍历 cached op：
    4. 只处理 op.opType == 0x0B，即 BattleOpEntityValueModify(11)。
    5. 通过 BATTLE_CLIENT_OP_DATA.get_EntityValueModifyData() 取 typed payload。
    6. 只处理 payload.Source == 1，即 ModifyValueType.Merge。
    7. 比较 payload.EntityInstId == entity.battleInstId（observed entity+0x40）。
    8. 命中已有 merge op 时：
    9. 读取当前 HP，并写入 payload.Value.Hp（nested BATTLE_INFO offset +0x18）。
    10. 读取 entity.abilityCom.m_ultimateSp（observed ability component offset +0x16C），写入 payload.Value.Ultimatesp（nested offset +0x20）。
    11. 返回，不追加新 op。
    12. 未命中时：
    13. 读取当前 HP 和当前 USP。
    14. 调 CreateModifyEntityValueOp(entity, currentHp, currentUsp, ModifyValueType.Merge)。
    15. native 对 null m_cachedOp、payload、entity、ability component、nested value 均走 assert/trap；正常 gameplay 路径不接受这些对象缺失。
