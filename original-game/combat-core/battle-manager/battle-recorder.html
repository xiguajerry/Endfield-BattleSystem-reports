    BattleManager.BattleRecorder¶

    BattleRecorder 是 BattleManager 下的战斗统计与明细记录器，覆盖技能、伤害、治疗、韧性、护盾、ATB/USP、敌人、异常、Buff、GlobalBuff、投射物和 AbilityEntity 召唤等战斗事件。

    字段¶

    分组 字段
    report state m_report, m_isWaitToEndRecord, m_inRecording, m_recordingTime, m_curBattleInfo, m_detailRecordLevel, m_detailReport, m_maxDetailRecordWindow, m_lastRecordAtb
    constants abnormal/burst/control ids such as FIRE_ABNORMAL_ID, CRUSH_ID, KNOCKDOWN_ID, NO_GUARD_ID

    已展开的 tactical item summary 字段：

    类型 字段 offset 说明
    CharacterSummaryInfo itemUse 0x1E8 指向 ItemInfo。
    ItemInfo itemId 0x10 ItemInfo.Init(slotIndex) 从 BattleManager.GetTacticalItemId(slotIndex) 读取。
    ItemInfo usedCnt 0x18 ItemInfo.Init 清 0；RecordUseTacticalItem native 主路径自增。
    ItemInfo remainCnt 0x1C ItemInfo.Init(slotIndex) 从 BattleManager.GetTacticalItemRemainingUseCount(slotIndex) 读取。

    已展开的 skill summary/detail 字段：

    CharacterSummaryInfo lastAttackInfoAI 0xB8 AI/非主控普通攻击最后段。
    CharacterSummaryInfo powerAttackInfo 0xC0 SkillType.BreakingAttack 中的 power attack。
    CharacterSummaryInfo plungingAttackInfo 0xC8 下落攻击。
    CharacterSummaryInfo normalSkillInfo 0xD0 战技。
    CharacterSummaryInfo comboSkillInfo 0xD8 连携技。

    SkillInfo skillId 0x10 SkillInfo.Init(string) 初始化。
    SkillInfo totalDamage 0x18 伤害累计，后续 damage recorder slice 补齐。
    SkillInfo totalPoiseDamage 0x20 韧性伤害累计，后续 damage recorder slice 补齐。

    TowerInfo totalDamageOutput / totalTakenDamage 0x38 / 0x40 _RecordTowerOutputDamage / _RecordTowerTakeDamage 累加 modifier delta。

    生命周期¶

    • 随 BattleManager 创建。
    • battle start/end 控制 m_inRecording 与 detail report。
    • OnTailLateTick 在 BattleManager tail-late root 中执行；active recording 时累计 recording time，并在 end-record flag 置位时进入结束记录流程。

    已确认行为¶

    原游戏 recorder 不是单一日志列表，而是按 battle point、buff、global buff、damage、heal、skill、projectile、ability entity 等 detail/summary 分层记录。

    BattleRecorder.RecordUseTacticalItem¶

    确认状态：已确认 native 主路径：

    1. 从 GameInstance.player 进入当前 player/squad 数据，读取 current squad slot list。
    2. List<Squad.Slot>.get_Item(slotIndex)。
    3. 从 slot 读取角色/AbilitySystem 指针：slot +0x28 -> +0x128。
    4. 调用 _GetOrCreateCharacterSummaryInfo(AbilitySystem)，确认状态 已确认。
    5. 若 summary 和 summary.itemUse 非空，执行 inc dword ptr [itemUse + 0x18]，即 usedCnt++。

    注意：

    BattleRecorder.ItemInfo.Init¶

    确认状态：已确认

    1. and dword ptr [this + 0x18], 0，清 usedCnt。
    2. 再从 GameWorld.battleManager 调 BattleManager.GetTacticalItemRemainingUseCount(slotIndex)，写入 [this + 0x1C] remainCnt。

    BattleManager.UseTacticalItem recorder 参数来源¶

    确认状态：已确认 成功调用 InventorySystem.UseItem 后：

    • 战斗中先扣 remainingUseCount，必要时扣 remainingToMainCharCount，并写回 m_charEquippedItemDict。
    • 随后取 remainCnt = remainingUseCount。
    • 取 usedCnt = maxUseCount - remainingUseCount。
    • 调 BattleRecorder.RecordUseTacticalItem(slotIndex,itemId,usedCnt,remainCnt)。

    BattleRecorder.OnTailLateTick¶

    确认状态：已确认 调用入口：BattleManager.TailLateTick 状态 已确认。 已确认主路径：

    1. 若 m_inRecording 为 false，直接返回。
    2. 若 end-record flag 为 true，调用 _DoEndRecord。
    3. 若调用后仍处于 recording 状态，则 m_recordingTime += deltaTime。

    BattleRecorder.RecordCastSkill¶

    确认状态：已确认 已确认主路径：

    1. m_inRecording 为 false 时直接返回。
    2. 若 caster.entity.objectType == Character(8)，调用 _RecordCharacterCastSkill(caster, skill)。
    3. 若 m_detailRecordLevel > 0，调用 _RecordSkillCastStartDetail(skill)。

    BattleRecorder.RecordEndSkill¶

    确认状态：已确认 已确认主路径：

    1. m_inRecording 为 false 时直接返回。
    2. 若 m_detailRecordLevel > 0，调用 _RecordSkillCastEndDetail(skill, finishType, interruptReason, isCancelled)。
    3. native 不在该方法更新 CharacterSummaryInfo cast count。

    BattleRecorder.RecordGlobalBuffStart / RecordGlobalBuffFinish¶

    确认状态s：

    • RecordGlobalBuffStart：已确认
    • _RecordGlobalBuffStartDetail：已确认
    • RecordGlobalBuffFinish：已确认
    • _RecordGlobalBuffFinishDetail：已确认
    • GlobalBuffDetail.Init：已确认
    • GlobalBuff.OnStart：已确认
    • GlobalBuff.OnFinish：已确认

    已确认主路径：

    1. RecordGlobalBuffStart / RecordGlobalBuffFinish 都受 m_inRecording 与 m_detailRecordLevel > 0 gate 控制。
    2. start/finish detail 都写入同一个 GlobalBuffDetail list，recordType 分别为 GlobalBuffStart(0) / GlobalBuffFinish(1)。
    3. _RecordGlobalBuffStartDetail / _RecordGlobalBuffFinishDetail 都先 _TrimDetailsByTime(globalBuffDetails)，再分配 GlobalBuffDetail，调用 Init，写 _RecordDetailTime，最后加入 detail list。
    4. GlobalBuffDetail.Init 写 globalBuffId、recordType、sourceId 和 lifeTime。sourceId 是 source ability 的 entity id/serverId 拼接；source 缺失时 native 有空字符串 fallback。

    5. GlobalBuff.OnStart 在 _AddBuffs() 之后写 start detail。GlobalBuff.OnFinish 先执行 finish event action，再 _FinishBuffs()，清状态/listener，最后调用 RecordGlobalBuffFinish。

    6. BattleRecorder.globalBuffDetails

    7. recordGlobalBuffStart(GlobalBuffInstance)
    8. recordGlobalBuffFinish(GlobalBuffInstance)
    9. GlobalBuffDetail / GlobalBuffDetailRecordType
    10. GlobalBuffManager 在 first-start _AddBuffs 等价路径后记录 start，在 finish child-buff cleanup 后记录 finish。

    未实现/保留：

    • native detail window trim、pooled allocate/recycle 和上传 JSON 格式。

    12. World.createAbilityEntity / removeEntity 记录 ability entity create/release。

    _RecordSkillCastStartDetail / _RecordSkillCastEndDetail¶

    确认状态s：已确认 / 已确认 已确认主路径：

    _RecordCharacterCastSkill¶

    确认状态：已确认 已确认主路径：

    • 先通过 _GetOrCreateCharacterSummaryInfo(target) 获取角色 summary。
    • 非 Attack 分支：
    • BreakingAttack(1) -> powerAttackInfo.castTimes++。
    • NormalSkill(2) -> normalSkillInfo.castTimes++。
    • ComboSkill(6) -> comboSkillInfo.castTimes++。
    • UltimateSkill(7) -> ultimateSkillInfo.castTimes++。
    • AttachSkill(3) / Dodge(5) 未见 summary cast count 更新。
    • Attack 分支会读取角色 normal/plunging 相关 skill list 和 CharacterCtrl 主控状态：
    • 下落攻击进入 plungingAttackInfo。
    • 普通攻击非最后段进入 attackInfo，并按 main char / AI 分桶到 attackInfoMainChar 或 attackInfoAI。

    • 普通攻击最后段进入 lastAttackInfoMainChar 或 lastAttackInfoAI。

    • 下落攻击用 SkillSpecification.CharacterPlungingAttack 或角色 bundle 的 plunging start/end skill id 判定。

    • power attack 用 SkillSpecification.CharacterPowerAttack / SkillType.BreakingAttack 判定。
    • normal/combo/ultimate 按 SkillSpecification.toSkillType() 判定。
    • Attack 最后一段按 CharacterTemplateSkillBundleResolver.normalAttackList.last() 判定。

    BattleRecorder.RecordDamage¶

    确认状态：已确认 已确认主路径：

    1. m_inRecording 为 false 时直接返回。
    2. 调 _RecordCharacterOutputDamage(modifier)。
    3. 调 _RecordCharacterTakeDamage(modifier)。
    4. 调 _RecordTowerOutputDamage(modifier)。
    5. 调 _RecordTowerTakeDamage(modifier)。

    6. 若 m_detailRecordLevel > 0，调 _RecordDamageDetail(modifier)。

    7. DamageApplication.applyDamageResult 在 HP 结算后调用 BattleRecorder.recordDamage。

    8. source 若为 AbilityEntityController component，输出归属到其 sourceEntity，否则用 source 本体。
    9. 角色 source summary 累计 totalDamageOutput 和 totalDamageEffective，并按 SkillCastInfo.skillSpec 写对应 SkillInfo.totalDamage。
    10. 角色 target summary 累计 totalTakenDamage、totalTakenDamageTimes 和 remainingHp。
    11. independent-health ability entity source 写 TowerInfo.totalDamageOutput，并按 SkillCastInfo.skillId 写 damageOutputBySkillId。
    12. independent-health ability entity target 写 TowerInfo.totalTakenDamage 和 remainingHp。

    BattleRecorder.IsIndependentHealth / tower summary¶

    确认状态s：

    • IsIndependentHealth：已确认
    • _GetOrCreateTowerInfo：已确认
    • _RecordTowerOutputDamage：已确认
    • _RecordTowerTakeDamage：已确认
    • TowerInfo.Init：已确认

    已确认主路径：

    1. IsIndependentHealth(target) 返回 target.get_healthType() == HealthType.Independent(1)。
    2. _GetOrCreateTowerInfo(target) 先经过 independent-health/type guard。
    3. tower key 是 target.entity.templateData.id。
    4. 已存在同 key TowerInfo 时复用；否则分配并调用 TowerInfo.Init(target)。
    5. _RecordTowerOutputDamage 读取 modifier source；source independent-health 时写 totalDamageOutput += abs(modifier delta)，并按 modifier action/skill index 更新 per-index damage map。

    6. _RecordTowerTakeDamage 读取 modifier target；target independent-health 时写 totalTakenDamage += abs(modifier delta)，并读取当前 HP 写 remainingHp。

    7. 当前 AbilityEntityController component 表示 ability entity，不是原游戏 AbilitySystem。本地 isIndependentHealth 仅把 AbilityEntityController component.isIndependentHealth 映射到 native AbilitySystem.healthType == Independent。

    8. TowerInfo.towerId 使用 ability entity id，对应 native entity/template id key。
    9. output damage 使用 DamageResult.finalDamage，taken damage 使用 HP delta；这是当前 deterministic modifier delta 等价值。

    BattleRecorder.RecordConsumeAtb¶

    确认状态：已确认

    1. m_inRecording 为 false 时返回。
    2. _GetOrCreateCharacterSummaryInfo(source)。
    3. summary.consumedAtb += value，字段 offset 0x100。
    4. 遍历 m_curBattleInfo.characterSummaryInfos。
    5. 对每个 summary 写 summary.naturalGainedAtb += currentAtb + value - m_lastRecordAtb，字段 offset 0x118。

    BattleRecorder.RecordGainAtb¶

    确认状态s：

    • RecordGainAtb：已确认
    • _RecordAtbDetail：已确认
    • BattlePointDetail.Init(double,double,string,string,SkillCastInfo)：已确认

    已确认主路径：

    1. 检查 m_inRecording；未 recording 直接返回。
    2. _GetOrCreateCharacterSummaryInfo(source)，source 非角色时不更新角色 summary。
    3. source summary 写 gainedAtb += value。
    4. 遍历所有 character summary，用当前 ATB、传入 gain value 和 m_lastRecordAtb 结算自然增长：currentAtb - value - m_lastRecordAtb。
    5. 更新 m_lastRecordAtb = currentAtb。
    6. 依据 environmentId 或 originSkill.m_skillId 子串写入来源分桶：_power_attack、_plunging_attack、_attack、_normal_skill、_combo_skill、_ultimate_skill、wpn_passive_、passive_equipsuit_、talent、精确 buff_common_dash_behit_listener，否则归入 other。
    7. m_detailRecordLevel > 0 时进入 _RecordAtbDetail。

    8. _RecordAtbDetail 裁剪 battlePointDetails，分配 BattlePointDetail，读取当前 ATB 为 afterEventValue，调用 Init(valueDelta, afterEventValue, sourceId, environmentId, skillCastInfo)，写 pointType=Atb(0)，写 detail time 后加入 list。

    9. BattleManager.gainAtb 作为 owner entry，先通过 AtbManager.gainCost 应用 clamp，再用实际 post-clamp delta 调 BattleRecorder.recordGainAtb。
