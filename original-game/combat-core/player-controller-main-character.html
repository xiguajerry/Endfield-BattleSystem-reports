    Beyond.Gameplay.Core.PlayerController mainCharacter 维护¶

    类定位¶

    • 控制链路总览: PlayerController 控制链路

    读取入口¶

    GetMainCharacter 只是读取全局 PlayerController 上的字段:

    CheckMainCharacterCondition 也是直接比较目标实体和这个字段。这里没有额外缓存层。

    写入口¶

    flowchart TD
    N0["SquadManager._SetMainCharacter(newLeader, reason)"]
    N1["GameInstance.get_playerController()"]
    N2["PlayerController.SetMainCharacter(newLeader, reason)"]
    N0 --> N1
    N1 --> N2

    所以 PlayerController 不是“决定谁是主控”的模块。它负责接收已经确定的 newMainChar，写入字段并刷新本地控制、表现和 ability 状态。

    副作用范围¶

    SetMainCharacter 不是单纯赋值。字段写入后会同步刷新:

    • 新主控的 CharacterController.OnMainCharacterChange
    • 主控 tag 迁移 _TransferTagsToNewMainChar
    • PlayerCenterUpdater 跟随目标
    • 非 CastSkill 切换下的走/跑状态
    • 主控和队员的 animator/ability LOD
    • 所有 squad member 的 AbilitySystem.OnMainCharacterChange
    • 移动动画速度、空中禁止检查、小半径 swap 处理

    因此 CheckMainCharacterCondition 看到的 mainCharacter 是运行时已完成本地主控刷新后的实体引用。

    与 SquadManager 的边界¶

    PlayerController.SetMainCharacter 不负责:

    • 查找 squad slot
    • 更新 leaderIndex
    • 更新 leaderId
    • 保存 leader 到服务端或客户端 squad 状态
    • 选择死亡/释放后的 fallback leader
    • 发起 SquadManager 级别的 formation、battle、navigation、event log 全局刷新

    这些都在 SquadManager / Squad 主控维护 中处理。 PlayerController 的职责可以概括为:

    flowchart TD
    N0["接收新主控实体"]
    N1["写 mainCharacter 字段"]
    N2["刷新本地控制/表现/ability 状态"]
    N0 --> N1
    N1 --> N2

    后台角色技能不会写 mainCharacter¶

    2026-06-01 复核 PlayerController.CastSkill(...)、CharacterSkillSystem 与 BattleManager.CastPendingComboSkill(...) 后确认：游戏内并不是所有按 slot 释放的技能都会切换主控。 当 CastSkill(source, skillId, target, ...) 的 source 是当前 mainCharacter 的 AbilitySystem 时，函数会创建 CmdType.Skill 的 SkillCommand，必要时立即 PlayerCommandController.Execute(CmdType.Skill)，从而进入主控 CharacterMainCtrlSystem 的 CenterSkillState。 当 source 不是当前主控时，函数不会调用 SquadManager.SetLeaderByClient，也不会写 PlayerController.mainCharacter。它会把技能输入写到后台角色自己的 AbilitySystem next-skill request，再交给 CharacterSkillSystem.CastComboSkill(entity, inputView)。 CharacterSkillSystem 会让该实体切到 CharacterAIModeType.CastSkill，随后由 CharacterCastSkillBehavior._TryCastSkill() 调用该实体自己的 AbilitySystem.TryCastSkill(...)。 连携技 UI 也走后台角色链路：ComboSkillPanel.CastComboSkill(charIndex) 调 BattleManager.CastPendingComboSkill(charIndex)，后者按 squad slot 找到对应角色的 AbilitySystem 并调用 _TryCastComboSkill(...) / AbilitySystem.TryCastComboSkill(...)。 这条链路同样不切主控。 因此主控字段只应在明确切 leader 的链路中更新；后台角色释放普通战技、连携技或终结技时， mainCharacter 应保持不变。
