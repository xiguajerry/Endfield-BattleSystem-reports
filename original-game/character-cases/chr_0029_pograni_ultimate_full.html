    chr_0029 骏卫 (Pograni) 终结技完整流程¶

    时间轴总览¶

    timeline
    title chr_0029_pograni_ultimate_skill
    0f : CG / 时停 / 运镜 / 无敌 / 霸体
    74f : 生成初始 AbilityEntity x4
    76f : Rush 伤害 400% ATK
        : 15 失衡
        : 击飞与物理异常触发入口
    82f : 主 skill 直接伤害窗口结束
    后续 : 敌人 aura 监听受击事件
         : count 驱动 attack2 追击与 finish4 终结
    阶段 关键行为
    初始 AE 3-17f 抢锁，首个成功者给骏卫创建主 buff
    主 buff DuringBuffEnable 给敌人挂 aura buff
    count buff OnBuffStart 创建 5 层 count buff
    aura 监听 监听 OnBeforeTakePhysicalInfliction / OnBeforeTakeDamage
    追击 count 从 5 递减到 1，依次生成 4 次 attack2
    终结 count 到 1 后创建 final_rush，再生成 finish4 x4
    结束 finish4 造成终结伤害并结束主 buff

    结论¶

    骏卫大招不是一个单一 timeline 技能。它的完整伤害链由 4 层系统组成：

    flowchart TD
    Cast["主 skill"]
    Spawn["生成初始 AbilityEntity x4"]
    Race{"任一 AE 抢锁成功"}
    RaceFail["其余 AE 检查失败<br/>跳过 CreateBuff"]
    Buff["创建主 buff"]
    Count["创建 count buff<br/>初始 5 层"]
    Aura["敌人获得 aura buff<br/>监听受击事件"]
    Hit{"触发受击事件"}
    CountCheck{"count 层数"}
    Attack2["生成 attack2<br/>先 spawn 后扣 1 层"]
    FinalRush["创建 final_rush<br/>扣最后 1 层"]
    Finish4["生成 finish4 x4<br/>终结伤害并结束主 buff"]
    NoAction["不生成追击"]

    Cast --> Spawn --> Race
    Race -->|成功| Buff --> Count --> Aura
    Race -->|失败| RaceFail
    Aura --> Hit
    Hit --> CountCheck
    CountCheck -->|count > 1| Attack2 --> Aura
    CountCheck -->|count == 1| FinalRush --> Finish4
    CountCheck -->|count == 0| NoAction

    如果只执行 chr_0029_pograni_ultimate_skill 的 timeline，只能覆盖 CG、初始召唤和 Rush 段，无法自然得到完整 attack2 x4 + finish4。完整模拟必须实现 SpawnAbilityEntity、AbilityEntity skill tick、AuraAction、buff timeline、受击事件回调和 count 驱动。

    主大招 chr_0029_pograni_ultimate_skill：

    字段 值
    消耗 100 USP
    持续 210f
    Rush 倍率 atk_scale_rush = 4.0
    追击倍率 atk_scale_trigger = 2.0
    终结倍率 atk_scale_final = 4.0

    CG / 防护阶段¶

    帧 行为
    0-75f UltimateShowAction、UltimateTimeAction、HideUIAction、运镜
    0-90f buff_common_damage_immune_medium，提供无敌
    0-90f SetSuperArmorAction，提供霸体/抗打断
    0-3f 清理上一次残留的 soldier AE 和主 buff

    初始 SpawnAbilityEntity¶

    74-77f 主 skill 连续生成 4 个 AbilityEntity。 共用字段：

    字段 值
    abilityEntityId abilityentity_chr_0029_pograni_ultimate_skill
    abilityEntitySkillId chr_0029_pograni_ultimate_skill_abilityentity
    setAbilityEntitySource true
    abilityEntitySource ActionOwner，即骏卫
    setAbilityEntityTarget true
    abilityEntityTarget 当前目标
    assignBlackboard true
    saveToContext true
    inheritSourceSkillCastId true

    4 个出生点：

    contextKey bornPosOffset
    ae1 (-5.06, 0, 0.57)
    ae2 (-5.13, 0, 8.85)
    ae3 (4.42, 0, 9.99)
    ae4 (5.33, 0, 1.84)

    assignBlackboard=true 很关键。主 skill 的 atk_scale_trigger、atk_scale_final、atb_trigger、atb_final、poise_final、duration 会传入 AE skill，后续再写入主 buff。

    AbilityEntityTable 模板¶

    abilityentity_chr_0029_pograni_ultimate_skill 是一个带技能系统的 AbilityEntity，而不是普通 projectile。 关键模板字段：

    字段 值/含义
    id abilityentity_chr_0029_pograni_ultimate_skill
    duration 50.0
    bornTag Immune/Damage、不可选中、Skill/Character/chr_0029_pgrani/Soldier
    hasSkill true
    detectedRadius 0.2
    detectedHeight 0.4

    技能列表：

    active skill
    chr_0029_pograni_ultimate_skill_abilityentity
    chr_0029_pograni_ultimate_skill_abilityentity_attack2
    chr_0029_pograni_ultimate_skill_abilityentity_finish1
    chr_0029_pograni_ultimate_skill_abilityentity_finish2
    chr_0029_pograni_ultimate_skill_abilityentity_finish3
    chr_0029_pograni_ultimate_skill_abilityentity_finish4

    后续所有 attack2 / finish4 仍然使用同一个 AbilityEntity 模板，只是 abilityEntitySkillId 不同。

    初始 AbilityEntity skill¶

    初始 skill：chr_0029_pograni_ultimate_skill_abilityentity。

    字段 值
    skillSpecification CharacterUltimateSkill
    durationFrame 300f

    关键 timeline：

    帧 行为
    1-7f MoveToAction，士兵冲向目标
    3-17f CheckBuffStackNum 检查骏卫身上主 buff 层数是否等于 0
    3-17f 成功者创建 buff_chr_0029_pograni_ultimate_skill
    50-53f FinishOwnerAction，初始 AE 自毁

    抢锁逻辑¶

    4 个初始 AE 都会执行同一段 3-17f 逻辑，但只有第一个看到 buff_chr_0029_pograni_ultimate_skill == 0 的 AE 会成功创建主 buff。 这等价于竞争锁：

    flowchart TD
    Start["AE1/AE2/AE3/AE4 同时进入 3-17f 检查窗口"]
    Loop["取一个 AE 执行 CheckBuffStackNum"]
    HasMain{"骏卫已有主 buff？"}
    Create["创建主 buff"]
    Skip["跳过 CreateBuff"]
    More{"还有 AE 未检查？"}
    End["检查窗口结束"]

    Start --> Loop --> HasMain
    HasMain -- "否" --> Create --> More
    HasMain -- "是" --> Skip --> More
    More -- "是" --> Loop
    More -- "否" --> End

    主 buff 黑板写入¶

    初始 AE 创建主 buff 时，从自身黑板写入：

    主 buff key 来源 key 值
    duration duration 30.0
    atk_scale_trigger atk_scale_trigger 2.0
    atk_scale_final atk_scale_final 4.0
    atb_trigger atb_trigger 15.0
    atb_final atb_final 50.0
    poise_final poise_final 15.0

    主 buff：连段系统的根¶

    主 buff：buff_chr_0029_pograni_ultimate_skill。 主 buff 有两个核心事件。

    DuringBuffEnable：给敌人挂 aura¶

    DuringBuffEnable 执行 AuraAction：

    字段 值
    auraDebugName pograni_ultimate
    auraType GlobalAura
    auraRoot Owner，即骏卫
    范围 球形半径 50
    目标 EnemyAll，反阵营，存活目标
    输入 buff buff_chr_0029_pograni_ultimate_skill_abilityentity_inaura
    buffSource ActionOwner

    这个 aura 会把 buff_chr_0029_pograni_ultimate_skill_abilityentity_inaura 挂到敌人身上。 注意：后续追击调度器在敌人身上，不在骏卫身上。这是完整链路里最容易漏掉的一环。

    OnBuffStart：创建 count buff¶

    OnBuffStart 创建： buff_chr_0029_pograni_ultimate_skill_count

    字段 值
    目标 Source，即骏卫
    asChildBuff true
    count 从主 buff 黑板 count 读取，默认主 buff 为 5.0
    写入黑板 duration

    因此完整连段的起始 count 是 5。

    OnBuffFinish：清理效果层¶

    OnBuffFinish 会通过 FinishBuffAdvanced 清理： buff_chr_0029_pograni_ultimate_skill_effect_layer

    敌人 aura buff：真正的调度器¶

    敌人 aura buff：buff_chr_0029_pograni_ultimate_skill_abilityentity_inaura。 这个 buff 自身没有 buffEventAction，而是通过 abilityEventAction 监听事件。 监听事件：

    abilityEvent 作用
    OnBeforeTakePhysicalInfliction 物理异常触发前，例如 Rush 的击飞/物理异常入口
    OnBeforeTakeDamage 受伤前，额外检查指定 damageDecorateMask

    两套事件逻辑基本一致，都是：检查防重复 marker、检查 count、根据 count 生成下一段 AE 或进入终结。

    防重复 timed marker¶

    关键 marker： chr_0029_pograni_soldier_attacked 每次触发前先执行 CheckTimedMarkerCondition。成功触发后执行 CreateTimedMarker，防止同一次受击/异常把 aura 逻辑重复跑多次。

    count > 1：生成 attack2¶

    当骏卫身上的 buff_chr_0029_pograni_ultimate_skill_count > 1：

    顺序 action
    1 CheckTimedMarkerCondition
    2 CheckBuffStackNumAdvanced(count > 1)
    3 SpawnAbilityEntity，skill = chr_0029_pograni_ultimate_skill_abilityentity_attack2
    4 FinishBuffAdvanced，移除 1 层 count
    5 CreateTimedMarker

    注意顺序是先 spawn，后扣 count。因此 attack2 启动时读到的是扣层前的 count。

    count == 1：进入 final rush¶

    当骏卫身上的 count 等于 1：

    顺序 action
    1 CheckTimedMarkerCondition
    2 CheckBuffStackNumAdvanced(count == 1)
    3 FinishBuffAdvanced，移除最后 1 层 count
    4 CreateBuff，创建 buff_chr_0029_pograni_ultimate_skill_finall_rush
    5 CreateTimedMarker

    count 变化：

    flowchart TD
    N0["5"]
    N1["spawn attack2 phase#1"]
    N2["4"]
    N3["spawn attack2 phase#2"]
    N4["3"]
    N5["spawn attack2 phase#3"]
    N6["2"]
    N7["spawn attack2 phase#4"]
    N8["1"]
    N9["create final_rush"]
    N10["0"]
    N0 --> N1
    N1 --> N2
    N2 --> N3
    N3 --> N4
    N4 --> N5
    N5 --> N6
    N6 --> N7
    N7 --> N8
    N8 --> N9
    N9 --> N10

    attack2 AbilityEntity skill¶

    追击 skill：chr_0029_pograni_ultimate_skill_abilityentity_attack2。

    字段 值
    skillSpecification CharacterUltimateSkill
    durationFrame 400f

    phase 选择¶

    skill 开头有 IfElse + CheckBuffStackNumAdvanced + JumpToAction，根据当前 count 跳到不同 phase。 因为 aura 是先 spawn 后扣 count，所以这里看到的是扣层前 count。

    attack2 启动时 count phase 起始帧
    5 第 1 段 0f
    4 第 2 段 100f
    3 第 3 段 200f
    2 第 4 段 300f

    每段攻击¶

    phase 攻击帧 行为
    1 9-13f FindTarget、ObtainCost、Damage、受击动画、HitStop
    2 108-112f 同上
    3 209-213f 同上
    4 307-311f 同上

    伤害/资源：

    参数 值
    atk_scale_trigger 2.0
    atb_trigger 15.0
    单段倍率 200% ATK
    四段合计 800% ATK
    四段 ATB 60

    每段 attack2 后还有对应的 FinishOwnerAction 和 InterruptCurSkillAction，用于结束当前 AE 和收束该段技能。

    final_rush buff¶

    当 aura 判断 count 为 1 时，它不会直接 spawn finish4，而是创建： buff_chr_0029_pograni_ultimate_skill_finall_rush 这个 buff 使用 timelineActions 调度终结 AE。 0-3f spawn 4 个 finish4 AE：

    bornPosOffset skill
    (2, 0, -2) chr_0029_pograni_ultimate_skill_abilityentity_finish4
    (-2, 0, -2) chr_0029_pograni_ultimate_skill_abilityentity_finish4
    (-2, 0, 2) chr_0029_pograni_ultimate_skill_abilityentity_finish4
    (2, 0, 2) chr_0029_pograni_ultimate_skill_abilityentity_finish4

    传入黑板：

    key 值
    atk_scale_final 4.0
    atb_final 50.0
    poise_final 15.0

    finish4 AbilityEntity skill¶

    终结 skill：chr_0029_pograni_ultimate_skill_abilityentity_finish4。

    字段 值
    skillSpecification CharacterUltimateSkill
    durationFrame 81f

    关键 timeline：

    帧 行为
    0-25f 多段 SelfRotate
    25-29f FindTarget("tar")
    25-52f IfElse(CheckTimedMarkerCondition)，防止多个 finish4 重复造成完整终结
    25-29f 成功分支执行 ObtainCost(atb_final)
    25-29f 成功分支执行 Damage(atk_scale_final) 和 Poise(poise_final)
    25-29f TakeDownAction
    31-38f FinishBuffAdvanced，结束 buff_chr_0029_pograni_ultimate_skill
    75-78f FinishOwnerAction，终结 AE 自毁

    终结效果：

    项 值
    伤害 400% ATK
    失衡 15
    ATB 50
    控制 击倒

    总倍率与收益¶

    完整触发后的理论总计：

    阶段 倍率 失衡 ATB
    Rush 400% 15 0
    attack2 x4 800% 0 60
    finish4 400% 15 50
    总计 1600% 30 110
    能力 对骏卫大招的影响
    SpawnAbilityEntity 完整 schema 必须解析 abilityEntityId、abilityEntitySkillId、source、target、assignBlackboard、context
    AbilityEntity runtime 需要创建可 tick 的 AE 实体，能持有 source/target/owner 和 skill
    AbilityEntityTable 根据 abilityEntityId 初始化模板 tag、duration、active skill bundle
    AuraAction 主 buff 必须能给敌人挂 abilityentity_inaura
    abilityEventAction 敌人 aura 必须监听 OnBeforeTakePhysicalInfliction 和 OnBeforeTakeDamage
    CheckBuffStackNumAdvanced aura 和 attack2 都依赖 count 判断
    JumpToAction attack2 必须按 count 跳到 0/100/200/300 phase
    CreateTimedMarker / CheckTimedMarkerCondition 防止同一触发重复 spawn attack2 或重复 final damage
    BuffData.timelineActions final_rush 必须能在 buff timeline 内 spawn finish4
    FinishOwnerAction AE 生命周期需要正确结束
    TakeDownAction 终结控制效果
