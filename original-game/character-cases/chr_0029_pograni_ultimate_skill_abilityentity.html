    chr_0029 骏卫 (Pograni) 终结技 — AbilityEntity¶

    基础数据¶

    字段 值
    skillId chr_0029_pograni_ultimate_skill_abilityentity
    castType Active
    skillSpecification CharacterUltimateSkill
    passiveSkillType AddBuff
    attackRangeType Melee
    selectStrategy NoTarget (无需目标选择，生成时已指定 target)
    durationFrame 300 (约 5.0s)
    exclusiveFrame 300 (不可被衔接)
    cooldownTime 0.0
    cost (UltimateSp) 0.0
    canMove false (移动由 MoveToAction 驱动)

    生成配置 (SpawnAbilityEntity)¶

    主 skill 在 frame 74-77 以相同参数生成 4 个实体，仅出生位移和旋转不同:

    参数 值 说明
    abilityEntityId abilityentity_chr_0029_pograni_ultimate_skill
    abilityEntitySkillId chr_0029_pograni_ultimate_skill_abilityentity
    assignBlackboard true 主 skill 黑板值复制到实体
    assignEntityBlackboard false 不额外分配 EntityBB
    assignPairs []
    setAbilityEntitySource true, ActionOwner 实体 source = 骏卫
    setAbilityEntityTarget true, Target 实体 target = 当前目标
    inheritSourceSkillCastId true 继承施放 ID
    dieOnEnd false 不自毁（手动控制）
    bornRotation SameWithSource

    四个实体的出生偏移¶

    实体 Context Key bornPosOffset (x, y, z) Y 旋转 (四元数) 大致方向
    ae1 ae1 (-5.06, 0, 0.57) y=0.446 左前 ~53°
    ae2 ae2 (-5.13, 0, 8.85) y=0.783 左远 ~103°
    ae3 ae3 (4.42, 0, 9.99) y=-0.895 右远 ~-127°
    ae4 ae4 (4.33, 0, 1.84) y=-0.616 右前 ~-76°

    四个实体在主角周围扇形展开，包围目标敌人。随后每个实体施加 SetIgnoreGlobalTimeScale(true) 确保不受全局时停影响。

    Blackboard¶

    实体自身的默认黑板（会被主 skill 的 assignBlackboard: true 覆盖）:

    Key 默认值 主 skill 覆盖值 实际使用
    atk_scale_rush 1.0 4.0 未使用（仅声明）
    atk_scale_trigger 0.0 2.0 → buff 传递
    atk_scale_final 0.0 4.0 → buff 传递
    atb_trigger 4.0 15.0 → buff 传递
    atb_final 0.0 50.0 → buff 传递
    poise_final 0.0 15.0 → buff 传递
    duration 20.0 30.0 buff 持续时间
    radius 5.0 5.0 未使用（仅声明）

    关键: 实体自身黑板的值大多是占位符，实际值通过 assignBlackboard: true 从主 skill 复制。最终写入 buff 的数值来自主 skill 的黑板配置。

    Action 详解¶

    Frame 0-3: SetAnimatorParam tRush¶

    flowchart TD
    N0["paramName: 'tRush'"]
    N1["paramAction: Trigger"]
    N2["true (触发冲刺动画)"]
    N3["endAction: Bool"]
    N4["false (结束时重置)"]
    N1 --> N2
    N3 --> N4

    Frame 1-7: MoveToAction — SoldierRush 冲刺¶

    参数 值 说明
    directionType OwnerForward 向前冲刺
    target Target (目标敌人)
    speed 36.0 固定速度
    totalTime 0.2s (12f) 移动时长
    rootMotionAnimKey SoldierRush RootMotion 动画驱动
    blockRadius 1.8 碰撞阻挡半径
    ignoreNavmesh true 忽略导航网格
    stopByCliff false 不掉崖

    实体以 36.0 的速度冲刺 0.2 秒（实际约 12 帧），由 SoldierRush root motion 动画控制具体位移曲线。

    Frame 0-1: CameraImpulseAction¶

    参数 值
    amplitudeGain 0.3
    frequencyGain 3.0
    impulseDuration 0.2s
    sustainTime 0.1s
    decayTime 0.5s
    rawSignalPath ...NoiseSettings_Level_3D_LargeY.asset

    实体生成时播放 3D 相机震动。

    Frame 3-17: CheckBuffStackNum + CreateBuff¶

    条件检查:

    CheckBuffStackNum:
 buffId: buff_chr_0029_pograni_ultimate_skill
 compareType: Equals
 value: 0.0
 checkTarget: Source (骏卫)

    如果 buff 层数为 0（即尚未创建），则:

    CreateBuff:
 buffId: buff_chr_0029_pograni_ultimate_skill
 count: 1
 buffSource: ActionSource (骏卫)
 target: Source (骏卫)
 inheritSourceSkillCastInfo: true
    Buff 参数 来源 Blackboard Key 主 skill 提供值
    duration duration 30.0 (秒)
    atk_scale_trigger atk_scale_trigger 2.0
    atk_scale_final atk_scale_final 4.0
    atb_trigger atb_trigger 15.0
    atb_final atb_final 50.0
    poise_final poise_final 15.0

    防重复机制: CheckBuffStackNum(Equals, 0) 确保 4 个实体中只有最先到达 frame 3 的那个能成功创建 buff，其余 3 个检查失败后跳过 CreateBuff。这避免了 buff 被重复叠加。

    Frame 50-53: FinishOwnerAction — 自毁¶

    参数 值
    owner Owner (实体自身)
    skipDieDisplay true，无死亡动画，直接消失

    实体生命周期约 50 帧（0.83 秒），在冲刺到达目标后自毁。

    Frame 0-60: VFX — 护盾特效¶

    P_fxbat_pograni_skill_ult_shield_start_01 (frame 0-60)
P_fxbat_pograni_skill_ult_shield_dissolve_asset_01 (frame 20-50)

    两个 VFX 叠加：护盾展开 + 护盾溶解，营造实体"出现→冲锋→消散"的视觉效果。

    核心机制总结¶

    初始 AbilityEntity 的出生点由主 skill 在 74-77f 创建：

    实体 出生偏移
    ae1 (-5.06, 0, 0.57)
    ae2 (-5.13, 0, 8.85)
    ae3 (4.42, 0, 9.99)
    ae4 (5.33, 0, 1.84) 
    flowchart TD
    A["主 skill 生成 4 个初始 AE"]
    B["0f 触发 tRush 动画"]
    C["1-7f SoldierRush 冲刺"]
    D["3-17f 竞争创建主 buff"]
    E["首个成功者给骏卫施加主 buff"]
    F["成功 AE 在 50f FinishOwner 自毁"]
    G["其余 AE 检查失败<br/>跳过 CreateBuff"]
    H["失败 AE 在 50f FinishOwner 自毁"]
    A --> B --> C --> D
    D -->|主 buff 不存在| E --> F
    D -->|主 buff 已存在| G --> H

    主 buff 写入以下关键参数：

    Key Value
    duration 30.0s
    atk_scale_trigger 2.0
    atk_scale_final 4.0
    atb_trigger 15.0
    atb_final 50.0
    poise_final 15.0

    这个 buff 后续被 attack2 和 finish4 的 CheckBuffStackNum 条件检查使用，控制追击和终结阶段的跳转逻辑。

    与主 skill、attack2、finish 的关系¶

    flowchart TD
    N0["主 skill (210f)"]
    N1["76f 突进伤害<br/>400% + 15 poise"]
    N2["74f 生成 4 个 AbilityEntity<br/>主 buff 施加到骏卫"]
    N3{"buff 存在期间受击"}
    N4["attack2 追击<br/>每段 200% + 15 ATB"]
    N5["finish4 终结<br/>400% + 15 poise + 50 ATB"]
    N0 --> N1
    N0 --> N2
    N2 --> N3
    N3 -->|count > 1| N4 --> N3
    N3 -->|count == 1| N5

    实体本身不造成伤害，它的唯一战斗作用是通过 CreateBuff 将主 skill 黑板中的攻击系数和 ATB 参数传递给 buff，供后续 attack2/finish 阶段读取。
