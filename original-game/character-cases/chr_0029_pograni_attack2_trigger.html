    chr_0029 骏卫 (Pograni) 追击 (attack2) 触发条件分析¶

    核心机制：buff 计数器驱动的多段跳转¶

    attack2 是一个 400 帧的 skill，内含 4 个结构相同的攻击 phase（帧 0/100/200/300）。但它并非一次跑完 400 帧——每段攻击后实体会自毁，然后外部重新触发下一段。通过 buff_chr_0029_pograni_ultimate_skill_count 的层数记住当前进度，用 IfElse + JumpTo 跳转到对应 phase。

    buff_chr_0029_pograni_ultimate_skill_count 数据¶

    字段 值
    lifeType Limited
    duration bbKey: duration (从主 buff 继承 → 主 skill duration=30.0)
    stackingType Stack, maxStackCnt=99
    blackboard count=4.0 (dynamic), duration=30.0
    abilityEventAction (空)
    buffEventAction (空)

    关键： count buff 本身没有任何事件 action，它是一个纯计数器。层数由外部修改。

    IfElse 跳转树¶

    attack2 在 frame 0-3 执行以下嵌套 IfElse：

    flowchart TD
    Start["frame 0-3<br/>检查 count buff 层数"]
    C5{"count == 5"}
    End["空 succeed 分支<br/>不执行 JumpTo"]
    C4{"count == 4"}
    J100["JumpTo 100f<br/>第 2 段"]
    C3{"count == 3"}
    J200["JumpTo 200f<br/>第 3 段"]
    C2{"count == 2"}
    J300["JumpTo 300f<br/>第 4 段"]
    J0["不跳转<br/>从 0f phase 开始"]

    Start --> C5
    C5 -->|是| End
    C5 -->|否| C4
    C4 -->|是| J100
    C4 -->|否| C3
    C3 -->|是| J200
    C3 -->|否| C2
    C2 -->|是| J300
    C2 -->|否| J0
    count 层数 动作 执行 phase
    5 命中空 succeed 分支，不执行 JumpToAction frame 0 phase
    4 JumpToAction(destFrame=100) frame 100 phase
    3 JumpToAction(destFrame=200) frame 200 phase
    2 JumpToAction(destFrame=300) frame 300 phase
    < 2 不跳转 frame 0 phase

    注: atk_scale_trigger 和 atb_trigger 的实际值来自主 buff 的黑板（被 abilityentity 的 assignBlackboard 从主 skill 覆盖），而非 attack2 自身黑板的默认值。

    结论¶

    追击的触发条件分为两层：

    1. 前置条件：buff_chr_0029_pograni_ultimate_skill 存在（由 abilityentity 施加，duration=30s），其 OnBuffStart 事件创建 count buff
    2. 阶段控制：buff_chr_0029_pograni_ultimate_skill_count 的堆叠层数决定 attack2 从哪个 phase 开始执行，每段攻击后层数递减
    3. 外部触发：attack2 的重复施放由游戏引擎/combo 系统驱动，非 JSON 数据层面可见
