    agtrinit¶

    本页合并 eny_0045_agtrinit 相关玩家版说明。结构固定为索敌机制与技能派生机制两部分；如果某部分暂无玩家版源文档，会在对应位置标出。

    索敌机制¶

    来源：technical-report/enemy-ai/aiconf_eny_0045_agtrinit_targeting.player.zh.md
    原标题：agtrinit 玩家版索敌说明

    日期：2026-06-28

    这篇把 aiconf_eny_0045_agtrinit 的索敌报告改写成普通玩家能理解的版本。它只描述基础配置，不包含 agtrinit_race 或其它近战/远程变体。

    一句话结论¶

    agtrinit 在正常战斗里非常明显地偏向“盯着当前主控角色打”。

    它并不是每次出招都重新从全队里公平挑一个目标。当前数据里可以看到两套目标：一套是敌人大脑里记住的目标，另一套是战斗树每轮直接取到的“当前主控角色”。很多主要攻击使用的是后者，所以玩家体感上它会一直追着主控角色施压。

    它怎么进入战斗¶

    这只怪的基础配置更像是“被关卡或脚本强制拉进战斗”的敌人，而不是普通巡逻怪那种“看见玩家、警觉值涨满、再入战”的流程。

    当前已确认的入战/离战逻辑可以这样理解：

    情况 玩家能理解的结果
    关卡给它强制战斗信号 它把主控角色记为目标，并进入战斗。
    已经有目标 它也可以直接进入战斗。
    关卡给它强制结束战斗信号 它清空目标，并离开战斗。
    没有结束战斗信号 它会继续保持战斗。

    目前没有确认到基础配置里有普通敌人那种完整的“被打了就警觉、警觉满再入战”的常规入口。也就是说，如果只看这份基础配置，不能把它当成普通野怪来理解。

    战斗中到底会打谁¶

    进入战斗后，它每轮都会记录两类目标：

    目标类型 玩家版解释
    敌人大脑目标 通常由强制入战或阵型系统写入，部分移动、回位、召唤、少量技能会用到。
    当前主控角色 每轮都会重新取一次，许多主要攻击直接用它。

    这就是它“像是在锁主控”的根本原因：即使敌人大脑目标将来被某些机制短暂改成别的角色，战斗树里仍然有大量攻击会重新读取当前主控角色，然后继续朝主控出招。

    所以要区分两件事：

    事情 是否等于完全转火
    改掉敌人大脑目标 不一定。
    让所有技能都打非主控角色 当前没有这样的确认路径。
    切换当前主控角色 会直接影响很多主要攻击的目标。

    阵型系统能不能让它转火¶

    基础配置里确实注册了战斗阵型系统，但这只怪的配置很特殊：它的确认分支更倾向于把目标重新压回主控角色。

    普通玩家可以这样理解：有些敌人会因为阵型刷新、仇恨权重、近战远程权重而有机会改打非主控队友；但 agtrinit 这份基础配置不是那一类。它的阵型刷新更像是在维护“主控就是目标”，而不是帮你把它引去打别人。

    因此，不建议期待用“让其他干员靠近、让其他干员先攻击、等阵型刷新”这类普通操作稳定骗它转火。当前证据更支持：它会被系统重新拉回主控目标。

    切主控有没有用¶

    有用，但它的含义是“改变当前主控是谁”，不是让怪物忘记玩家。

    因为很多攻击每轮都取当前主控角色，所以你切换主控后，后续很多招式会跟着新的主控走。这个效果比试图让它通过阵型系统转火更直接。

    不过也要注意：它仍然有少量分支会用敌人大脑目标，召唤、回位、跳退、部分位置行为也不完全等同于“攻击目标”。所以切主控能改变大量攻击指向，但不代表所有动作都立刻围绕新角色重算。

    站位不是换目标¶

    agtrinit 经常会做一些看起来像“换目标”的移动：后撤、回出生点附近、绕着目标走、离太远就靠近、离太近就找点拉开。

    这些动作更多是在调整距离和站位，不是重新索敌。尤其在搜查和诡计状态下，它会反复用这种站位逻辑保持合适距离，然后再继续出招。

    玩家视角总结¶

    你可以把它的索敌逻辑理解成三层：

    1. 关卡/战斗系统让它进入战斗，并通常把主控写成敌人大脑目标。
    2. 战斗中，它每轮都会重新确认“当前主控是谁”。
    3. 大量攻击直接打当前主控，少量动作使用敌人大脑目标或位置点。

    因此，正常游玩时它基本会表现为主控压力怪。想改变它打谁，最明确有效的是切换当前主控；想靠普通仇恨、阵型或队友站位让它稳定忘掉主控，目前没有确认机制支持。

    阶段机制不改变这个大方向：一阶段冲刺/近战、二阶段源石树全图高伤机制、三阶段隐身后的派生攻击，核心压力仍然主要围绕当前主控角色展开。二阶段的源石树更像“让主控进入安全区规避爆炸”的机制，而不是让怪物改锁其它队友。

    仍需保留的不确定点¶

    • 基础配置之外的 agtrinit_race 和其它相关变体没有纳入本篇。
    • 当前没有确认它每帧都会重新扫描一个新目标。
    • 少数底层节点仍未逐个展开到实现级，但不影响“多数攻击瞄当前主控”的玩家版结论。

    技能派生机制¶

    来源：technical-report/enemy-ai/aiconf_eny_0045_agtrinit_skill_derivation.player.zh.md
    原标题：agtrinit 玩家版出招逻辑说明

    日期：2026-06-28

    这篇只讲 aiconf_eny_0045_agtrinit 基础配置在战斗中大概怎么出招、什么时候切状态、玩家能看到哪些主要效果。索敌另见 aiconf_eny_0045_agtrinit_targeting.player.zh.md。

    总体规律¶

    agtrinit 不是单一技能循环怪，而是三种战斗状态来回组织出招：

    状态 玩家版理解 主要压力
    一阶段 / 强攻 开场陨石后的正面压制状态 冲刺、近战连段、突进、范围打击、陨石/投射物。
    二阶段 / 搜查 全图高伤机制 + 源石树安全区 + 召唤 先放树让玩家进护盾/安全区躲爆炸，然后召唤小怪，之后继续铺场。
    三阶段 / 诡计 隐身开始的特殊阶段 隐身/显形、通道伤害、连续投射物、大范围落弹、短眩晕。

    它开战后会先打一套固定入场：入场动作、朝主控砸陨石，然后进入一阶段。后续会根据状态标记、低血量/弱点相关事件和技能冷却，在一/二/三阶段之间切换。当前基础配置里最清晰的血量阈值是极低血量 2% 附近的保护/切换逻辑；不要把它理解成每掉一大段血就必定固定转阶段。

    开场会做什么¶

    第一次进入战斗时，它会按固定顺序行动：

    1. 做一次入场/接近动作。
    2. 朝当前主控角色发射一轮陨石类投射物。
    3. 切入一阶段，也就是强攻状态。

    所以它开场不是随机发呆，而是很快开始给主控角色压力。

    一阶段：强攻状态¶

    一阶段可以理解成“正面连招池”。它会围绕当前主控角色打近中距离技能，并且有几组成对的前后段技能。

    招式组 玩家能看到的行为
    skill01 -> skill011 近战双段打击，后续版本更重，带多段伤害、范围爆发和击飞。
    skill02 -> skill021 多段近战/范围打击，后续版本终段范围更大，也有击飞。
    skill33 -> skill331 突进或光环式范围伤害；后续版本是更强的双段突进/范围打击。
    skill35 / skill351 陨石或投射物类攻击；skill351 更像对位置连续砸三次。
    jumpback / jumpback0 距离过近或离出生点太远时后撤、回位。

    它如果离自己的起始位置太远，会优先尝试回位。它如果离主控太近，也可能先后撤再接后续技能。一阶段的核心体感是：主控被冲刺、近战和中距离范围技连续压住。

    二阶段：全图高伤、源石树和召唤¶

    二阶段开场不是普通技能随机池，而是一套固定机制：先放源石树，再引导玩家去树的安全区/护盾里躲避全图高伤，随后召唤小怪。

    第一次进入二阶段时，它会尝试固定做一套：

    1. 如果离起始位置太远，先回位。
    2. 放出两棵源石树。
    3. 树附近会形成安全区/护盾表现，玩家需要进入里面规避后续大伤害。
    4. 随后爆发一次大范围高额伤害；没有进入树安全区的主控会被命中。
    5. 接着召唤小怪或召唤实体。
    6. 记录这次召唤，然后开始绕着目标调整距离。

    数据里这套机制对应 skill232 -> summon。skill232 会在左右两侧生成两棵树，树的范围会给进入者一个“已经进安全区”的标记；后面的爆炸伤害会排除带这个标记的主控目标。因此，游戏里看到的“带护盾的源石树”本质上可以理解成躲全图伤害的安全区。

    这里的“护盾”更像视觉和机制安全区，不是单纯给角色一个可计算的数值盾。文档里看到的 shieldFx 是护盾视觉效果；真正规避伤害靠的是进入树范围后获得的安全区标记。

    二阶段开场之后，搜查状态的主要分支是：

    分支 玩家能看到的行为
    放树 skill232 在左右固定点生成两棵树，并触发大范围爆炸机制；进入树安全区可规避。
    召唤 summon 生成召唤相关实体；小怪死亡后，超过约 30 秒会再次尝试召唤。
    skill21 -> skill211 生成多个区域攻击实体，带区域伤害和可能的眩晕。
    skill231 -> skill23 搜查态的前后段投射物/冲击波链。
    skill352 对位置连续砸投射物，命中后还能继续派生冲击波/投射物效果。
    站位子树 绕着当前主控调整距离，太远靠近，太近找点拉开。

    这里有两个重要限制：有小怪存在时，部分“再召唤/冲击波”分支会被挡住；有冲击波标记时，部分同类分支也会互斥。玩家体感上，它不会无脑把所有场地技能同时堆出来，而是在召唤物、冲击波标记和冷却之间轮换。

    三阶段：隐身和诡计状态¶

    三阶段开始时会进入隐身相关流程。数据里对应 shift_skill21：它会创建隐身和不可受伤/特殊标记相关 buff，并切到 invisible 状态。玩家体感上，这就是“三阶段开场隐身”。

    进入三阶段后，它更偏“特殊机制和派生攻击”。

    招式组 玩家能看到的行为
    skill14 -> skill141 通道/持续类伤害；其中后续段少数情况下会改用敌人大脑目标。
    skill15 -> skill131 第一次进入诡计时优先级很高，先连续发射投射物，再接额外生成物/派生实体。
    skill354 大范围落弹/落雷，命中会制造约 0.8 秒短眩晕。
    shift_skill21 隐身/显形/恢复相关的大切换动作，优先级非常高。
    站位子树 绕目标保持距离，更多是调整位置，不是直接伤害。

    如果它身上出现 undeadable 相关状态，诡计池里会极高优先级触发 shift_skill21。玩家体感上，这类时刻会像是它进入一段特殊保护、隐身或显形切换流程，而不是普通技能随机。

    树和召唤机制怎么理解¶

    树不是普通一次性伤害。二阶段的树是“躲全图高伤”的核心机制。数据里它会生成两棵持续存在的实体，树自身还会在“正常、触发、破坏”几种状态之间切换。

    普通玩家可以先这样记：

    • 它第一次进二阶段会先放树，再结算大范围高伤，然后召唤。
    • 树周围有安全区/护盾表现，玩家进树范围会获得规避爆炸的标记。
    • 爆炸会检查主控是否没有这个安全区标记；没有标记才会吃高额伤害。
    • 树会作为场地机制持续一段时间，被触发或破坏后会切换不同状态。

    召唤也类似：它不是一句“召小怪”就结束，而是生成一个召唤相关实体，再由这个实体走后续效果链。玩家体感上可以理解成搜查状态的核心机制之一。

    它为什么经常绕来绕去¶

    很多技能之后都会接一段站位逻辑。这段逻辑大概是：

    1. 面向目标。
    2. 随机决定顺时针或逆时针绕。
    3. 如果距离合适，就绕着目标移动。
    4. 如果太远，就靠近。
    5. 如果太近，就找一个点拉开。

    这段逻辑本身不放技能，只是让它保持一个更舒服的攻击距离。你看到它绕圈、后撤、回位，不一定代表它换目标了。

    主要伤害来源速记¶

    类型 代表招式 玩家应注意
    近战连段 skill01/011, skill02/021 主控贴近时压力大，后续段范围和击飞更明显。
    突进/范围 skill33/331 中距离也会突然压上来。
    二阶段全图高伤 skill232 看到源石树后要进树的安全区/护盾里躲爆炸。
    陨石/投射物 skill35/351/352, skill15, skill23 不只瞄人，也可能砸位置，注意落点。
    大范围落弹 skill354 命中可带短眩晕，容易打乱节奏。
    区域实体 skill21/211, skill232, summon 场上会留下持续机制或派生效果。
    状态切换 shift_skill02, shift_skill21 可能伴随回位、保护、隐身/显形和恢复相关效果。

    对战体感总结¶

    agtrinit 的核心不是单招爆发，而是“主控锁定 + 多状态压制”：

    1. 开场先对主控打固定入场攻击。
    2. 一阶段用冲刺、近战、突进和陨石压主控。
    3. 二阶段开场固定放源石树安全区，随后结算全图/大范围高额伤害，再召唤小怪。
    4. 二阶段后续继续放树、召唤、铺场，并频繁调整距离。
    5. 三阶段开场隐身，之后使用投射物、通道伤害、大范围落弹和特殊切换。
    6. 很多攻击直接读取当前主控角色，所以切主控会比普通引仇恨更直接影响后续攻击方向。

    仍需保留的不确定点¶

    • self_heal 的具体回血数值这次没有精确展开。
    • skill23 命中后的部分内部条件还可以继续深挖。
    • 树的安全区/规避爆炸链路已经能从数据对上；但树和召唤里的少数底层命令仍没有逐条 IDA 展开。
