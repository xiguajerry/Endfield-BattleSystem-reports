    天鼓 hsbear¶

    本页把同一类怪物 eny_0082_hsbear 合并为一篇：基础版 aiconf_eny_0082_hsbear 与高难 S 版 aiconf_eny_0082_hsbear_hdg018 / indie_hard018_s 都放在这里。结构固定为两部分：索敌机制与技能派生机制。

    资料来源：

    • technical-report/enemy-ai/aiconf_eny_0082_hsbear_targeting.player.zh.md
    • technical-report/enemy-ai/aiconf_eny_0082_hsbear_skill_derivation.player.zh.md
    • technical-report/enemy-ai/aiconf_eny_0082_hsbear_hdg018_skill_derivation.player.zh.md

    索敌机制¶

    基础结论¶

    eny_0082_hsbear 的索敌很直接：进入战斗后，它的大多数实际出招都盯当前主控角色。

    它每轮战斗树开头会做三件事：

    1. GetMainCharacter -> Target：把主要目标 Target 写成当前主控角色。
    2. EnemyGetTargetFromBrain -> SubTarget：再读一次系统 brain 目标，但这份目标在当前主战斗树里基本只是保存为 SubTarget。
    3. GetRandomMember -> RandomMember：随机取一个队伍成员，给少数冲刺、侧移或派生攻击分支使用。

    所以玩家视角可以这样理解：

    现象 实际含义
    大多数技能都朝当前操控角色放 主战斗树每轮都会重新把 Target 写成主控角色。
    换主控后它很快转向新主控 下一轮行为树刷新时，Target 会变成新的主控角色。
    它偶尔突然冲向别的干员 多数是 RandomMember 分支，不代表仇恨永久转移。
    愤怒后仍然追主控 愤怒改变技能池和数值压力，不改变“主目标=主控”的规则。

    它什么时候会打其他队员？¶

    天鼓不是完全不会看其他队员，但这些分支更像“临时动作目标”：

    • eny_0082_hsbear_dash_f / dash_b / dash_l / dash_r：当分支把目标绑定成 RandomMember 时，位移会按随机队员调整方向。
    • dash_f -> eny_0082_hsbear_skill02：近距离调位池里，若对应角度分支成立且 skill02(RandomMember) 可放，就会先冲向随机队员，再接三段近战连击。
    • eny_0082_hsbear_dash_l_angry / dash_r_angry：愤怒态随机队员 dash 分支被选中后，会按 RandomMember 角度决定左/右侧移，并且带伤害。

    这些动作结束后，下一轮行为树仍会把主要攻击目标刷回当前主控角色。

    和通用仇恨/brain 的关系¶

    基础配置里天鼓的仇恨等级是 Middle，attackRangeType=Melee，且没有覆盖仇恨概率；通用配置里 Middle 对应的仇恨抽样参数是 12%，阵型目标立即刷新冷却是 2s，忽略主控相关冷却是 6s。这些底层通用系统会参与入战、站位、brain target 与阵型刷新。但本怪的 Battle 行为树会直接用 GetMainCharacter 覆盖攻击用 Target，因此：

    • 通用仇恨系统可以影响它是否进入战斗、怎样靠近、brain 里临时记录谁；
    • 但大多数 EnemySwitchBehavior(... target=Target) 最终读到的仍是主控角色；
    • SubTarget 不是主要出招目标；RandomMember 只解释少数随机队员分支。

    技能派生机制¶

    基础版整体循环¶

    基础版 eny_0082_hsbear 可以理解成三套行为：

    平时盯主控角色
  -> 按距离、角度、冷却选择 skill01~skill09 或 dash/fightwalk
  -> 血量和计时条件满足时尝试 skill06
  -> skill06 按当前 HP 给自己叠 angry_fx
  -> 有 angry_fx 时进入愤怒技能池
  -> 愤怒技能池使用 angry 变体、长连段、多 projectile 和带伤害 dash

    权重/概率读法¶

    行为树里有两种不同的“随机”：

    • 权重池：ConditionalProbabilitySelector / ProbabilitySelector 使用权重。表里写成 权重；全可用归一化，例如 35；43.75% 的意思是：如果同一个权重池里所有列出的分支都满足冷却、距离、角度等条件，就按 35 / 权重总和 抽取。实际战斗中如果某个分支 CD 没好、距离/角度不对，它会被排除，剩余分支重新按权重抽。
    • 真实概率门：Probability(0.7/1.0) 就是 70% 通过；Probability(0.6/1.0) 就是 60% 通过。
    • 普通 Selector：只按顺序尝试，不是随机。前面的分支成功后，后面的分支不会再执行。

    基础行为里还有几个固定数值会影响节奏：每轮会随机写入 Time_StarfeMove = 1.2s ~ 1.8s；随机队员调位/偷袭池有 >=8s 的刷新门；周期 dash 调位有 >=6s 的刷新门。

    基础版普通状态怎么选招？¶

    基础版普通态先处理一个“随机队员调位/偷袭池”，然后再进按距离分段的主技能池。

    随机队员调位/偷袭池¶

    触发门槛：RandomMember <= 8m、Target <= 8m，并且 NowTime_AdjustAttack >= 8s。通过后会按角度进入下面的顺序 Selector；这个池不是权重随机，而是按表内顺序依次尝试。

    角度/距离门槛 顺序尝试
    Target 角度 >90° 且 <270° 先 CanCast(skill04) 则放 eny_0082_hsbear_skill04；否则如果 CanCast(skill02, RandomMember)，则 dash_f(RandomMember) -> eny_0082_hsbear_skill02(RandomMember)。
    Target 角度 >45° 且 <135° 先 CanCast(skill01)，再过 70% 概率门，成功则放 skill01；否则尝试 CanCast(skill03) + 70% 概率门，成功则放 skill03；再否则尝试 dash_l(RandomMember)。
    Target 角度 >225° 且 Target >5m 先 CanCast(skill02)，再过 70% 概率门，成功则放 skill02；否则尝试 CanCast(skill03) + 70% 概率门，成功则放 skill03；再否则尝试 dash_r(RandomMember)。

    这个分支成功后会刷新 SaveTime_AdjustAttack，所以下一次至少要满足 >=8s 的间隔才能重新进入。

    基础版愤怒怎么触发？¶

    AI 会在若干 HP 与计时门满足时尝试 eny_0082_hsbear_skill06。这些是“是否尝试转愤怒”的门槛，不是随机概率：

    AI 尝试门槛 数值
    第一轮 HP 门 HP <=90% 且 Hpcheck01 < 1。
    中血重复门 HP <=85% 且 HP >30%，并且愤怒计时门 >=10s，Hpcheck02 < 3。
    低血重复门 HP <=30% 且 Hpcheck03 < 3。
    skill06 自身 CD 16s。

    skill06 真正决定 angry_fx 层数的是技能内 HP 检查：

    当前 HP buff_eny_0082_hsbear_angry_fx 层数 玩家能看到的压力
    HP < 25% 3 层 进入最高压愤怒；最多提供 +30% 攻击。
    25% <= HP < 55% 2 层 中等愤怒；最多提供 +20% 攻击。
    HP >= 55% 1 层 低层愤怒；提供 +10% 攻击。

    只要 angry_fx 至少 1 层，行为树就会切入愤怒技能池。angry_fx 每层提供 +10% 攻击，最大 3 层；多个弱点窗口被触发时会消掉 angry_fx 层数。愤怒不改变索敌规则，仍主要盯主控。

    高难 S 版 indie_hard018_s 的额外机制¶

    高难 S 版仍是天鼓，但关卡把它改成“平静态气旋 -> 愤怒护盾 -> 异常消盾”的循环。

    indie_hard018_s 只刷一只 90 级 eny_0082_hsbear_hdg018：出生 3s 后出现，面向主控干员。出生自带：

    项目 数值/机制
    最大生命 buff_common_maxhpup 让最大生命变为 150%。
    平静态护盾 五种承伤倍率都是 50%，也就是平静态硬打只吃一半伤害。
    平静态气旋 天鼓对主控干员造成伤害后，若内置 1s 冷却结束，会生成 1 个气旋。
    S 版开关 S 版没有 normaltag，所以气旋生成条件可以通过；非 S 版带 normaltag，不按同样方式触发。

    核心循环：

    平静态盯主控出招
  -> 打到主控后留下气旋（1 秒最多 1 个）
  -> 开战计时至少 20 秒且 skill06_hdg018 可放时，尝试转愤怒
  -> skill06_hdg018 吸收自己生成的气旋
  -> 护盾层数 = 气旋数量 + 固定 5 层 + phase 层
  -> 每层护盾同时给天鼓 +10% 攻击
  -> 我方每造成 1 次法术异常或物理异常，消 1 层护盾并移除 1 层 +10% 攻击
  -> 护盾消完后回到平静态，并给后续循环追加 phase（最多 2 层）

    项目 机制
    触发条件 天鼓在平静态对主控干员造成伤害。
    内置冷却 1s 最多生成 1 个。
    生成物 abilityentity_eny_0082_hsbear_hdg018。

    skill06_hdg018 怎么把气旋变成护盾？¶

    帧段 发生的事
    6-9 关闭平静态气旋生成 buff，避免转愤怒过程中继续生成。
    15-73 创建转阶段保护 buff：恢复削韧，并让承伤降低 90% 左右。
    68-69 寻找 owner 自己生成过的气旋实体，把数量存进 counts，并让气旋向天鼓发射吸收 projectile。
    70-73 清掉被吸收的气旋实体。
    73-104 转阶段本体伤害：物理 HP 25% 攻击倍率，单目标最多 2 次，每 0.35s 触发一次。
    112-115 移除平静态 50% 承伤护盾，创建愤怒护盾。

    进入愤怒护盾时：

    护盾层数 = 吸收的气旋数 + 5 + phase

    phase 是上一次愤怒结束后叠的额外层数，最多 2 层。所以没有气旋时也至少有 5 层；气旋越多，下一轮护盾和攻击提升越高。

    愤怒护盾期间发生什么？¶

    项目 机制
    承伤倍率 物理、晶格、火、脉冲、自然五种承伤倍率都变为 20%，也就是 80% 减伤。
    攻击提升 每层护盾对应 1 层 count_atkup，每层 +10% 攻击。例如 8 层就是 +80% 攻击。
    周期追击 每 2s 创建一段 3s timeline，在第 15/25/33/38/41/43 帧连续发射 attack06_01。
    追击命中 skill06_01_projhit_hdg018 第 33-40 帧造成自然 HP 75% 攻击倍率。
    消盾方式 我方每施加 1 次法术异常或物理异常，消 1 层护盾，并同时消 1 层 +10% 攻击。
    护盾结束 最后一层被移除后，愤怒护盾结束，恢复平静态气旋机制，并让后续 phase 增加 1 层，最多累计 2 层。

    玩家应对重点¶

    1. 它主要盯主控：换主控会明显改变主要压制目标，但不会让随机队员分支消失。
    2. 看到它冲向其他干员，多数是 RandomMember 位移/派生，不等于永久换仇恨。
    3. 基础版重点看 skill06：HP 越低，angry_fx 层数越高；每层 +10% 攻击，最多 +30% 攻击。
    4. 基础版愤怒后特别注意 skill02_angry_02 的超长连段、skill04_angry 的五枚小弹、skill07_angry 的三枚大弹。
    5. 高难 S 版不要只看气旋数量：愤怒护盾层数是“气旋数 + 固定 5 层 + phase 层”，每层还给 +10% 攻击。
    6. 高难 S 版愤怒护盾期间硬打效率很低，因为五种承伤只有 20%；稳定施加法术异常或物理异常可以逐层拆盾，并同步移除攻击提升。
