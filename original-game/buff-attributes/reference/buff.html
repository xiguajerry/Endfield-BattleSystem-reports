    Buff¶

    更新时间：2026-06-15

    字段¶

    字段 offset 说明
    m_container 0x10 owner BuffContainer。
    m_data 0x18 BuffData。
    m_duration 0x20 duration。

    m_triggerCnt 0x28 trigger 次数。

    m_buffActions 0x40 buff event actions。
    m_abilityActions 0x48 ability event actions。
    m_damageModifiers / m_healModifiers / m_poiseModifiers / m_globalModifiers 0x58 / 0x60 / 0x68 / 0x70 modifier lists。
    m_childrenBuff 0x78 child buffs。
    m_inheritSkillIdList 0x80 skill inherit list。
    m_shieldInsts 0x88 shield instances。
    m_timelineActionProcessor 0x90 timeline action processor。
    m_parentBuffRoot 0x98 parent root。
    m_lifeTime / m_passedTime 0xA0 / 0xA4 lifetime / elapsed time。
    m_enhanceCnt / m_priority 0xA8 / 0xAC enhance count / priority。
    m_serverInstId / m_clientInstId 0xB8 / 0xC8 server/client buff instance id。
    instanceUid backing field 0xFC local buff instance uid。
    _blackboard 0x158 action blackboard。
    _owner / _source 0x170 / 0x180 owner/source AbilitySystem。
    _curFrame 0x19C current frame。

    方法¶

    方法 状态 状态 结论
    IsFinishEarly(FinishReason) 已确认 已确认 compiler-shared body；native code returns true for enum code 2 (Early) or 1 (Ignite)，otherwise false。
    OnStart() 已确认 已定位 buff start lifecycle。
    OnIgnite(IgniteType, AbilitySystem, AbilitySystemForEnemyPart, IActionEnvironment) 已确认 已确认 遍历 BuffData.igniteEventAction；igniteType 匹配时执行同 index 的 ignite SequenceAction.ExecuteInstant 列表并返回 true。若 IgniteActionMap.finishAfterIgnited 为 true，则调用 BattleRecorder.RecordAbnormalConsumeInfliction(igniteSource, igniteType, m_enhanceCnt)，循环结束后以 FinishReason.Ignite 调 MarkFinish。
    OnFinish(FinishReason) 已确认 已定位 finish lifecycle。
    MarkFinish(...) 已确认 已定位 mark finish with reason/source/cast info。
    Release(bool) 已确认 已定位 release lifecycle。

    调用入口¶

    • AbilitySystem.m_buffContainer 管理实体 buff。
    • BuffContainer.CreateBuff / StackBuff 创建 runtime buff。
    • buff 的 passive/timeline actions 触发 ability event、damage/heal/poise/global modifiers、child buff、shield、time dilation、timed marker 等副作用。

    生命周期¶

    1. BuffContainer 根据 BuffData 和 AddBuffOption 创建/stack buff。
    2. Buff.OnStart 执行 start actions 和 modifier apply。
    3. Buff.OnTick 推进 trigger timer、timeline actions、duration。
    4. finish/ignite/dispel/absorb 走 MarkFinish / OnFinish，携带 FinishReason。
    5. release 回收 child handles、modifiers、timeline processor 和 owner/source references。

    已确认行为¶

    • FinishReason.Other 原始 numeric code 是 100，不能按 enum ordinal 解析。
    • IsFinishEarly 在原游戏中认为 Ignite(1) 与 Early(2) 都属于 early finish helper 返回 true。
    • Buff.CompareTo 排序优先级是 priority 高者靠前，其次 lifeTime 长者靠前，最后 instanceUid 小者靠前。
    • Buff.OnTick 的单 buff lifecycle 不在 BuffContainer 内部实现；container 只遍历，trigger/lifetime/timeline 属于 Buff.OnTick。
    • Buff.OnIgnite 中 abnormal consume recorder 的 count 来源是当前 buff 的 m_enhanceCnt，offset 0xA8；只有匹配的 IgniteActionMap.finishAfterIgnited 为 true 时记录。
