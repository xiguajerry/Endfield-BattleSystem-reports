    GlobalBuff¶

    更新时间：2026-06-14

    字段¶

    字段 offset 说明
    m_data 0x10 GlobalBuffData。
    m_priority 0x18 stacking / priority queue 排序。
    m_isFinished / m_isFinishing 0x1C / 0x1D finish state。
    m_isStarted / m_isEnabled 0x1E / 0x1F lifecycle state。
    m_buffListMap 0x20 按角色 server id 记录由 global buff 添加的实体 buff。
    childrenBuff backing field 0x28 IBuffRoot child buff list。
    m_globalModifiers 0x30 global modifier runtime。
    m_globalBuffActions 0x38 global buff event actions。
    source backing field 0x48 source AbilitySystem。
    isServerGlobalBuff backing field 0x58 server/global origin flag。
    instId backing field 0x60 global buff instance id。
    context / owner / executingState 0x68 / 0x70 / 0x80 action environment。
    curFrame 0x84 current frame。
    sourceCharInstId 0x88 source character instance id。
    lifeTime / duration 0x90 / 0x94 runtime lifetime / total duration。
    m_triggerCnt / m_triggerTimer 0x98 / 0xA0 trigger timer。
    blackboard 0xA8 action blackboard。
    instanceUid 0xB0 local uid。

    枚举¶

    enum 值
    GlobalBuff.Event OnBuffStart=0, OnBuffTrigger=1, OnBuffFinish=2, OnBuffEnable=3, OnBuffDisable=4, DuringBuffEnable=5, OnBuffStartButWaitForLevelLoaded=6, EnumMax=7
    GlobalBuff.LifeType Limited=0, Infinity=1
    GlobalBuff.FinishReason Default=0, Early=1, Server=2
    GlobalBuffStackingSettings.IdentifierType Id=0, StackingKey=1
    GlobalBuffStackingSettings.StackingType Unlimited=0, HighPriority=1, Stack=2, Refresh=3, Extend=4, StackAndRefresh=5, OverwriteDuration=6

    调用入口¶

    • BattleManager.AddGlobalBuff(...) is the owner entry. It resolves GlobalBuffData, resolves/creates BattleManager.GlobalBuffStackingGroup, calls StackGlobalBuff, then only raises added events when the result is non-null.
    • BattleManager.RemoveGlobalBuff(...) resolves GlobalBuffData.get_stackingKey() then removes by id/count/reason inside the matched stacking group。
    • BattleManager._TickGlobalBuff(float) ticks global buff stacking groups。
    • GlobalBuff applies entity child buffs via GlobalBuffData.buffInputs and owns those child buff references through IBuffRoot。

    生命周期¶

    1. BattleManager.AddGlobalBuff resolves GlobalBuffData and stacking key。
    2. GlobalBuffStackingGroup.StackGlobalBuff allocates / refreshes / stacks a GlobalBuff instance。
    3. GlobalBuff.Init loads blackboard, modifiers and actions。
    4. OnStart / OnEnable apply child buffs and global modifiers。
    5. OnTick advances global buff action tick / trigger / lifetime。
    6. MarkFinish / OnFinish disable modifiers and finish child buffs。
    7. GlobalBuffStackingGroup recycles finished instances。

    已确认行为¶

    • 原游戏 global buff owner 在 BattleManager，不是 AbilitySystem.BuffContainer。
    • BattleManager.AddGlobalBuff 的 added event / predefined global event side effect 依赖 StackGlobalBuff 返回新实例；refresh/extend/overwrite branch 返回 null 时不触发 added lifecycle。
    • GlobalBuff 是 IBuffRoot，会持有并管理由 buffInputs 创建的实体 child buffs。
    • GlobalBuff.Event 与 Buff.Event 类似但额外包含 OnBuffStartButWaitForLevelLoaded=6。
    • GlobalBuffData.stackingSettings 使用 GlobalBuffStackingSettings 独立 enum；本地数据 global_buff_mifu_chrdg_atbrecover 的 stackingType=5 应解析为 StackAndRefresh。
    • GlobalBuffData.get_stackingKey() 使用 IdentifierType 决定 group key；BattleManager.RemoveGlobalBuff 不直接按 id 查 group，而是先从 data 解析 stacking key。
    • GlobalBuff.CompareTo 使用 m_priority 降序；当 priority 差值在 float epsilon 内时使用 instanceUid.CompareTo(other.instanceUid)，即更早创建的实例优先。
    • GlobalBuff.Init priority 分支已确认：stackingSettings.usePriorityKey=false 时使用配置 priority；为 true 时从 blackboard 读取 priorityKey，negatePriority=true 时乘以 -1，否则乘以 +1。
    • GlobalBuff.Init 会从 GlobalBuffData.globalModifier 创建 GlobalModifier 列表，保存到 m_globalModifiers；GetGlobalModifiers 直接返回该列表。
    • GlobalBuff.OnEnable 会通过 BattleManager.AddGlobalModifiers(this) 注册自身 global modifiers；OnDisable 通过 BattleManager.RemoveGlobalModifiers(this) 移除。也就是说只有 enabled global buff 的 modifier 应当影响 BattleManager 当前全局属性。
    • GlobalBuff.OnTick local lifecycle 已确认：先更新 curFrame，enabled 时进入 _TickGlobalBuffAction，随后 limited lifetime 扣减并在过期时 MarkFinish(Default)。
    • GlobalBuff.Init trigger 分支已确认：m_triggerCnt 来自 GlobalBuffData.maxTriggerCnt，按 BlackboardInt 从 global buff blackboard 解析；triggerInterval <= 0 时 m_triggerTimer.MarkInvalid()，否则 m_triggerTimer.Reset(triggerInterval, waitFirstTriggerInterval)。
    • GlobalBuff._TriggerInternal 使用 PrecisePeriodicTimer.UpdateMultiple/NextMultiple catch-up，因此单个大 delta 可以触发多次 OnBuffTrigger；waitFirstTriggerInterval=false 会让 timer 初始 ready，首次 tick 即可触发。
    • MarkFinish 防重入，先 OnFinish，再 disable，最后置 finished；native 还发送全局事件。
    • GlobalBuff._AddBuffsToCharacter 是实例级 child buff owner：child buff refs 存在当前 GlobalBuff.m_buffListMap 中，而不是只按 global buff id 管理。
    • GlobalBuff._DisableBuffs 只 disable 当前实例记录的 child buffs；_FinishBuffs 才将这些 child buffs 以实体 Buff.FinishReason.Other(100) finish，并清空实例 map。
