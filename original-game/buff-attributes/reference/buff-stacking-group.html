    Buff.BuffStackingGroup¶

    更新时间：2026-06-14

    字段¶

    字段 offset 说明
    m_stackingKey 0x10 stacking group key。
    m_stackingType 0x18 BuffStackingSettings.StackingType。
    m_maxStackCnt 0x1C 最大层数。
    m_curStackCnt 0x20 当前层数缓存。
    m_lastEffectStackCnt 0x24 stack effect 上次层数。
    m_isNeedStackEffect 0x28 是否需要 stack effect。
    m_stackBuffEffectData 0x30 stack effect 数据。
    m_effectInstances 0x38 stack effect 实例。
    m_owner 0x40 owner AbilitySystem。
    m_source 0x48 source AbilitySystem。
    m_buffs 0x50 PriorityQueue<Buff>。

    方法¶

    方法 状态 状态 结论
    StackBuff(BuffData, AbilitySystem, AbilitySystem, AddBuffOption, out Buff) 已确认 已确认 按 m_stackingType 分支处理新增/刷新/增强/替换；多数 case 被编译到 0x186339xxx code chunk。
    _AllocateBuff(AbilitySystem, BuffData, in AddBuffOption) 已确认 已确认 从 BuffPool.Allocate(buffData.id) 取实例，然后调用 Buff.Reset(buffData, owner, source, option)。
    InitMaxStackCount(Buff) 已确认 已确认 若 BuffStackingSettings.isStackType，从 buff 读取 maxStackCount 写入 m_maxStackCnt。
    RemoveBuff(Buff) 已确认 已确认 从 priority queue 删除；删除成功后调用 Refresh()。
    Refresh() 已确认 已确认 根据 stacking type 重算层数或 enabled 状态；server enable-state op 属于联网/上报边界。
    RefreshPriority(Buff) 已确认 部分确认 priority type 使用 queue 顺序刷新 enabled 状态；server op 忽略。
    _SetBuffDurationRefresh(Buff, Buff) 已确认 已确认 有任一 Infinity 则旧 buff 变 Infinity；否则旧 lifetime 取 max(new.duration, old.lifeTime - epsilon)。
    _SetBuffDurationExtend(Buff, Buff) 已确认 已确认 有任一 Infinity 则旧 buff 变 Infinity；否则旧 lifetime 加新 duration。
    _SetBuffDurationOverwrite(Buff, Buff) 已确认 已确认 复制新 buff 的 life type，并把旧 lifetime 设置为新 duration。
    _GetFirstUnFinishedBuff() 已确认 已确认 按 priority queue 顺序取第一条未 finished buff。
    _GetLastUnFinishedBuff() 已确认 已确认 按 priority queue 顺序从末尾取未 finished buff。
    _GetUnFinishedBuffCount() 已确认 已确认 统计未 finished buff。
    ## 调用入口
    - Buff.BuffContainer.CreateBuff(...) 按 buff id 或 stacking key 找到/创建 BuffStackingGroup 后调用 StackBuff(...)。
    - Buff.BuffContainer.FinishBuff(...) / RemoveBuff(...) 结束实例后通过 group Refresh() 保持 enabled 和层数状态。

    生命周期¶

    1. group 构造时从 BuffData.stackingSettings 读取 key/type/max/effect 数据并创建 priority queue。
    2. StackBuff 根据 stacking type 分支分配新 buff、复用旧 buff、刷新 duration 或增强旧 buff。
    3. RemoveBuff 成功删除实例后调用 Refresh()。
    4. Release 清理 stack effects 和 queue。

    已确认行为¶

    • BuffStackingSettings.StackingType 原始 code：Unlimited=0、HighPriority=1、Stack=2、Enhance=3、Refresh=4、Extend=5、Modify=6、Unique=7、EnhanceAndRefresh=8、OverwriteDuration=9、EnhanceAndOverwriteDuration=10、HighPriorityWithMaxStack=11。
    • Stack 达到 m_maxStackCnt 时先取最后一个未完成 buff，调用 MarkFinish(..., FinishReason.Other=100)，再加入新 buff，并用 _GetUnFinishedBuffCount 更新 m_curStackCnt。
    • Enhance 已有 buff 时不加入新实例，而是对第一条未完成 buff 调 _OnBeforeTryEnhanced、必要时 m_curStackCnt++、_RefreshStackEffects()、Buff._Enhance()、_OnAfterTryEnhanced，并返回 null。
    • EnhanceAndRefresh / EnhanceAndOverwriteDuration 已有 buff 时先按 Enhance 规则增强旧实例，再分别 refresh/overwrite duration；不加入新实例。
    • Refresh / Extend / OverwriteDuration / Modify 已有 buff 时复用第一条未完成 buff，不加入新实例；native StackBuff 返回 null，并通过 oriBuffInst 输出旧实例。
    • Unique 已有未完成 buff 时 oriBuffInst=null 且返回 null。
