    BattleManager.GlobalBuffStackingGroup¶

    更新时间：2026-06-14

    字段¶

    字段 offset 说明
    m_stackingKey 0x10 global buff stacking key；constructor 调 GlobalBuffData.get_stackingKey() 写入。
    m_stackingType 0x18 GlobalBuffStackingSettings.StackingType；constructor 从 GlobalBuffData.stackingSettings.stackingType 写入。
    m_maxStackCnt 0x1C max stack count；constructor 从 GlobalBuffData.stackingSettings.maxStackCnt 写入。
    m_curStackCnt 0x20 current stack count cache；get_curStackCnt() 直接返回该字段。
    globalBuffs 0x28 PriorityQueue<GlobalBuff>。

    方法¶

    方法 状态 状态 结论
    GlobalBuffData.get_stackingKey() 已确认 已确认 IdentifierType.Id(0) 返回 GlobalBuffData.id；IdentifierType.StackingKey(1) 返回 stackingSettings.stackingKey。
    get_curStackCnt() 已确认 已确认 直接返回 this+0x20。
    BattleManager.AddGlobalBuff(string, AbilitySystem, List<DataPair>, bool) 已确认 已确认 外层 owner：TryGetData 取 GlobalBuffData，用 get_stackingKey() 查/建 m_stackingGroups，调用 StackGlobalBuff；只有返回新 GlobalBuff 时才触发 ON_GLOBAL_BUFF_ADDED / predefined global event。
    StackGlobalBuff(GlobalBuffData, AbilitySystem, List<DataPair>, bool) 已确认 已确认 按 global stacking type 执行 allocate/enable/finish/refresh/extend/overwrite。
    BattleManager.RemoveGlobalBuff(string, int, FinishReason) 已确认 已确认 外层先 BattleAssetTable.TryGetData 取 GlobalBuffData，再用 get_stackingKey() 查 m_stackingGroups，命中后转发 group remove。
    RemoveGlobalBuff(string, int, FinishReason) 已确认 已确认 从 queue 尾到头遍历 id，相等则 MarkFinish(finishReason)；finishReason == Server(2) 时只 finish isServerGlobalBuff。finishCount <= 0 表示不限数量。
    OnTick(float) 已确认 已确认 先从 queue 末尾扫描并 _RecycleGlobalBuff 已 finished/null 实例；随后枚举 queue，对未 finished 实例执行 GlobalBuff.OnTick(float) 语义。
    _AllocateGlobalBuff(...) 已确认 已定位 allocate/init global buff。
    _RecycleGlobalBuff(GlobalBuff) 已确认 已确认 queue remove 成功后 RefreshStackingGroup()，再回收到 GenericStaticPool<GlobalBuff>。
    _GetFirstUnFinishedGlobalBuff() 已确认 已定位 queue lookup helper。
    RefreshStackingGroup() 已确认 已确认 HighPriority 只 enable 第一个未 finished 项，其余 disable；Stack / StackAndRefresh 刷新 m_curStackCnt = globalBuffs.count。

    StackGlobalBuff 行为¶

    已确认:

    GlobalBuffStackingSettings.StackingType code 行为
    Unlimited 0 _AllocateGlobalBuff，set_isEnabled(true)，加入 globalBuffs。
    HighPriority 1 _AllocateGlobalBuff，加入 queue；随后遍历 queue，只启用第一个未 finished global buff，其余 disable。最终排序依赖 PriorityQueue / GlobalBuff.CompareTo。
    Stack 2 _AllocateGlobalBuff；若 m_maxStackCnt > 0 && m_curStackCnt >= m_maxStackCnt，取 _GetFirstUnFinishedGlobalBuff() 并 MarkFinish(Default)；加入 queue，刷新 m_curStackCnt，启用新 buff。
    Refresh 3 若无未 finished buff，退回 Unlimited allocate path；否则 allocate 一个临时 global buff，仅用其 duration，把旧 buff lifeTime 设为 max(old.lifeTime, new.duration)，返回 null。
    Extend 4 若无未 finished buff，退回 Unlimited allocate path；否则 allocate 临时 global buff，仅用其 duration，把旧 buff lifeTime += new.duration，返回 null。
    StackAndRefresh 5 先执行 Stack 类似的 max stack gate、加入 queue、启用新 buff；然后遍历 queue，把所有未 finished buff lifeTime 设为 max(existing.lifeTime, new.duration)。
    OverwriteDuration 6 若无未 finished buff，退回 Unlimited allocate path；否则 allocate 临时 global buff，仅用其 duration，把旧 buff lifeTime = new.duration，返回 null。
    default - 返回 null。

    BattleManager.AddGlobalBuff 外层流程¶

    已确认:

    1. 从 GameInstance.dataManager.battleAssetTable 调 BattleAssetTable.TryGetData<GlobalBuffConfig, GlobalBuffData>(globalBuffId)。
    2. 调 GlobalBuffData.get_stackingKey() 得到字典 key。
    3. 在 BattleManager.m_stackingGroups 查找现有 GlobalBuffStackingGroup。
    4. 未命中时创建 group：写 m_stackingKey、m_stackingType、m_maxStackCnt，并分配 PriorityQueue<GlobalBuff>，随后 add 到字典。
    5. 调 GlobalBuffStackingGroup.StackGlobalBuff(data, source, assignItems, isServerGlobalBuff)。
    6. StackGlobalBuff 返回 null 时，直接返回 null ObjectPtr<GlobalBuff>，不触发 added event。
    7. 返回新 GlobalBuff 时，如果 LevelEventManager.NeedTriggerLevelEvent(ON_GLOBAL_BUFF_ADDED) 为 true，则构造 EventParams，sender 为 source.entity，写入 global_buff 参数并 RaiseLevelEvent。
    8. 随后通过 EventManager.SendGlobal 发送 predefined global event，参数是 GlobalBuff 和 globalBuffId。

    调用入口¶

    • BattleManager.AddGlobalBuff 按 GlobalBuffData.get_stackingKey() 获取或创建 group，然后调用 StackGlobalBuff。
    • BattleManager.RemoveGlobalBuff 先用 GlobalBuffData.get_stackingKey() 查找 group，再调用 group remove。
    • BattleManager._TickGlobalBuff 遍历所有 global buff groups 并调用 OnTick。

    生命周期¶

    1. group 由第一层 global buff 创建。
    2. 后续同 stacking key/id 的 global buff 进入同一 group。
    3. OnTick 先回收上一轮已 finished instance，再 tick 当前未 finished global buff。
    4. group empty 后由 BattleManager 从全局字典移除。

    已确认行为¶

    • Global buff stacking 与实体 buff stacking 分离；原游戏 owner 是 BattleManager.GlobalBuffStackingGroup。
    • group 内部使用 PriorityQueue<GlobalBuff>，因此 GlobalBuff.CompareTo 会影响 HighPriority/priority 类 stacking 行为。
    • StackGlobalBuff 完整 stacking type switch 已 已确认，见上表。
    • BattleManager.AddGlobalBuff 外层 owner 已 已确认；字典 key 与 group constructor 都使用 GlobalBuffData.get_stackingKey()。
    • BattleManager.AddGlobalBuff added event 只在 StackGlobalBuff 返回新实例时触发；refresh/extend/overwrite 仅修改旧实例并返回 null。
    • GlobalBuffData.get_stackingKey() 受 IdentifierType 控制：Id 使用自身 id，StackingKey 使用配置的 stacking key。
    • BattleManager.RemoveGlobalBuff 的字典 key 是 get_stackingKey()，不是无条件 global buff id。
    • RemoveGlobalBuff 从 queue 尾到头查 id 并调用 GlobalBuff.MarkFinish；server reason 分支只作用于 server global buff。
    • _TickGlobalBuff 传入的 scaled delta 会直接进入 group tick；TimeManager/BattleManager tick owner 的 global scale 会影响 global buff lifetime。
    • OnTick 回收发生在 tick 当前实例之前；本 tick 刚 MarkFinish 的实例会在下一次 group tick 被 _RecycleGlobalBuff 移出 queue。
    • _RecycleGlobalBuff remove 成功后调用 RefreshStackingGroup。
    • RefreshStackingGroup 中 HighPriority 只启用第一个未 finished 实例，其他实例 disabled。
