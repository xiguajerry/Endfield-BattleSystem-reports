    Buff.BuffContainer¶

    更新时间：2026-06-14

    字段¶

    字段 offset 说明
    m_buffs 0x10 DoubleBufferedBuffList，所有 buff。
    m_stackingGroups 0x18 Dictionary<string, BuffStackingGroup>。
    m_owner 0x20 owner AbilitySystem。
    buffEventActions 0x28 buff event callback。
    m_buffChangeCharHpBarVfx 0x30 HP bar VFX related buffs。
    s_tempBuffIdSet static temp buff id set。

    方法¶

    方法 状态 状态 结论
    CreateBuff(AbilitySystem, BuffData, AddBuffOption, out Buff) 已确认 已确认 创建 buff 的外层入口：构建 source/owner AddBuffContext，触发 before-events，按 stacking key 获取/创建 group，调用 StackBuff；仅 StackBuff 返回新 buff 时追加到 m_buffs 并继续 added/output lifecycle。具体 stacking 分支见 buff-stacking-group.md。
    ContainsBuff(string) 已确认 已定位 id query。
    FinishBuff(Buff, ServerActionParams) 已确认 已定位 finish specific buff。
    FinishBuff(string, FinishBuffOption, ServerActionParams) 已确认 已确认 finish by id；needLimitLayerCnt 为 false 时结束全部匹配，true 时按 finishLayerCnt 限制；enhance 类 stacking 改走 Buff.DecreaseEnhanceCnt。
    FinishBuffByTag(GameplayTagQuery, FinishBuffOption, ServerActionParams) 已确认 已确认 遍历 m_buffs，按 BuffData.applyTags 匹配 GameplayTagQuery，收集匹配 buff id 到静态临时 set 去重，再逐 id 调 FinishBuff(id, option, opParams)。
    Buff.DecreaseEnhanceCnt(FinishBuffOption) 已确认 已确认 enhance 类 buff 扣层入口；扣完或不足时调用 MarkFinish，否则减少 m_enhanceCnt 并触发 enhance change 相关事件。
    IgniteBuff(...) 已确认 已定位 ignite buff path。
    DispelBuff(...) 已确认 已定位 dispel path。
    Clear() / Release() / RecycleBuff() 多个 已定位 container cleanup。

    调用入口¶

    • AbilitySystem.m_buffContainer 是实体 buff container。
    • AbilitySystem.AddBuffByAbilityAction / FinishBuff / IgniteBuff / DispelBuff 是外层 facade，实际委托 container。
    • BattleManager 只管理 global buff；实体 buff 应归 AbilitySystem.BuffContainer。

    生命周期¶

    1. AbilitySystem 初始化时创建 BuffContainer(owner)。
    2. add buff 时按 stacking key/id 找或创建 BuffStackingGroup。
    3. container tick 遍历 m_buffs 并分发到每个 valid Buff.OnTick(deltaTime, allScaledDeltaTime)。
    4. finish/ignite/dispel 根据 option/source/cast info 修改 buff 状态。
    5. release 时清理所有 buff 与 stacking group。

    已确认行为¶

    • 原游戏 nested class 是 Buff.BuffContainer，不是单独 BuffTable。
    • container owner 是 AbilitySystem。
    • stacking group 是 container 内部结构，负责相同 stacking key 的叠层/优先级。
    • BuffContainer.OnTick 不自行决定 fixed frame / scaled frame；delta 来源必须由 AbilitySystem / TimeManager tick owner 传入。
    • CreateBuff 会先构建两份 AbilitySystem.AddBuffContext：
    • source context：写入 source owner info、buffData.id、AddBuffOption.skillCastInfo、partTransferInfo、isExtra，并在 stack 前触发 AbilitySystem.Event.OnBeforeOutputBuff，code 206。
    • owner context：写入 owner info 和同一份 add option 语义，并在 stack 前触发 AbilitySystem.Event.OnBeforeAddedBuff，code 205。
    • CreateBuff 按 BuffData.stackingKey / identifier 取得或创建 BuffStackingGroup。新建 group 时会先加入 m_stackingGroups，再调用 BuffStackingGroup.StackBuff(...)。
    • StackBuff 返回非空新实例时，CreateBuff 才把该 buff 的 ObjectPtr 追加到 m_buffs (DoubleBufferedBuffList.Add)。Refresh / Extend / Enhance / Modify / Unique reject 等不产生新实例的分支不会追加到 m_buffs。
    • StackBuff 返回新实例后，CreateBuff 会继续触发 owner 的 AbilitySystem.Event.OnAddedBuff，code 9，以及 source 的 AbilitySystem.Event.OnOutputBuff，code 102。随后调用 owner 侧 TryEnhancingKeywordBuff(onAddedBuffId)。
    • FinishBuffAction.ExecuteInternal 状态 已确认 与 FinishBuffAdvanced.ExecuteInternal 状态 已确认 在构建 FinishBuffOption 时使用 needLimitLayerCnt = !data.finishAll。因此 finishAll=true 不受 finishLayerCnt 限制；finishAll=false 才按 finishLayerCnt 做普通实例数量限制或 enhance 层数扣减。
    • FinishBuff(string, option) 的 needLimitLayerCnt=true 路径会先读取目标 BuffData.stackingSettings.stackingType。当 stacking type 是 Enhance(3)、EnhanceAndRefresh(8) 或 EnhanceAndOverwriteDuration(10) 时，遍历 m_buffs 中匹配 id/source 的实例并调用 DecreaseEnhanceCnt(option)，而不是直接 MarkFinish。
    • FinishBuff(string, option) 的普通 finish 路径遍历 m_buffs，跳过 invalid / already-finishing buff，匹配 id/source 后调用 MarkFinish(upload=true, opParams, option.reason, option.finishSource, option.finishCastInfo)。当 needLimitLayerCnt=true 时，最多处理 finishLayerCnt 个实例；当 needLimitLayerCnt=false 时，结束所有匹配实例。
    • DecreaseEnhanceCnt(option) 中如果 m_enhanceCnt <= option.finishLayerCnt，直接用 option.reason、option.finishSource 和 option.finishCastInfo 调 MarkFinish。否则 m_enhanceCnt -= finishLayerCnt，执行 Buff.Event.OnBuffEnhanceChanged(6)，重新加载 attribute modifier，刷新 stacking group，标记 owner attributes dirty，调用 BattleRecorder.RecordBuffEnhanceChange，再触发 ability events。
    • DecreaseEnhanceCnt(option) 的 ability event 侧效果：
    • reason == Early(2) 或 Ignite(1) 且 finishSource 有效时，触发 finishSource.OnConsumeBuff(208)。
    • reason == Absorbed(4) 且 finishSource 有效时，触发 owner OnBuffAbsorbed(210) 与 finishSource.OnAbsorbBuff(211)。
    • 最后 owner 触发 OnBuffEnhanceChanged(209)，context 中 changed layer 为负的 finishLayerCnt。
