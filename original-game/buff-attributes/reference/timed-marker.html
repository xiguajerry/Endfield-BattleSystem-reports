    TimedMarker / GlobalTimedMarker¶

    更新时间：2026-06-14

    字段¶

    AbilitySystem.m_timedMarkers¶

    字段 offset 说明
    AbilitySystem.m_timedMarkers 0x3D8 实体 timed marker 列表，元素为 TimedMarker。

    TimedMarker¶

    字段 offset 说明
    m_owner 0x10 ObjectPtr<AbilitySystem> owner。
    id 0x18 marker id。
    m_lifeTime 0x20 剩余秒数。
    m_duration 0x24 初始秒数。
    m_useTimeDilationDt 0x28 是否使用 allScaledDeltaTime。

    BattleManager.m_globalTimedMarkers¶

    字段 offset 说明
    BattleManager.m_globalTimedMarkers 0x260 global timed marker 列表。

    GlobalTimedMarker¶

    字段 offset 说明
    buffId 0x10 marker id。
    charServerId 0x18 character server id key。
    m_lifeTime 0x20 剩余秒数。

    方法¶

    方法 状态 状态 结论
    AbilitySystem.AddTimedMarker(string,float,bool) 已确认 已确认 分配新 TimedMarker，构造后追加到 m_timedMarkers；不按 id 去重。
    TimedMarker.ctor(AbilitySystem,string,float,bool) 已确认 已确认 保存 owner/id/lifeTime/duration/useTimeDilationDt。
    AbilitySystem._UpdateTimedMarkers(float,float) 已确认 已确认 倒序遍历 m_timedMarkers，TimedMarker.Update 返回 true 时 RemoveAt(index)。
    TimedMarker.Update(float,float) 已确认 已确认 useTimeDilationDt=false 用 deltaTime，否则用 allScaledDeltaTime；剩余秒数减去 delta，lifeTime < -1e-5 过期。
    AbilitySystem.RemoveTimedMarker(TimedMarker) 已确认 已确认 对 marker 调 MarkInvalid()，再从 m_timedMarkers 移除 exact object。
    TimedMarker.MarkInvalid() 已确认 已确认 m_lifeTime = -1.0f。
    TimedMarker.Remove() 已确认 已确认 若 marker 有效且 owner valid，锁 owner 并调用 AbilitySystem.RemoveTimedMarker(this)；否则返回 false。
    AbilitySystem.RemoveAllTimedMarkerById(string) 已确认 已确认 前向遍历，id 匹配时 MarkInvalid() 并 RemoveAt(index)；remove 后仍执行循环自增。
    AbilitySystem.HasMarker(string) 已确认 已确认 线性扫描，id 匹配且 lifeTime >= -1e-5 返回 true。
    AbilitySystem.GetTimedMarkerById(string) 已确认 已确认 返回第一个 id 匹配且有效的 marker。
    BattleManager.AddGlobalTimedMarker(string,ulong,float) 已确认 已确认 倒序查找同 (id,charServerId)；命中则 SetLifeTime(lifeTime)，否则分配新 GlobalTimedMarker 并追加。
    AddGlobalCDTimer.ExecuteInternal(TargetHandleView) 已确认 已确认 通过 AbilityActionUtils.GetFirstTarget 解析 Data.target，读取目标 entity serverId，从 action blackboard 解析 cdTime 秒数，调用 BattleManager.AddGlobalTimedMarker(buffId, serverId, cdTime)；target 为空时 no-op 返回 true。

    GlobalTimedMarker.SetLifeTime(float) 已确认 已确认 直接写 m_lifeTime。
    GlobalTimedMarker.Update(float) 已确认 已确认 m_lifeTime -= deltaTime，lifeTime < -1e-5 过期。
    BattleManager.ClearGlobalTimedMarker() 已确认 已确认 遍历 recycle 所有 global marker，然后 clear list。

    调用入口¶

    • 实体 marker：CreateTimedMarker.ExecuteInternal 或 gameplay code 调 AbilitySystem.AddTimedMarker。
    • 实体 marker 条件：CheckTimedMarkerCondition 走目标 AbilitySystem.HasMarker。
    • global marker：battle/global flow 调 BattleManager.AddGlobalTimedMarker / HasGlobalTimedMarker / _TickGlobalTimedMarker。
    • action global-CD：AddGlobalCDTimer / CheckGlobalCDTimerAction 也是 global marker path，key 为 (buffId,target.serverId)，不应实现成目标 AbilitySystem 私有帧倒计时。
    • action lifetime：CreateTimedMarker.OnEnd 根据 autoFinishByAction 调每个创建 marker 的 TimedMarker.Remove()。

    生命周期¶

    1. CreateTimedMarker.ExecuteInternal 解析 target 后创建实体 marker。
    2. AbilitySystem.AddTimedMarker 总是追加新 list entry，允许同 id 多个 marker 同时存在。
    3. AbilitySystem._UpdateTimedMarkers 由 owner tick 推进，倒序移除过期 marker。
    4. 若 action 配置 autoFinishByAction=true，action end 时调用 exact marker object 的 Remove()。
    5. global marker 不归实体 AbilitySystem，而是归 BattleManager.m_globalTimedMarkers。
    6. BattleManager.AddGlobalTimedMarker 对同 (id,charServerId) 覆盖 lifetime，不追加重复项。
    7. _TickGlobalTimedMarker 只使用 BattleManager tick delta，不读取目标实体 time scale。

    已确认行为¶

    • 实体 marker 和 global marker 是两套 owner，不应使用同一个 world-level store。
    • 实体 marker 同 id 不去重；auto-finish 必须持有 exact marker instance。
    • useTimeDilationDt 只存在于实体 TimedMarker，决定 TimedMarker.Update 用哪个 delta。
    • marker 有效性阈值是 lifeTime >= -1e-5。
    • global marker 添加同 key 会覆盖 lifetime。
    • AddGlobalCDTimer.cdTime 是秒数，进入 BattleManager.AddGlobalTimedMarker；过期阈值继承 GlobalTimedMarker.Update 的 lifeTime < -1e-5。
