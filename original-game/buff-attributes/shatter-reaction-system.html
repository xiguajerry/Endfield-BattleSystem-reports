    碎冰反应系统分析¶

    最后更新: 2026-05-30

    结论¶

    碎冰不是 DamageAction 内部写死的特殊分支，而是一条数据驱动的 Buff 链:

    flowchart TD
    Trigger["破防或物理异常 Buff OnBuffStart"]
    Frozen{"目标有 Frozen tag"}
    Skip["不触发碎冰链"]
    Break["CreateBuff(buff_physical_handle_cryst_break)"]
    Count["读取冻结 Buff 黑板 count"]
    Scale["ReadSkillSettingData<br/>按 count 写入 atk_scale"]
    Finish["FinishBuffAdvanced<br/>移除 Frozen tag Buff"]
    DamageBuff["CreateBuff(buff_common_cryst_triggered_physical_break)"]
    Damage["Physical Damage<br/>DamageDecorateMask.Shatter"]
    Trigger --> Frozen
    Frozen -->|否| Skip
    Frozen -->|是| Break --> Count --> Scale --> Finish --> DamageBuff --> Damage

    因此，游戏里的触发条件可以概括为:

    • 目标身上有 Skill/Character/Common/SpellStatus/Frozen tag。
    • 目标受到破防 buff_physical_no_guard 或物理异常 buff_physical_airborne / buff_physical_knockdown / buff_physical_crushed / buff_physical_do_fracture。
    • 物理侧 Buff 的 OnBuffStart 会创建 buff_physical_handle_cryst_break，由它统一处理碎冰。

    关键 Buff¶

    Buff 作用
    buff_common_cryst_cryst_frozen_triggered 寒冷 + 寒冷反应入口，读取冰冻持续时间并创建实际冻结状态
    buff_common_cryst_cryst_frozen_triggered_do 实际冻结状态 Buff，带 Frozen tag 和 count
    buff_physical_handle_cryst_break 碎冰调度器，读取冻结层数、计算倍率、清除冻结、创建伤害 Buff
    buff_common_cryst_triggered_physical_break 实际碎冰伤害 Buff，造成物理 Shatter 伤害
    buff_physical_no_guard 破防状态，OnBuffStart 会尝试碎冰
    buff_physical_airborne 击飞物理异常，OnBuffStart 会尝试碎冰
    buff_physical_knockdown 倒地物理异常，OnBuffStart 会尝试碎冰
    buff_physical_crushed 猛击物理异常，OnBuffStart 会尝试碎冰
    buff_physical_do_fracture 碎甲物理异常执行 Buff，OnBuffStart 会尝试碎冰

    冻结状态如何产生¶

    角色侧寒冷附着的同属性反应会进入冻结链:

    buff_common_cryst_cryst_frozen_triggered
 OnBuffStart:
 ReadSkillSettingData("冰冻持续时间", column=count) -> duration
 CreateBuff(buff_common_cryst_cryst_frozen_triggered_do)

    buff_common_cryst_cryst_frozen_triggered_do 是真正可被碎冰识别的状态 Buff。它的关键点是:

    • applyTags 包含 Skill/Character/Common/SpellStatus/Frozen。
    • 黑板包含 count，默认 1。
    • duration 由上游 冰冻持续时间 写入。

    • stackingKey 是 cryst_triggered，最大 1 层。

    • AKEDataMining/data/Json/BuffData/buff_common_cryst_cryst_frozen_triggered.json

    • AKEDataMining/data/Json/BuffData/buff_common_cryst_cryst_frozen_triggered_do.json

    注意: buff_common_enemy_spell_cryst_triggered_frozen 是敌人对角色施加的角色侧冻结异常。它会创建 buff_common_enemy_spell_status_frozen / buff_common_enemy_spell_status_do_frozen，不是角色输出到敌人时的碎冰主链。

    破防和物理异常如何接入碎冰¶

    以下 Buff 的 OnBuffStart 都会创建 buff_physical_handle_cryst_break:

    触发 Buff 含义
    buff_physical_no_guard 破防 / no guard
    buff_physical_airborne 击飞
    buff_physical_knockdown 倒地
    buff_physical_crushed 猛击
    buff_physical_do_fracture 碎甲执行

    这说明“对冻结目标施加破防或物理异常触发碎冰”在配置上是通过物理侧 Buff 的开始事件实现的，而不是物理异常 helper 直接硬编码碎冰。

    • AKEDataMining/data/Json/BuffData/buff_physical_no_guard.json
    • AKEDataMining/data/Json/BuffData/buff_physical_airborne.json
    • AKEDataMining/data/Json/BuffData/buff_physical_knockdown.json
    • AKEDataMining/data/Json/BuffData/buff_physical_crushed.json
    • AKEDataMining/data/Json/BuffData/buff_physical_do_fracture.json

    buff_physical_handle_cryst_break 的执行流程¶

    buff_physical_handle_cryst_break 是碎冰反应的核心调度 Buff。

    读取冻结层数¶

    它先使用 GetTargetBuffBBAdvanced:

    "buffSettings": {
 "checkType": "Tag",
 "tagQuery": {
 "queryType": "HasAny",
 "tags": [
 "Skill/Character/Common/SpellStatus/Frozen"
 ]
 }
},
"desiredKey": "count",
"blackboardKey": "count"

    含义:

    • 在目标身上查找带 Frozen tag 的 Buff。
    • 从该 Buff 的黑板读取 count。
    • 写入当前 handle buff 的黑板 count。

    读取碎冰倍率¶

    随后执行 ReadSkillSettingData:

    {
 "dataKey": "碎冰倍率",
 "column": {
 "useBlackboardKey": true,
 "blackboardKey": "count"
 },
 "storeKey": "atk_scale"
}

    碎冰倍率 当前表值为:

    count 基础倍率
    1 2.4
    2 3.6
    3 4.8
    4 6.0

    该数据项带 enhanceFormulaKey = Damage，会受到 PhysicalAndSpellInflictionEnhance 影响。

    消耗冻结状态¶

    读取倍率后，handle buff 使用 FinishBuffAdvanced 清理目标身上的 Frozen tag Buff:

    "buffSettings": {
 "checkType": "Tag",
 "tagQuery": {
 "queryType": "HasAny",
 "tags": [
 "Skill/Character/Common/SpellStatus/Frozen"
 ]
 }
},
"finishAll": true

    创建实际伤害 Buff¶

    最后创建:

    buff_common_cryst_triggered_physical_break

    并把 handle buff 中算出的 atk_scale 传过去。

    碎冰伤害如何计算¶

    实际伤害在 buff_common_cryst_triggered_physical_break 的 OnBuffStart 中打出:

    "damageType": "Physical",
"damageDecorateMask": 134217728,
"atkCalculation": {
 "$type": "Beyond.Gameplay.Core.AtkScaleCalculation, Gameplay.Beyond",
 "atkScale": {
 "useBlackboardKey": true,
 "blackboardKey": "atk_scale"
 }
},

    134217728 对应:

    因此碎冰是:

    • 物理伤害。
    • 使用 Shatter 伤害标签。
    • atk_scale 来自 碎冰倍率。
    • 视觉文本使用 ui_bat_elemental_cryst_break。

    与其他寒冷反应的区别¶

    寒冷和其他元素组合也会读取 碎冰倍率，但它们不是“冻结被物理打碎”这条链:

    Buff 场景
    buff_common_try_cryst_fire_triggered 寒冷 + 灼热
    buff_common_try_cryst_pulse_triggered 寒冷 + 电磁
    buff_common_try_cryst_natural_triggered 寒冷 + 自然
    buff_physical_handle_cryst_break 冻结 + 破防/物理异常

    真正符合“冻结状态下施加破防或物理异常触发碎冰”的是 buff_physical_handle_cryst_break -> buff_common_cryst_triggered_physical_break。

    最小测试场景建议¶

    可用以下场景验证:

    1. 给敌人挂 buff_common_cryst_cryst_frozen_triggered_do，设置 count = 3。
2. 对敌人施加 buff_physical_no_guard 或任一物理异常 Buff。
3. 期望:
 - buff_physical_handle_cryst_break 被创建。
 - Frozen tag Buff 被 FinishBuffAdvanced 移除。
 - buff_common_cryst_triggered_physical_break 被创建。
 - battle log 中出现 Physical + Shatter 伤害。
 - atk_scale 使用 count=3 对应的碎冰倍率计算。

    如果要模拟“先寒冷附着堆到冻结，再打碎”，则还需要完整跑 buff_common_cryst_cryst_frozen_triggered 的冰冻持续时间读取和 buff_common_cryst_cryst_frozen_triggered_do 创建链路。
