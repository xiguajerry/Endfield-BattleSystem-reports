    数据结构¶

    AttributesData (属性数据容器)¶

    • 使用 get_Item(AttributeType) 按属性类型读写 double 值
    • 内部维护 fieldMetas[] 记录每个属性的 min/max 约束
    • 支持 BinaryClone 深拷贝和 CreateFrom 复制构造

    EnemyData (敌人配置数据)¶

    • 从 SparkMapTable<String, EnemyData> 按 enemyId 查表获取
    • 包含 attrModifiers: Nullable<BeanArray<AttributeModifierData>>
    • AttributeModifierData 包含 modifierType 和 formula params

    GetCharacterAttributeDataFromTable¶

    流程说明¶

    步骤 说明
    1 懒初始化 MethodInfo: SafeCount<T>, get_Item, BinaryClone<T> (byte_18D8E8484 门控)
    3 SafeCount(keyFrames) → 空列表直接返回 false
    4 线性遍历，比较每个 keyframe 的 level 和 breakStage
    5 匹配成功 → BinaryClone(attributesData) 深拷贝写入 outResult
    6 写屏障通知 GC (sub_1800054D0)
    7 返回 true (找到) 或 false (未找到)

    关键点¶

    • O(n) 线性搜索，未用二分查找 — 说明角色属性表条目数通常不多
    • 深拷贝 通过 BinaryClone，避免调用方修改影响原始数据

    breakStage 数据来源¶

    本地数据 AKEDataMining/data/TableCfg/CharBreakStageTable.json 记录角色等级到突破阶段的映射：

    字段 语义
    breakStage 突破阶段
    minCharLevel 该阶段最小角色等级
    maxCharLevel 该阶段最大角色等级
    normalAttackSkillLevel / normalSkillLevel / comboSkillLevel / ultimateSkillLevel 该阶段默认技能等级上限/展示等级数据

    GetEnemyBasicAttributeData¶

    流程说明¶

    步骤 说明
    1 懒初始化 MethodInfo: get_Count, get_Item (byte_18D909029 门控)
    3 空列表检查 → 返回 false
    4 读取首帧等级 firstLevel 和末帧等级 lastLevel (通过 get_Item(AttributeType.Level))
    5 边界外推：低于最低→取首帧 / 高于最高→取末帧
    6 二分查找定位目标等级所在区间
    7 精确匹配 → CreateFrom 浅拷贝
    8 区间匹配 → _Lerp 线性插值两侧关键帧

    关键点¶

    • 线性插值 — 当等级落在两个关键帧之间时，使用 _Lerp 按比例混合
    • 边界钳制 — 超出表范围时钳制到最近端点，不是报错
    • CreateFrom vs BinaryClone — 此处用 CreateFrom (轻量复制)，角色函数用 BinaryClone (完整深拷贝)

    GetFinalEnemyAttributeData¶

    流程说明¶

    步骤 状态 说明
    1 已确认-已确认 懒初始化 8 个 TypeInfo/MethodInfo 指针 (byte_18D909028 门控)
    3 已确认-已确认 获取 attrModifiers: Nullable<BeanArray<AttributeModifierData>>，检查 HasValue
    4 已确认-已确认 分配 8 个 double 数组 (各 94 元素)，初始化加算=0、乘算=1.0
    5 已确认-已确认 调用 _ApplyAttributeModifierFromEnemy 填充修正器数组，返回 64 位 bitmask
    6 已确认-已确认 遍历 94 个属性，逐属性检查 bitmask
    7 已确认-已确认 对每个需修正的属性，读取 rawValue + 8 个修正值 + min/max，调用 _CalculateFinalAttribute
    8 已确认-已确认 WriteSingleAttribute(attrType, finalValue) 写回计算结果

    修饰器数组对应关系¶

    数组 寄存器 含义 初始值 是否被 Apply 修改 对应 modifierType
    addition rdi→r12 主加算 0.0 是 (ADD) type 0
    multiplier rsi→r15 主乘算 1.0 是 (ADD) type 1
    finalAddition r14→r14 主最终加算 0.0 是 (ADD) type 3
    finalMultiplier r15→arg_20 主最终乘算 1.0 是 (MUL) type 4
    baseAddition r12 基础加算 0.0 否 (恒为 0) —
    baseMultiplier r13 基础乘算 1.0 否 (恒为 1.0) —
    baseFinalAddition arg_18 基础最终加算 0.0 否 (恒为 0) —
    baseFinalMultiplier var_190 基础最终乘算 1.0 否 (恒为 1.0) —

    关键点¶

    • 两阶段计算：先通过 GetEnemyBasicAttributeData 获取基础值，再通过本函数叠加修改器得到最终值
    • 敌人只用主修正阶段：8 个数组中的 4 个基础阶段数组保持恒等值 (0/1.0)，敌人在 base 阶段只做 Clamp(raw, min, max)
    • 仅有 4 个数组被 _ApplyAttributeModifierFromEnemy 修改：主加算、主乘算、主最终加算、主最终乘算
    • bitmask 优化：用 64 位 bitmask 标记哪些属性被修改，未修改的属性跳过计算
    • 94 个属性：Core.Attributes 枚举共有 94 种属性类型 (loop 上限 0x5E)
    • 静默生效：无修改器时直接返回 true，不修改 rawData

    属性计算公式：汇编级精确推导¶

    辅助函数: Clamp = Beyond::MathUtils::Clamp<double> (sub_182778EC0), Max = Beyond::MathUtils::Max<double> (sub_182A53D10)

    _CalculateBaseAttribute (7 参数)¶

    参数: rawValue, baseAddition, baseMultiplier, baseFinalAddition, baseFinalMultiplier, min, max
寄存器: xmm6=rawValue, xmm9=baseAddition, xmm8=baseMultiplier, xmm7=baseFinalAddition
栈参: arg_20=baseFinalMultiplier, arg_28=min, arg_30=max

    逐指令追踪¶

    状态 指令 / 操作 公式 / 说明
    已确认 addsd xmm6, xmm9 T0 = rawValue + baseAddition
    已确认 call Clamp(T0, min, max) T1 = Clamp(T0, min, max)
    已确认 call Max(0, baseMultiplier) M1 = Max(0, baseMultiplier)
    已确认 mulsd xmm0, xmm6 T2 = M1 × T1
    已确认 addsd xmm0, xmm7 T3 = T2 + baseFinalAddition
    已确认 mulsd xmm0, [arg_20] T4 = T3 × baseFinalMultiplier
    已确认 call Clamp(T4, min, max) result = Clamp(T4, min, max)

    精确公式¶

    _CalculateBaseAttribute(rawValue, baseAddition, baseMultiplier, baseFinalAddition, baseFinalMultiplier, min, max)
 = Clamp(
 ( Max(0, baseMultiplier) × Clamp(rawValue + baseAddition, min, max) + baseFinalAddition )
 × baseFinalMultiplier,
 min, max
 )

    _CalculateFinalAttribute (11 参数)¶

    参数: rawValue, addition, multiplier, finalAddition, finalMultiplier,
 baseAddition, baseMultiplier, baseFinalAddition, baseFinalMultiplier, min, max
寄存器: xmm6=rawValue, xmm11=addition, xmm10=multiplier, xmm9=finalAddition
栈参: arg_20=finalMultiplier, arg_28=baseAddition, arg_30=baseMultiplier,
 arg_38=baseFinalAddition, arg_40=baseFinalMultiplier, arg_48=min, arg_50=max

    逐指令追踪¶

    阶段 A — 基础加算 & 第1次 Clamp:

    状态 指令 / 操作 公式 / 说明
    已确认 addsd xmm6, [arg_28] T0 = rawValue + baseAddition
    已确认 call Clamp(T0, min, max) T1 = Clamp(T0, min, max)

    阶段 B — 基础乘算 & 第2次 Clamp:

    状态 指令 / 操作 公式 / 说明
    已确认 call Max(0, baseMultiplier) M1 = Max(0, baseMultiplier)
    已确认 mulsd xmm0, xmm6 T2 = M1 × T1
    已确认 addsd xmm0, [arg_38] T3 = T2 + baseFinalAddition
    已确认 mulsd xmm0, [arg_40] T4 = T3 × baseFinalMultiplier
    已确认 call Clamp(T4, min, max) B = Clamp(T4, min, max)

    至此 B 的结果与 _CalculateBaseAttribute 完全一致。 阶段 C — 主修正 & 第3次 Clamp:

    状态 指令 / 操作 公式 / 说明
    已确认 call Max(0, multiplier) M2 = Max(0, multiplier)
    已确认 addsd xmm6, xmm11 T5 = B + addition
    已确认 mulsd xmm6, xmm0 T6 = T5 × M2
    已确认 addsd xmm6, xmm9 T7 = T6 + finalAddition
    已确认 mulsd xmm6, [arg_20] T8 = T7 × finalMultiplier
    已确认 call Clamp(T8, min, max) result = Clamp(T8, min, max)

    精确公式¶

    _CalculateFinalAttribute(rawValue, addition, multiplier, finalAddition, finalMultiplier,
 baseAddition, baseMultiplier, baseFinalAddition, baseFinalMultiplier,
 min, max)
 = Clamp(
 ( (
 Clamp( // ← 基础阶段 Clamp
 Max(0, baseMultiplier)
 × Clamp(rawValue + baseAddition, min, max) // ← 第1次 Clamp
 + baseFinalAddition
 ) × baseFinalMultiplier,
 min, max // ← 第2次 Clamp
 ) + addition
 ) × Max(0, multiplier)
 + finalAddition
 ) × finalMultiplier,
 min, max // ← 第3次 Clamp
 )

    两函数关系¶

    _CalculateBaseAttribute = _CalculateFinalAttribute 的 "基础阶段" (阶段 A + B)
即: _CalculateFinalAttribute 在 _CalculateBaseAttribute 的结果 B 之上，
 再叠加 (addition, multiplier, finalAddition, finalMultiplier) 四个主修正器

    等价表示：

    B = _CalculateBaseAttribute(rawValue, baseAddition, baseMultiplier, baseFinalAddition, baseFinalMultiplier, min, max)
_CalculateFinalAttribute = Clamp(
 ((B + addition) × Max(0, multiplier) + finalAddition) × finalMultiplier,
 min, max
)

    baseAddition 基础 固定数值加成 角色/敌人自身属性成长
    baseMultiplier 基础 百分比增减 天赋、潜能加成
    baseFinalAddition 基础 最终固定追加 装备固定值
    baseFinalMultiplier 基础 最终倍率 全局属性倍率
    addition 主修正 固定数值 Buff Buff "+100 攻击力"
    multiplier 主修正 百分比 Buff Buff "攻击力 +30%"
    finalAddition 主修正 最终固定追加 最终伤害附加
    finalMultiplier 主修正 最终倍率 伤害倍率、暴击倍率

    设计要点¶

    1. Max(0, ×) — 所有乘算值取 Max(0, value)，防止负乘数导致属性值反向
    2. 三层 Clamp — 基础加法后、基础乘算后、最终结果后各钳制一次，确保中间值也不会溢出
    3. base 组先于 main 组 — 基础属性修正（来自角色/敌人自身）先计算，Buff/Debuff 修正叠加在结果上
    4. 加法先于乘法 — 每个子层内部都是 (value + add) × mul 结构，符合 RPG 战斗公式惯例
    5. 最终层结构 — finalAddition 在乘 finalMultiplier 之前加，意味着它是一个不受倍率影响的固定偏移

    _ApplyAttributeModifierFromEnemy — 汇编级精确推导¶

    函数签名¶

    注意: GetFinalEnemyAttributeData 分配了 8 个数组，但只将前 4 个 (主修正阶段) 传入此函数。 后 4 个 (基础阶段) 保持初始化值不变 (0 和 1.0)，使 base 阶段成为恒等变换。

    AttributeModifierData 结构体布局¶

    AttributeModifierData:
 +0x00: attributeIndex (int32) — 目标属性枚举值
 +0x04: (padding)
 +0x08: formulaValue (double) — 修正数值
 +0x10: modifierType (int32) — 修正类型 (0~4+)

    该结构体通过 sub_182328590(ptr, offset) 读 int32 字段，通过 sub_182C5AA60(ptr, offset) 读 double 字段。

    modifierType 枚举与行为¶

    枚举值 目标数组 操作 语义
    0 additions (r12) ADD 加算累加: arr[i] += formulaValue
    1 multipliers (r15) ADD 乘算累加: arr[i] += formulaValue (多个 buff 的百分比相加)
    2 — 仅设 bit 标记该属性被引用，不修改任何数组值
    3 finalAdditions (r14) ADD 最终加算累加: arr[i] += formulaValue
    4 finalMultipliers (arg_20) MUL 最终乘算累乘: arr[i] *= formulaValue (多个 buff 的倍率相乘)
    其他 — 仅设 bit 未知类型仅标记，不修改数组

    关键设计：加算堆叠 vs 累乘堆叠¶

    • 类型 0, 1, 3 使用 ADD — 多个同类型修正器的效果相加。例如两个 "+30% 攻击" buff 叠加 = 1.0 + 0.3 + 0.3 = 1.6
    • 类型 4 使用 MUL — 最终倍率类修正器使用累乘。例如两个 "×1.5 伤害" = 1.5 × 1.5 = 2.25

    bitmask 构建¶

    所有枚举分支 (包括未知类型) 最终都执行:

    状态 指令 / 操作 公式 / 说明
    已确认 and eax, 3Fh attrIndex & 0x3F (取低 6 位 = 属性索引 mod 64)
    已确认 bts rbx, rax rbx \|= (1 << (attrIndex & 0x3F)) (设置对应 bit)

    最终 rbx 作为 64 位返回值，每位表示对应属性是否被任何修正器引用过。

    完整流程¶

    _ApplyAttributeModifierFromEnemy(modifiers, adds[], muls[], finalAdds[], finalMuls[]):
 rbx = 0 // bitmask 初始化为 0
 enumerator = modifiers.GetEnumerator()
 while enumerator.MoveNext():
 modifier = enumerator.Current
 type = modifier.modifierType // offset 0x10
 attrIndex = modifier.attributeIndex // offset 0x00
 value = modifier.formulaValue // offset 0x08
 switch type:
 case 0: adds[attrIndex] += value // ADD
 case 1: muls[attrIndex] += value // ADD (加算堆叠)
 case 2: break // 仅标记
 case 3: finalAdds[attrIndex] += value // ADD
 case 4: finalMuls[attrIndex] *= value // MUL (累乘堆叠)
 default: break // 未知类型仅标记
 rbx |= (1 << (attrIndex & 0x3F)) // 设置 bitmask
 return rbx // 64 位标记位掩码

    各数组最终值语义¶

    调用此函数后，四个数组从初始值变为：

    数组 初始值 叠加后
    additions[i] 0.0 Σ modifier_j.formulaValue (所有类型0修正器的值之和)
    multipliers[i] 1.0 1.0 + Σ modifier_j.formulaValue (所有类型1修正器的值之和 + 底数)
    finalAdditions[i] 0.0 Σ modifier_j.formulaValue (所有类型3修正器的值之和)
    finalMultipliers[i] 1.0 Π modifier_j.formulaValue (所有类型4修正器的值之积)

    代入 _CalculateFinalAttribute 公式：

    B = Clamp(rawValue, min, max) // base阶段均为恒等变换 (baseAdd=0, baseMul=1.0, baseFinalAdd=0, baseFinalMul=1.0)
final = Clamp(
 ((B + Σadds) × Max(0, 1.0 + Σmuls) + ΣfinalAdds) × ΠfinalMuls,
 min, max
)

    与 GetFinalEnemyAttributeData 的连接¶

    在 GetFinalEnemyAttributeData 中：

    • 数组 1-4 (adds, muls, finalAdds, finalMuls) — 被 _ApplyAttributeModifierFromEnemy 填充
    • 数组 5-8 (baseAdd, baseMul, baseFinalAdd, baseFinalMul) — 保持初始化值，所以 base 阶段是恒等变换

    Attributes.AttributeModifierLoader — 运行时 modifier 聚合¶

    初始化¶

    AttributeModifierLoader 持有 8 个数组：

    • attributeAdditions
    • attributeMultipliers
    • attributeFinalAdditions
    • attributeFinalScalars
    • attributeBaseAdditions
    • attributeBaseMultipliers
    • attributeBaseFinalAdditions
    • attributeBaseFinalScalars

    _InitAttributeModifierValue 会将加算数组清零，并将 scalar/multiplier 类数组初始化为恒等值。运行时聚合不是直接改最终属性值，而是先把 modifier 写入这些数组，再由 Attributes._CalculateFinalAttributeValue 读取。
