    BuffContainer¶

    概述¶

    BuffContainer 是挂载在实体 AbilitySystem 上的 buff 管理器，负责该实体上所有 buff 的创建、移除、查询和 tick。

    字段¶

    Offset 字段 类型 说明
    0x10 m_buffs DoubleBufferedBuffList 双缓冲 buff 列表
    0x18 m_stackingGroups Dictionary<String, BuffStackingGroup> buffId 到堆叠组
    0x20 m_buffChangeCharHpBarVfx List<ObjectPtr<Buff>> HP 条 VFX 变化的 buff 列表
    0x28 m_owner AbilitySystem* 所属实体的 AbilitySystem

    CreateBuff 流程概要¶

    CreateBuff 是最核心的方法 (0x2cdc 字节), 大致流程:

    CreateBuff(buffSource, buffData, addBuffOption, oriBuffInst):
 2. 校验 buffSource entity 和 team 有效性
 3. 获取 BuffStackingGroup (按 buffData.buffId)
 4. 处理堆叠逻辑:
 - 检查堆叠限制 (maxStackCount)
 - RemoveStackEffects 如需要
 - StackBuff / RefreshPriority
 5. 使用 NewBuff 从对象池获取 Buff 实例 / 或复用现有实例
 6. 初始化 Buff 实例:
 - SetBuffParent
 - 加载 Actions / Modifiers / ShieldInsts
 - 设置 priority, filerTypeMask, attributeMask
 7. 添加到 m_buffs (DoubleBufferedBuffList)
 8. 触发 Event::OnAdd 事件
 9. 上传到服务器 (如需要)
 10. 返回 ObjectPtr<Buff>

    OnTick 流程¶

    void OnTick(float deltaTime, float allScaledDeltaTime):
 if (m_buffs == null): throw // [this+0x10]
 enumerator = m_buffs.GetEnumerator()
 while (enumerator.MoveNext()):
 current = enumerator.get_Current() // ObjectPtr<Buff>
 if (!current.isValid()): continue
 buff = current.Lock()
 buff.OnTick(deltaTime, allScaledDeltaTime)
 enumerator.Dispose()

    关键设计:

    • 使用 DoubleBufferedBuffList 的版本号机制保证 tick 期间安全添加/移除 buff
    • 每个 buff 接收两个 delta：deltaTime(实际) 和 allScaledDeltaTime(缩放后)

    RecycleBuff 流程¶

    void RecycleBuff():
 // 阶段 1: 清理队长/队员变更导致的过期 buff
 foreach slot in squadManager.currentSquad.slots:

 // 阶段 2: 遍历 m_buffs, 回收已标记结束的 buff

    EmptyBuffContainer¶

    EmptyBuffContainer 是 BuffContainer 的子类，所有操作都是 no-op。用于没有 buff 系统的实体 (如建筑物、子弹等)。

    Buff¶

    概述¶

    Buff 是单个 buff 实例，包含来源信息、持续时间、行为(action)列表、修饰器(modifier)列表等。使用对象池 (BuffPool) 管理生命周期。

    BuffStackingGroup¶

    概述¶

    管理同类 buff (相同 stackingKey) 的堆叠行为。当添加一个已有 buffId 的新 buff 时，由 StackingGroup 决定是创建新实例、刷新旧实例、还是拒绝。

    结构体字段布局¶

    BuffStackingGroup:
 +0x18: int16 stackingType (switch key: 0~11)
 +0x1C: int32 maxStackCount (堆叠上限)
 +0x20: int32 currentCount (当前未结束 buff 数)
 +0x38: List<EffectInstance>* effectList (_ClearStackEffects 发现)
 +0x40: AbilitySystem* buffSource (buff 施放者)
 +0x48: AbilitySystem* containerOwner (容器所属实体)
 +0x50: PriorityQueue<Buff>* priorityQueue (内部由 UniqueList<Buff> 支持)

    Case 类型 行为 输出
    0 Unlimited 总是 Allocate + Add + enabled newBuff
    1 HighPriority Allocate + Add + 只启用最高优先级 1 个 + ServerOp newBuff
    2 Stack 上限检查 → 溢出则 MarkFinish 最后一个未完成实例 newBuff
    3 Enhance 找到现有则 _Enhance/加层，否则 Allocate existingBuff 或 newBuff
    4 Refresh Allocate 用于读取 duration → _SetBuffDurationRefresh existingBuff
    5 Extend Allocate 用于读取 duration → 累加 remaining existingBuff
    6 Modify 复用旧实例并修改参数 existingBuff
    7 Unique 已有实例则拒绝新增，否则 Allocate null 或 newBuff
    8 EnhanceAndRefresh Enhance + Refresh existingBuff 或 newBuff
    9 OverwriteDuration Allocate 用于读取 duration → _SetBuffDurationOverwrite existingBuff
    10 EnhanceAndOverwriteDuration Enhance + OverwriteDuration existingBuff 或 newBuff
    11 HighPriorityWithMaxStack Allocate + Add + 启用最高优先级 maxStackCount 个 + ServerOp newBuff

    Case 4/5 中 _AllocateBuff 创建的新 buff 仅用于提取 duration 数据，随后被丢弃。 Case 1/11 中超出启用数量的 buff 不会被移除，只会 disabled；disabled buff 不应参与 timeline、trigger、modifier、ability event。

    三种持续时间策略¶

    Overwrite (_SetBuffDurationOverwrite, 0x2b):

    target.lifeType = source.lifeType // [target+0x24] = [source+0x24]
target._SetLifeTime(source.totalLifeTime) // xmm1 = [source+0x20]

    Refresh (_SetBuffDurationRefresh, 0x57):

    if target.lifeType == 1: return // infinite, skip
if source.lifeType == 1: [special]
remaining = target.remaining - epsilon
if source.totalLifeTime >= remaining:
 target._SetLifeTime(source.totalLifeTime)
else:
 target._SetLifeTime(target.remaining) // keep old

    Extend (在 Case 5 中内联实现):

    target._SetLifeTime(target.remaining + source.totalLifeTime)

    GlobalBuffStackingGroup¶

    方法 状态 说明
    Ctor 已确认 从 GlobalBuffData 构造
    StackGlobalBuff 已确认 叠加全局 buff (类似 StackBuff)
    RefreshStackingGroup 已确认 刷新堆叠组状态
    OnTick 已确认 Tick 更新 (0x13d8, 大函数)
    _AllocateGlobalBuff 已确认 分配全局 buff 实例

    与普通 BuffStackingGroup 的关键区别：操作对象是 GlobalBuff；不直接创建 Effect（效果通过子 Buff 间接实现）；最终通过 set_isEnabled(bool) 控制启用/禁用。

    m_buffListMap¶

    GlobalBuff 维护 Dictionary<ulong, List<ObjectPtr<Buff>>> 以 serverBuffId 为键追踪所有创建的子 Buff:

    • 清理时: 按 serverId 批量移除
    • 叠加时: 判断是否需要先移除旧 buff
    • 队伍变化时: 知道哪些 buff 需要重新应用到新成员

    BuffInput:
 +0x00: String buffId // 要添加的普通 Buff 配置 ID
 +0x08: ulong serverBuffId // 服务器端唯一 ID (用于追踪)
 +0x10: AssignPair assignPair // Blackboard 数据赋值对

    属性 Buff GlobalBuff
    数据类 BuffData GlobalBuffData

    关键设计模式¶

    1. 双缓冲列表 (DoubleBufferedBuffList) — 允许在 tick 遍历过程中安全地添加/移除 buff
    2. 对象池 (NewBuff / RecycleBuff) — Buff 实例从 BuffPool 分配和回收，减少 GC 压力
    3. 堆叠组 (StackingGroup) — 按 stackingKey 聚合，统一管理同类型 buff 的叠加规则
    4. 修饰器独立加载 — 不同 modifier 类型 (damage/heal/poise/global) 由独立方法加载
    5. Action 驱动 — buff 效果通过 Event→Action 映射触发，支持高度可配置的行为
    6. 双层 Buff 架构 — GlobalBuff (元 buff) → 批量创建普通 Buff → 实际效果
    7. 全量刷新 — _RefreshStackEffects 采用"全部清除→全部重建"模式保证一致性
    8. 精密周期计时器 — PrecisePeriodicTimer 批量触发保证帧率波动下行为一致
    9. Blackboard 参数系统 — GlobalBuff 通过 Blackboard 动态传递参数 vs Buff 直接读配置字段
