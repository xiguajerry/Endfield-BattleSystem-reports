    DataManager 配置读取路径分析¶

    分析对象: Beyond.Gameplay.DataManager

    加载入口¶

    DataManager 的配置加载分三段:

    DataManager::.ctor
 创建 loader、字典和懒加载 KVTable；大多数懒表只构造，不读文件。
OnPreInit
 创建/注册 m_kvTables。
 加载 AudioGlobalConfig、WorldInitSetting 两个 InitialAssets 配置。
LoadGameConfigs
 _LoadConstConfigs()
 _LoadJsonConfigs()
 _PostProcessConfigs()
 _PreloadAllLevelData()

    三类基础读取算法¶

    InitialAssets asset¶

    入口:

    路径:

    Assets/Beyond/InitialAssets/Settings/{configName}.asset

    读取方式:

    ResourceRouter.GetInitialConfigFullPath(configName)
StringPathHash.op_Implicit(path)
m_initialAssetLoader.Load<T>(hash)
handle.Get<T>()

    这里使用 m_initialAssetLoader，不是 m_mainAssetLoader。

    GameplayConfig ScriptableObject asset¶

    入口:

    路径:

    Assets/Beyond/DynamicAssets/GameData/GameplayConfig/{configName}.asset

    读取方式:

    ResourceRouter.GetGameplayConfigFullPath(configName)
StringPathHash.op_Implicit(path)
DataManager.LoadAsset<T>(hash)
 ResourceManager.CheckExists(hash, RootCategory.Main)
 m_mainAssetLoader.Load<T>(hash)
 handle.Get<T>()

    GameplayConfig JSON¶

    入口:

    路径分支:

    canUseMemoryPack = true:
 Data/Json/GameplayConfig/{configName}.json
canUseMemoryPack = false:
 Assets/Beyond/DynamicAssets/GameData/GameplayConfig/JsonCfg/{configName}.json

    读取方式:

    ResourceRouter.GetGameplayJsonConfigPath(configName, canUseMemoryPack)
StringPathHash.op_Implicit(path)
DataManager.LoadJson<T>(hash, settings=null, canUseMemoryPack, useCache=canUseMemoryPack)

    DataManager.LoadJson<T> 的底层分支:

    • canUseMemoryPack=true: 走 ResourceManager.DeserializeFromJson<T>(hash, settings)，并按 useCache 使用 JSON/MemoryPack 缓存。
    • canUseMemoryPack=false: 先 ResourceManager.CheckExists(hash, Main)，再加载 TextAsset，取 TextAsset.text，最后 JsonHelper.TryDeserializeObject<T>，也就是 Newtonsoft JSON 路径。

    特殊 Initial/构造字段¶

    字段 路径 读取方式
    audioGlobalConfig Assets/Beyond/InitialAssets/Settings/AudioGlobalConfig.asset OnPreInit 调 _LoadSingleInitialConfig<AudioGlobalConfig>("AudioGlobalConfig")
    worldInitSetting Assets/Beyond/InitialAssets/Settings/WorldInitSetting.asset OnPreInit 调 _LoadSingleInitialConfig<WorldInitSetting>("WorldInitSetting")
    cameraShakeConfigTable Assets/Beyond/DynamicAssets/GameData/Common/CameraShake/{key}.asset 构造时创建 CameraShakeConfigTable；按 key 懒加载 CameraShakeConfig asset 并缓存
    movementEnv 见下方两条 MovementSetting 构造时创建，_LoadConstConfigs 调 MovementEnv.LoadSettings(...)
    movementSetting 无独立路径 属性直接返回 movementEnv.currentSetting

    MovementEnv.LoadSettings 额外读取两个 MovementSetting:

    Assets/Beyond/DynamicAssets/GameData/GameplayConfig/MovementSetting/MovementSetting_Default.asset
Assets/Beyond/DynamicAssets/GameData/GameplayConfig/MovementSetting/MovementSetting_Aglina.asset

    后续 MovementSettingModifier 也是按名字通过 _LoadSingleConfig<MovementSettingModifier> 走同一 GameplayConfig asset 规则。

    懒加载 KVTable 字段¶

    这些表通常在构造函数里创建、OnPreInit 注册到 m_kvTables，读取发生在 TryGetData/ReloadTable/预加载时。通用流程是:

    先查 KVTable 内部缓存
缓存未命中时按 key 拼路径
StringPathHash.op_Implicit(path)
LoadAsset<T> 或 LoadJson<T>
成功后 Add 到 KVTable 缓存
    字段 路径/规则 读取方式
    interactiveTemplateIdTable Data/Json/Interactive/InteractiveTable.json ReloadTable() 读取 InteractiveTable，建立 template id/path 和父子 template 映射
    interactiveMeshTable Assets/Beyond/DynamicAssets/Gameplay/Interactives/MeshPrefabs/{key} 按 key 加载 GameObject 主包 asset
    interactiveModelTable Assets/Beyond/DynamicAssets/Gameplay/Interactives/PostModels/{key} 按 key 加载 GameObject 主包 asset
    interactiveSafeZoneSdfTable Assets/Beyond/DynamicAssets/Gameplay/Interactives/Generated/SafeZone/{key} 按 key 加载 Texture2D 主包 asset
    cameraControlStateTable Assets/Beyond/DynamicAssets/Gameplay/Cameras/ControlStates/{key} 按 key 加载 CameraControlStateCustomConfig asset
    mapConfigTable Data/Json/MapConfig/{mapIdStr}.json ResourceRouter.GetMapJsonConfigFullPath 后 LoadJson<MapConfig>
    levelConfigTable Data/Json/LevelConfig/{levelId}.json LoadJson<LevelConfig>
    specialLevelToMapTable 无独立 JSON 路径 从 Beyond.Cfg.GameData.specialLevelToMapTable 查表后缓存
    spawnerConfigTable 普通: Data/Json/SpawnerConfig/{configId}.json；sc_ 开头: Data/Json/SpawnerConfig/{levelId}/{configId}.json TryGetData 从 configId 解析 level 目录，LoadJson<SpawnerConfigData>
    blockDataTable Assets/Beyond/DynamicAssets/GameData/Minigame/Puzzle/Block/{blockId}.json ResourceRouter.GetBlockDataFullPath 后 LoadJson<BlockData>，TextAsset/Newtonsoft
    chessboardDataTable Assets/Beyond/DynamicAssets/GameData/Minigame/Puzzle/Chessboard/{chessboardId}.json ResourceRouter.GetChessboardDataFullPath 后 LoadJson<ChessboardData>，TextAsset/Newtonsoft
    modelViewStateControllersDataTable Data/Json/Interactive/ModelViewStateControllerData/{modelId}.json LoadJson<MVSCModelViewStateControllerData>
    levelMountPointTable Data/Json/LevelMountPoint/{levelId}.json GetLevelMountPointFullPath 后 LoadJson<LevelMountPointRoot>，带自定义 JsonSerializerSettings
    interactiveTemplateIdGroupTable Assets/Beyond/DynamicAssets/GameData/GameplayConfig/InteractiveTemplateIdGroupConfig.asset 首次查询时 _LoadSingleConfig<InteractiveTemplateIdGroupConfig>，再读 groupDict[id]
    missionRuntimeAssetTable Data/Json/MissionRuntimeAsset/{missionId}.json GetMissionRuntimeAssetJsonFullPath 后 LoadJson<MissionRuntimeAsset>，并返回 loadFromCache/loadFromDev
    missionRuntimeMetaTable Data/Json/MissionRuntimeAsset/{missionId}_meta.json GetMissionMetaJsonFullPath 后 LoadJson<MissionRuntimeMetaAsset>，并返回 loadFromDev
    matrixShockWaveBeatConfigTable Data/Json/GameplayConfig/MatrixShockWaveBeatConfigTable.json TryGetMatrixShockWaveBeatConfig 触发读取/查找

    还原和导出时的注意点¶

    • AbilityEntityTable、GlobalBuffTable、ProjectileTable 等 ScriptableObject 表不是 JSON，路径在 Unity asset bundle 里，必须按 Unity asset 反序列化或运行时导出。
    • Data/Json/GameplayConfig/*.json 这一类大多可以复用现有 MemoryPack JSON 读取逻辑。
    • JsonCfg/*.json、MiniGame Puzzle 的 block/chessboard JSON 走 TextAsset + Newtonsoft，不能简单套 MemoryPack schema。
    • DataManager 中不少字段是派生字典或 KVTable 缓存；导出时不要把属性 getter 当作独立资源路径。
