    伤害计算公式 (NormalEntity 路径)¶

    NormalEntity 是玩家↔怪物的常规伤害路径，不包含 Tower 和 IndependentHealth。

    flowchart TD
    N0["_CalculateDamageResultByType"]
    N1["IndependentHealth 路径 → _CalculateDamageResultForIndependentHealth"]
    N2["Tower 路径 → _CalculateDamageResultForTower"]
    N3["NormalEntity 路径 → _CalculateDamageResultForNormalEntity ← 本文重点"]
    N0 --> N1
    N0 --> N2
    N0 --> N3

    完整公式¶

    BattleFormula::CalculateDamage(已确认)，该函数在 DamagePackData::GetFinalAttackValue 之后额外乘上 DEF、受伤/抗性、弱点、庇护等直接乘子。

    finalAttackValue = calcResult × finalDamageScale
finalDamage =
 finalAttackValue
 × criticalScale
 × defResistance
 × damageTypeResistance
 × weaknessDmgScalar
 × shelterDmgScalar
 × optionalIgniteDamageScalar
 × optionalPhysicalInflictionDamageScalar

    其中：

    字段 计算路径 来源 说明
    finalDamageScale DamagePackData::_GetFinalDamageScale(已确认) zone 系统 DamageScaleProcessorConfig.allZones 的 attacker/defender zone 总乘积
    criticalScale attackerAttributes[10] CalculateDamage 暴击时 1 + CriticalDamageIncrease，未暴击为 1
    defResistance defenderAttributes[3] _GetDefResistanceValue(已确认) DEF 专用公式，见 §2.8
    damageTypeResistance defenderAttributes[4/5/6/7/48/60] × defenderAttributes[80..85] _GetDamageTypeResistanceValue(已确认) 按伤害类型选择，最后 max(0, value)
    weaknessDmgScalar attackerAttributes[62] CalculateDamage 直接乘子，不进 zone
    shelterDmgScalar defenderAttributes[63] CalculateDamage 直接乘子，不进 zone
    optionalIgniteDamageScalar attackerAttributes[49] CalculateDamage damageDecorateMask & 已确认 != 0 时乘
    optionalPhysicalInflictionDamageScalar attackerAttributes[25] CalculateDamage damageDecorateMask & 已确认 != 0 时乘

    DamageType.Real(1) 会让 defResistance = 1，即跳过 DEF；DamageType.LifeDrain(5) 在 CalculateDamage 里走特殊分支，不进入这套普通伤害公式。

    attackerAttributes[i] = attacker.Attributes.GetValue((AttributeType)i) // i=0..77+
defenderAttributes[i] = defender.Attributes.GetValue((AttributeType)i)

    每个属性值经过 Attributes::_CalculateFinalAttributeValue(attrType, baseMask, filterMask) 计算：

    finalAttr = base + sum(additions) × product(multipliers) + sum(finalAdditions) × product(finalMultipliers)

    关键属性索引 (来自 Common.Beyond.AttributeType):

    索引 属性 用途
    0 Level 等级
    1 MaxHp 最大生命
    2 Atk 攻击力（基础伤害输入）
    3 Def 防御力（减伤输入）
    4 PhysicalDamageTakenScalar 物理受伤倍率
    9 CriticalRate 暴击率
    10 CriticalDamageIncrease 暴击伤害加成
    25 PhysicalInflictionDamageScalar 特定 decorateMask 下的额外乘子
    17 NormalAttackDamageIncrease 普攻增伤
    28 UltimateSkillDamageIncrease 终结技增伤
    32 NormalSkillDamageIncrease 技能增伤
    33 ComboSkillDamageIncrease 连击技增伤
    39 Str 力量
    40 Agi 敏捷
    48 NaturalDamageTakenScalar 自然受伤倍率
    49 IgniteDamageScalar 特定 decorateMask 下的额外乘子
    50 PhysicalDamageIncrease 物理伤害加成
    51-54 Fire/Pulse/Cryst/NaturalDamageIncrease 元素伤害加成
    60 EtherDamageTakenScalar 以太受伤倍率
    61 DamageToBrokenUnitIncrease 对破韧单位增伤
    62 WeaknessDmgScalar 弱点伤害倍率，直接乘最终伤害
    63 ShelterDmgScalar 庇护伤害倍率，直接乘最终伤害
    64-69 *EnhancedDmgIncrease 增幅伤害加成
    70-75 *VulnerableDmgIncrease 脆弱伤害加成
    80-85 *DmgResistScalar 类型抗性倍率，直接参与 damageTypeResistance

    阶段二：BeforeCalculate 处理器¶

    在计算 rawDamage 之前，处理器修改 packData 中的属性：

    处理器 作用
    AttackerAtkProcessor attackerAttributes[Atk] *= scale (BlackboardDouble)
    AttackCriticalRateProcessor 调整暴击率 addition
    AttackerCriticalDamageProcessor 调整暴击伤害 scale
    InstantModifyAttribute 任意属性即时修改
    DamageIndependentHealthProcessor IndependentHealth 专用

    因此 DEF 的实战结算不依赖这些 processor，而是在 BattleFormula::_GetDefResistanceValue 中直接读取防御方属性快照。

    阶段三：主伤害计算 (rawDamage)¶

    atkCalculation.Evaluate(attacker, defender, blackboard, context, attackerAttributes, defenderAttributes, argsForServer)

    类型一：AtkScaleCalculation（最常用）¶

    rawDamage = 攻击力 × atkScale
其中:
 攻击力 = attackerAttributes[Atk] // 属性索引 2
 atkScale = 从技能 BlackboardDouble 解析 // 例如 1.5 = 150%

    类型二：PrimaryAttrCalculation¶

    attr = GetAttribute(source, attrType)
rawDamage = applyScale ? attr × valueScale : attr
其中:
 source = AttackerOrHealer (0) 或 Target (1)
 attrType = Main (1) 或 Sub (2) // 主属性或副属性
 valueScale 从 BlackboardDouble 解析

    类型三：MultiplyAttributeCalculation¶

    attr = GetAttribute(source, attributeType)
rawDamage = attr × multiplier + addition
其中:
 multiplier, addition 从 BlackboardDouble 解析

    类型四：DefiniteValueCalculation（固伤）¶

    rawDamage = applyScale ? value × valueScale : value

    类型五：BreakingAttackCalculation（破防攻击）¶

    attr = GetAttribute(source, Atk)
rawDamage = attr × atkScale × multiplier

    GetAttribute 实现¶

    直接按属性枚举值索引 attackerAttributes[] 或 defenderAttributes[]。

    阶段四：DamageScale Zone 系统¶

    六区定义¶

    Zone alias isMultiply 说明
    ProdCalcZone 独立增减伤 true (乘算) 独立因子逐个相乘 (1+a)×(1+b)×...

    EnhancedDmgIncrease 增幅增伤 false (加算) 增幅反应
    VulnerableDmgIncrease 脆弱增伤 false (加算) 脆弱反应
    AbnormalAndBurstIncrease 法术爆发与异常增伤 false (加算) 异常伤害 + 爆发伤害
    ComboCalcZone 连击增伤 false (加算) 连击技能

    ProdCalcZone (Defender)：

    来源 说明
    技能内 DamageScaleProcessor zoneName=ProdCalcZone, side=Defender 的 processor

    抗性和受伤倍率不在 Zone 系统内，而是在 BattleFormula::_GetDamageTypeResistanceValue(已确认) 中按伤害类型直接取属性相乘。 NormalCalcZone / EnhancedDmgIncrease / AbnormalAndBurstIncrease / ComboCalcZone (Defender)： 防御方通常不向这些 zone 添加属性值（初始值为 1.0），除非技能/Buff 中包含 DamageScaleProcessor(side=Defender, zoneName=...) 的 processor。

    Zone 计算公式¶

    加算区 (isMultiply=false)：

    zoneValue = 1 + sum(该 zone 所有 modifier 值)

    乘算区 (isMultiply=true, 仅 ProdCalcZone)：

    zoneValue = ∏(1 + modifier_i) // 每个 modifier 独立相乘

    单 zone 最终值（mergeAttackerAndDefender=false，全部 zone 均为此情况）：

    finalZoneValue = zoneValue_attacker × zoneValue_defender

    总 zone 倍率（_GetFinalDamageScale 已确认 确认）：

    finalDamageScale = ∏(for each zone: zoneValue_attacker × zoneValue_defender)
 = NormalCalcZone_a × NormalCalcZone_d
 × EnhancedDmgIncrease_a × EnhancedDmgIncrease_d
 × VulnerableDmgIncrease_a × VulnerableDmgIncrease_d
 × AbnormalAndBurstIncrease_a × AbnormalAndBurstIncrease_d
 × ComboCalcZone_a × ComboCalcZone_d
 × ProdCalcZone_a × ProdCalcZone_d

    动态注入：DamageScaleProcessor¶

    DamageScaleProcessor {
 side: DamageScaleSide (Attacker=0 / Defender=1)
 zoneName: string // 目标 zone 名称
 addition: BlackboardDouble // 加值 (从 Blackboard 解析)
}

    通过 DamagePackData::ModifyDamageScaleZone(side, zoneName, value) (已确认) 将 addition 值加到对应 side 的对应 zone 数组中。 DamageScaleProcessor 可在 BeforeCalculate 或 AfterCalculate 阶段生效（取决于 processTiming 设置）。AfterCalculate 阶段注入的值在 rawDamage 计算之后，仅影响最终的 zone 倍率。

    阶段五：AfterCalculate 处理器¶

    处理器 作用
    ModifyCalcResult result *= baseMultiplier ^ multiplierCnt (仅 Multiply 类型)
    DamageScaleProcessor 向指定 zone 添加 addition 值

    Def 与 ShelterDmgScalar 的最终乘区¶

    Def 的读取来源¶

    Common.Beyond.AttributeType.Def = 3。DamagePackData 创建时会快照双方属性：

    所以 Def 的结算输入是 DamagePackData.defenderAttributes[3]，也就是创建伤害包那一刻 防御者的最终属性快照。

    Def 公式¶

    BattleFormula::_GetDefResistanceValue(已确认) 硬编码读取

    以当前表值 efficiencyOfDEF = 0.01 计算：

    def >= 0 => defResistance = 1 / (1 + def * 0.01)
def < -0.00001 => defResistance = 2 - pow(0.99, -def)

    这不是 DamageScaleProcessor zone。它是 BattleFormula 的独立直接乘区：

    finalDamage *= defResistance

    没有 AttackerPenProcessor、DefenderDefProcessor、BattleFormula.CalculateDamage 或 _GetDefResistanceValue 补丁。因此当前本地版本以 native 公式为准：没有额外的 “攻击方穿透先修正 DEF”步骤，defResistance 只由 defenderAttributes[3] 和 BattleConst.efficiencyOfDEF 决定。

    ShelterDmgScalar / ShelterDmgTakenScalar¶

    代码枚举名是 AttributeType.ShelterDmgScalar = 63；数据或策划语义上可理解为

    归属结论：

    字段 Zone 作用对象 方法 说明
    Def(3) defender 数据模板决定 _GetDefResistanceValue defResistance 直接乘区
    ShelterDmgScalar(63) defender 1.0 CalculateDamage shelterDmgScalar 直接乘区
    WeaknessDmgScalar(62) attacker 1.0 CalculateDamage weaknessDmgScalar 直接乘区

    AttributeMetaTable.json 中 62/63 默认值都是 1.0。示例 buff：

    • buff_common_affixes_shelter.json 使用 BaseFinalMultiplier 修改 ShelterDmgScalar，黑板参数为 rate。

    • buff_common_affixes_weak.json 使用 BaseFinalMultiplier 修改 WeaknessDmgScalar，黑板参数为 rate。 BattleFormula.CalculateDamage 阶段读取 defender 快照属性 63 并直接乘最终伤害。

    怪物间伤害转移¶

    纠正：游戏中不存在"部位"转移。transferredSource 字段用于怪物之间的伤害转移（如 A 怪物死亡后剩余伤害转给 B 怪物）。这由 AbilitySystemForEnemyPart::get_useMainBodyHp 调用证实——这里的 "EnemyPart" 指多阶段战斗中的不同单元，而非身体部位。

    暴击计算¶

    暴击相关处理器:

    • AttackCriticalRateProcessor: 增加暴击率 (addition)
    • AttackerCriticalDamageProcessor: 暴击伤害倍率 (scale)

    最终暴击判定在 BattleFormula::CalculateDamage(已确认) 内完成：

    暴击不走 ProdCalcZone。AttackCriticalRateProcessor 和 它们本身不会修改属性。基础暴击率/暴击伤害仍来自 DamagePackData 创建时的属性快照。

    韧性(Poise)并行计算¶

    与伤害计算并行运行（_ProcessDamage 中第 7 步）：

    • PoisePackData::Create(attacker, defender, damageDecorateMask, actionEnvironment) — 创建韧性包
    • PoisePackData::ApplyPoiseModifier(BeforeCalculate) — 应用韧性前处理器
    • BattleFormula::CalculatePoise(poisePackData, reportInfo) — 韧性伤害计算
    • PoisePackData::ApplyPoiseModifier(AfterCalculate) — 应用韧性后处理器
    • Modifier::NewPoise(...) — 应用韧性伤害
    • PoisePackData::ClearInstantAttributeModifier() — 清理即时修改
