    AuraAction 逻辑分析¶

    分析目标: Beyond.Gameplay.Core.AuraAction 数据类: Beyond.Gameplay.Core.AuraAction+Data - AKEDataMining/data/Json/SkillData - AKEDataMining/data/Json/BuffData

    最后更新: 2026-05-29

    结论摘要¶

    AuraAction 是持续型光环 Action。它维护一组候选 AbilitySystem，每 tick 判断哪些目标进入/离开光环；进入时可以添加 Buff 并执行 actionInAura，离开时结束由本光环创建的 Buff 并执行 actionWhenExitAura。 运行时分成两类:

    1. RangedAura: 以 auraRoot 指向的目标为光环根节点，按 shapeData 做范围查询。
    2. GlobalAura: 不做形状范围，初始化时扫描全局实体，并监听 AbilitySystem 创建/释放事件维护候选集。

    Data 字段¶

    字段 偏移 类型 说明
    auraDebugName 0x20 string 调试名，无逻辑用途
    auraType 0x30 AuraType RangedAura=0, GlobalAura=1
    auraRoot 0x38 TargetSettings RangedAura 的跟随/中心目标
    shapeData 0x48 ColliderShapeData Box / Capsule / Sphere 范围
    targetObjectType 0x50 ObjectType GlobalAura 初始化候选时的对象类型过滤
    targetFilter 0x58 TargetFilter 阵营、存活、槽位、tag 等过滤
    excludeOwner 0x60 bool 是否排除 owner
    includeUnmarkable 0x61 bool 是否允许不可标记目标
    limitInfluenceCountPerTarget 0x62 bool 是否限制单目标影响次数
    maxInfluenceCountPerTarget 0x64 int 单目标最大影响次数
    buffSource 0x68 ActionTargetType 添加 Buff 时使用的 Buff 来源
    buffInput 0x70 List<BuffInput> 进入光环时添加的 Buff 列表
    overrideBuffIconDuration 0x78 bool 是否覆盖 Buff 图标时长来源
    buffIconDurationSource 0x80 BuffIconDurationSourceSetting 图标时长来源配置
    inheritSourceSkillCastId 0x88 bool 是否继承源技能 cast id
    actionInAura 0x90 SequenceActionData 目标进入光环时执行
    actionWhenExitAura 0x98 SequenceActionData 目标离开光环时执行

    ColliderShapeData 的关键字段:

    字段 偏移 说明
    _shape 0x10 Box=0, Capsule=1, Sphere=2, Point=3
    _rotationOffset 0x14 形状旋转偏移
    _useExtentKey 0x20 是否从 blackboard 读 extent
    _extent 0x24 Box/Capsule 尺寸向量
    _useCenterKey 0x48 是否从 blackboard 读 center
    _center 0x4C 形状中心偏移
    _height / _heightKey 0x78 / 0x70 Capsule 高度
    _radius / _radiusKey 0x88 / 0x80 Sphere/Capsule 半径

    OnEnd 遍历 m_inAuraTargets，对每个仍然有效的目标调用 _RemoveInfluence。之后:

    • m_actionWhenEnterAura.End()
    • m_actionWhenExitAura.End()

    OnTick 是光环的核心刷新逻辑:

    要点:

    • 已在光环内的目标如果不再在候选集里，或不再通过 _Filter，会被移除影响。
    • 新候选必须同时通过 _Filter 和 _CanApplyInfluence。
    • 进入前还会经过 FilterEnemyPartBlockAngle，用于处理敌人部位/遮挡角度，避免同一敌人的多个 part 重复或非法命中。

    RangedAura 候选集更新¶

    1. 读取 m_followedTarget 对应实体的 meshForward；如果没有实体，使用 Vector3.forward。
    2. 读取 curPosition 和 lastPosition，计算位移向量。
    3. 读取 data.shapeData，按 _shape 分支:
    4. Sphere: 使用中心偏移和半径。
    5. Capsule: 使用中心偏移、高度、半径。
    6. Box: 使用中心偏移、extent、rotationOffset。
    7. 根据位移长度选择查询方式:
    8. 位移较小时，使用 PhysHelper.OverlapSphereNonAlloc / OverlapCapsuleNonAlloc / OverlapBoxNonAlloc。
    9. 位移较大时，使用 SphereCastAll / CapsuleCastAll / BoxCastAll 做 swept 查询，避免光环高速移动时穿过目标。

    过滤规则¶

    _Filter(target) 主要包含三层:

    1. excludeOwner
    2. 如果开启，会比较目标与 owner/source 侧 AbilitySystem。
    3. 命中 owner/source 时返回 false。
    4. includeUnmarkable
    5. 如果为 false，调用 AbilitySystemUtils.IsUnmarkable(target, factionIndex)。
    6. 目标不可标记时返回 false。
    7. 如果为 true，跳过这一步。
    8. targetFilter
    9. 使用 TargetFilter 的阵营、存活、slot、tag 等配置。
    10. 汇编中能看到按 source faction / target faction 生成 bit mask，再与配置 mask 做比较。
    11. JSON 中常见的是 autoSetTargetFaction=true 与 factionTarget=Anti/Ally。 _CanApplyInfluence(target) 负责防重复和最大次数:

    施加影响¶

    _ApplyInfluence(target) 的流程:

    1. 解析实际 key target。
    2. 如果目标是敌人部位/part，且存在主身体引用，会把主身体作为字典 key。
    3. 这样可以避免同一怪物多个 part 重复叠光环影响。
    4. 更新 m_inInfluenceTargets。
    5. 如果 key 不存在，TryAdd(key, 1)。
    6. 如果 key 已存在，只递增计数并返回，不重复添加 Buff，也不重复执行进入 action。
    7. 如果开启 limitInfluenceCountPerTarget，更新 m_targetInfluenceCountDic。
    8. 如果 buffInput 不为空:
    9. 通过 AbilityActionUtils.GetActionTarget(this, data.buffSource, ...) 解析 Buff source。
    10. 创建 List<ObjectPtr<Buff>>，用于记录本次给目标添加的 Buff。

    11. 对每个 BuffInput 调用:

    12. 如果 overrideBuffIconDuration=true，会根据 buffIconDurationSource 额外生成图标时长来源参数。

    13. 如果 m_actionWhenEnterAura 不为空:
    14. 创建单目标 TargetHandle。
    15. SetTarget(target)。
    16. 调 SequenceAction.ExecuteInstant(TargetHandleView, AbilitySystem.EventContext)。

    移除影响¶

    _RemoveInfluence(target) 与 _ApplyInfluence 使用同样的 key target 归一化规则。 流程:

    1. 查 m_inInfluenceTargets[key]。
    2. 如果不存在，直接返回。
    3. 如果计数大于 1，只递减计数并返回。
    4. 如果计数小于等于 1:
    5. 从 m_inInfluenceTargets 移除 key。
    6. 如果 m_createdBuffDic[key] 存在，遍历本光环创建的 Buff。
    7. 对仍然有效的 Buff 调用 AbilitySystem.FinishBuff(buff, ServerActionParams)。
    8. 从 m_createdBuffDic 移除 key。
    9. 如果 m_actionWhenExitAura 存在，构造单目标 TargetHandle 并 ExecuteInstant。 这说明 AuraAction 对 Buff 的生命周期是强绑定的: Buff 是进入光环时创建、离开光环或 Action 结束时由 AuraAction 主动结束。
