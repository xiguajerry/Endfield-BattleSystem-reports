    TickIntervalAction¶

    更新时间：2026-06-14

    Class¶

    • 原游戏类：Beyond.Gameplay.Core.TickIntervalAction
    • 数据类：Beyond.Gameplay.Core.TickIntervalAction+Data

    Fields¶

    Data 字段：

    • executeEachFrame
    • tickInterval
    • tickIntervalBlackboardKey
    • useTickIntervalBlackboardKey
    • actionOnTick
    • useTimeDilationDt

    Runtime 字段：

    • m_inputTarget
    • m_tickIntervalTimer
    • m_tickedCount
    • m_checkFrame

    Methods¶

    方法 状态 结论
    ExecuteInternal 已复核 复制输入 TargetHandleView 到 m_inputTarget，初始化 timer/count/frame。
    OnTick 已复核 按 delta 累加 timer，并按 each-frame 或 interval 条件执行 actionOnTick。
    OnEnd 已复核 End(actionOnTick sequence)，dispose m_inputTarget，清空 runtime state。
    OnReset 已复核 重置 timer、count 和输入 target。
    Data.ConvertToServerActionMap 已复核 只 merge actionOnTick.ConvertToServer()；tick interval 本身不产生额外 server payload。

    Call Entrypoints¶

    • Skill / buff / aura / move tick 子 action 执行 TickIntervalAction.ExecuteInternal。
    • action runtime 每帧调用 TickIntervalAction.OnTick(deltaTime)。
    • action owner 结束时清理 handle，对应 OnEnd。

    Lifecycle¶

    1. ExecuteInternal 解析 interval，并保存输入 target/context 快照。
    2. 非 each-frame 模式 m_tickedCount=0，第一次 OnTick 即满足 timer >= 0 并触发首跳。
    3. executeEachFrame=true 每帧触发；若非 each-frame 但 interval < 0.03299s，原游戏强制切到 each-frame。
    4. useTimeDilationDt=true 时按 action source 的 time scale 推进。

    5. OnEnd 停止后续 tick 并清理 target 快照。

    6. useTickIntervalBlackboardKey=true 时从 action blackboard 读取 interval key，否则使用 data 上的 tickInterval。

    7. 非 each-frame 首跳立即触发，而不是等待一个完整 interval。
    8. 后续 tick 使用 m_inputTarget 快照，不重新读取外部 target handle。
    9. OnTick 触发时调用 SequenceAction.ExecuteInstant(targetHandle, eventContext=0)；ExecuteInstant 会调用 SequenceAction.Execute(..., resetAfterExecute=true)，因此 child action state 保留在同一个 TickIntervalAction child object 上，但每次触发后收到 ResetAfterExecute(1)。
    10. OnEnd 调 SequenceAction.End()；OnReset(reason) 调 SequenceAction.Reset(reason)。
