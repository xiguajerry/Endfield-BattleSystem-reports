    Methods¶

    方法 确认状态 确认行为
    OnCreate 已确认 创建进入/离开光环的 SequenceAction runtime。
    OnReAssign 已确认 重绑定进入/离开 action environment。
    ExecuteInternal 已确认 初始化候选集、根目标、GlobalAura 监听、skill cast info。
    OnTick 已确认 每 tick 刷新候选，应用进入影响，移除离开影响。
    _ApplyInfluence 已确认 进入光环：增加影响计数、添加 buff、执行 actionInAura。
    _RemoveInfluence 已确认 离开光环：结束本光环创建的 buff、执行 actionWhenExitAura。
    _CanApplyInfluence 已确认 防重复并检查单目标最大影响次数。
    _Filter 已确认 owner、unmarkable、TargetFilter、alive/tag 等过滤。
    OnEnd 已确认 结束所有影响、结束子 action、释放根目标。

    Call Entrypoints¶

    • Skill / buff / global buff action timeline 执行 AuraAction.ExecuteInternal。
    • 原游戏 generic action scheduler 每帧调用 AuraAction.OnTick。
    • action 生命周期结束时调用 AuraAction.OnEnd。

    Lifecycle¶

    1. OnCreate 创建 actionInAura / actionWhenExitAura 对应的 SequenceAction。
    2. ExecuteInternal 清空 runtime 容器，填充 skill cast info。
    3. GlobalAura 初始化候选；RangedAura 解析 auraRoot 和空间根。
    4. OnTick 刷新候选，目标进入时 _ApplyInfluence，目标离开或 release 时 _RemoveInfluence。

    5. OnEnd 移除所有还在影响中的目标并结束 action runtime。

    6. GlobalAura 的 ability start/release 事件只维护候选；真实进入/离开仍由 OnTick 判定。

    7. release 事件会从候选和 in-aura 集合移除目标，并立即 _RemoveInfluence。
    8. targetObjectType 是 bitmask 过滤。
    9. excludeOwner、includeUnmarkable、targetFilter.checkAlive 和 tagQuery 都属于 deterministic gameplay filter。
    10. actionInAura / actionWhenExitAura 是进入/离开时执行的子 action，不是配置加载时执行。
