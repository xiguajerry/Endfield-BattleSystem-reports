    FindTargetAction 行为规则¶

    • selectorOwner is resolved through AbilityActionUtils.GetActionTarget(..., selectorOwner, inputTarget, currentTarget, selectorOwnerContextKey).
    • center is resolved through the overload that accepts Nullable<Vector3>& targetPos.
    • If the by-ref center position is missing, native derives a position from the center entity root/mount point and optionally projects it to ground.
    • Immediately before calling SelectorUtils.FindTarget, the local Nullable<Vector3> has hasValue = 1; therefore normal FindTargetAction execution uses the center-present path in SelectorUtils.FindTarget.
    • The returned TargetHandle is passed to IActionEnvironment.Context.UpdateTargetGroup when the action context exists.

    FindTarget¶

    1. Creates a mutable TargetHandle.
    2. Throws/logs through the native error path when selectorData is null.
    3. Reads selectorData.finderData at offset 0x10.

    4. Returns the resulting TargetHandle.

    5. 目标选择的确定性结果主要来自 finder 子类、过滤器和权重排序。

    6. 空间相关行为包括角度、距离、屏幕、碰撞、挂点和地面投影；只有当这些因素影响确定性战斗结果时才需要进入完整模拟范围。

    PriorityFilter 后处理规则¶

    • Reads limitMaxNum at this+0x15.
    • When limitMaxNum == true, passes this.maxNum (this+0x18) as maxNum.
    • When limitMaxNum == false, passes 0x80 as maxNum.
    • Reads onlyReserveMaxPriorityTargets at this+0x14.
    • Reads buffFilterSettings at this+0x20.

    FilterUtils 权重规则¶

    • _Filter(...) 状态 已确认 forwards to _Filter(..., DEFAULT_SMALL_ENOUGH_GAP) 状态 已确认.
    • _Filter_CurHpAsc 状态 已确认 and _Filter_CurHpDes 状态 已确认 use current HP. CurHpDes sets weight to negative current HP.
    • _Filer_CurHpRatioAsc 状态 已确认 and _Filer_CurHpRatioDes 状态 已确认 use current HP and max HP. Constants show native attribute slots 0x4c and 0x4d.

    • _Filter_BuffStackNumAsc 状态 已确认 and _Filter_BuffStackNumDes 状态 已确认 query AbilitySystem.GetBuffCountById, GetBuffCountByTag, GetBuffIdCountByTag, and AbilitySystemUtils.ResolvePartBuffSource depending on BuffFilterSettings.

    • PriorityFilter.Data 不应被视为空对象；limitMaxNum、maxNum、onlyReserveMaxPriorityTargets 和 buffFilterSettings 都会影响目标列表。

    • 非空间过滤器的确定性核心是 Buff 层数排序、最大数量截断，以及最优权重目标保留。
    • limitMaxNum=false 时最大数量使用默认值 0x80。

    BattleManager Smart-Target Scorers¶

    1. Each scorer checks BattleManager target state exists, then calls _TryRefreshMarkTargetScore(checkInScreen=true).
    2. Each scorer first checks get_lockTarget().
    3. If the lock target satisfies the scorer-specific condition, the method writes bestTarget and returns true immediately.
    4. If lock target does not satisfy the condition, the method enumerates BattleManager.MarkTargetPriorityStruct from the mark-target priority queue.
    Method 状态 Behavior
    • Implement SelectByBuff, SelectByTag, and SelectByBuffStackNum in InternalSkill.storeSmartTarget.
    • Preserve the native BuffStackNum difference from Buff/Tag: Buff/Tag are first-match scorers, BuffStackNum is max-stack scorer with strict > replacement.
    • Keep angle, distance, screen and enemy-part source transfer documented but not simulated at the current granularity.

    Lifecycle¶

    • Return value is a mutable native TargetHandle copied into action context through IActionEnvironment.Context.UpdateTargetGroup.

    • Exact mark-target priority queue construction remains a broader BattleManager target-scoring topic.
