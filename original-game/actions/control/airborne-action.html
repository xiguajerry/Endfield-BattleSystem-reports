    AirborneAction (击飞)¶

    类结构¶

    • AirborneAction extends AbilityAction
    • 数据类: AirborneActionData (仅包含 isEnable, priorityLevel, priorityOffset, serverActionIndex)
    • 核心执行在 AirborneAction::ExecuteInternal (已确认, ~3320 指令)
    • 子逻辑在 _TryAirborne (已确认, 827 bytes)

    完整事件流程¶

    flowchart TD
    Start["AirborneAction 处理目标列表"]
    Loop["取下一个目标"]
    HasTarget{"还有目标？"}
    Parent{"目标是不可直接受击的敌人部位？"}
    UseParent["改用父级 AbilitySystem"]
    Event["触发 BattlePhysicalContext 事件"]
    Interrupt["打断目标当前动作"]
    SkipGuard{"当前状态跳过 no-guard 检查？"}
    HasGuard{"已有 buff_physical_no_guard？"}
    AddGuard["添加 buff_physical_no_guard 并记录 no-guard"]
    TryAirborne["执行击飞施加流程"]
    ObjectEvent{"目标对象类型触发关卡事件？"}
    RaiseEvent["抛出 0x19 / 0x1A 关卡事件"]
    Result["合并 success / interruptedWeakness 结果"]
    End["返回聚合结果"]

    Start --> Loop --> HasTarget
    HasTarget -- "否" --> End
    HasTarget -- "是" --> Parent
    Parent -- "是" --> UseParent --> Event
    Parent -- "否" --> Event
    Event --> Interrupt --> SkipGuard
    SkipGuard -- "是" --> TryAirborne
    SkipGuard -- "否" --> HasGuard
    HasGuard -- "否" --> AddGuard --> TryAirborne
    HasGuard -- "是" --> TryAirborne
    TryAirborne --> ObjectEvent
    ObjectEvent -- "是" --> RaiseEvent --> Result
    ObjectEvent -- "否" --> Result
    Result --> Loop

    _TryAirborne 详细流程¶

    flowchart TD
    Start["进入击飞施加流程"]
    Before["触发 BeforeControlledStateApplied"]
    BeforeEvents["输出方/承受方物理异常前置事件"]
    AddBuff["添加 buff_physical_airborne"]
    ReadParams["读取 airborneTime / airborneHeight / magnitude"]
    Clamp["数值按最小 1.0 钳制"]
    Direction["计算击飞方向"]
    Launch["LaunchAirborne"]
    Result{"施加结果"}
    Success["标记 success"]
    Weakness["标记 hasInterruptedWeakness"]
    Other["保持当前结果"]
    After["触发 AfterControlledStateApplied"]
    AfterEvents["承受方/输出方物理异常后置事件"]

    Start --> Before --> BeforeEvents --> AddBuff
    AddBuff --> ReadParams --> Clamp --> Direction --> Launch --> Result
    Result -- "Success" --> Success --> After
    Result -- "InterruptedWeakness" --> Weakness --> After
    Result -- "其他" --> Other --> After
    After --> AfterEvents

    与 Crush 的对比¶

    特性 Crush Airborne
    数据类 Crush (damageMultiplier, isExtra) Airborne (仅基础字段)
    damageMultiplier 从 JSON 读取 (BlackboardDouble) 固定 1.0 (静态常量)
    两阶段设计 ✓ (buff_physical_no_guard) ✓ (相同逻辑)
    Before/After 函数 相同 相同
    PhysicalInflictionType Crush (3) Airborne (0)
    应用的 buff buff_physical_crushed buff_physical_airborne
    ControlledState 方法 ApplyBlowOff LaunchAirborne
    额外参数 无 airborneTime, airborneHeight, direction
    前置事件 无 Event 0xB5 (BattlePhysicalContext)
    后置事件 无 RaiseEntityEvent(0x19, 0x1A)
    结果追踪 无 success, hasInterruptedWeakness
