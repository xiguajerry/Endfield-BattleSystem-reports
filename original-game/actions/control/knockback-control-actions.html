    击飞与控制 Action 逻辑分析¶

    范围: Beyond.Gameplay.Core 中击飞、浮空、击倒、拉拽、推开、打断、减速以及相关控制 Action。 - AKEDataMining/data/Json/SkillData - AKEDataMining/data/Json/BuffData

    最后更新: 2026-05-23

    结论摘要¶

    1. 敌人受控状态: AirborneAction / KnockDownAction / BlowOffEnemyAction / CrushAction / FractureAction / LaunchUpwardAction / TakeDownAction
    2. 最终落到 ControlledStateComponent
    3. 再进入 AirborneComponent、KnockDownComponent、EnemyBlowOffComponent
    4. 角色击飞
    5. BlowOffCharacterAction
    6. 最终落到 CharacterBlowOffComponent.ApplyBlowOff
    7. 不走敌人的 ControlledStateComponent 流程，而是通过角色的 MoveInput.Blown 施加速度。
    8. 位移控制/软控制
    9. PullAction 使用 PullComponent 持续拉拽。
    10. PushBackAction 是一次性推退位移。
    11. SlowAction 是 KeywordAction，本质是创建关键词 Buff，不是物理控制状态。
    12. InterruptAction 打断技能并附加短硬直 Buff。 需要修正之前分类中的一个点: AddDynamicCcsAction 和 InheritCCSAction 里的 CCS 不是 Crowd Control Stack，而是 Camera Control State。这两个 Action 属于相机控制状态，不应放在“击飞与控制”玩法逻辑分类里。

    公共枚举与底层组件¶

    ControlledStateDeadOption¶

    这解释了多个控制 Action 中的 deadOption: 配置控制是否允许作用于死亡/存活目标。

    PhysicalInflictionType¶

    BlowOffEnemyAction.DoBlowOff 会把该类型传给 AbilitySystem.BeforeControlledStateApplied / AfterControlledStateApplied，用于事件、物理异常、弱点击破等后续逻辑区分。

    ReturnTrueMethod¶

    AirborneAction、KnockDownAction、LaunchUpwardAction、TakeDownAction 均有这个字段。它控制 Action 的布尔返回值如何由“成功施加控制”和“是否打断弱点/状态”组合得出。

    敌人受控状态链路¶

    ControlledStateComponent.LaunchAirborne¶

    关键逻辑:

    1. 取 airborneComponent。
    2. 调用 AirborneComponent.BeforeAirborne(source, isLaunchUpward)。
    3. 只有返回 PhysicsStatusApplyResult.Success 才继续。
    4. 如果当前不是浮空状态，切换 ControlledState 到 Airborne。
    5. 如果当前是 BlowOff，会先保存并恢复 rootHeight。
    6. 调用 AirborneComponent.Launch(source, airborneOption, fromKnockDown, restartAirborne, isLaunchUpward)。
    7. 触发实体事件。
    8. 若目标死亡相关对象存在，会用剩余受控时间延迟死亡特效/释放。

    这说明浮空不是简单改坐标，而是一个正式的敌人受控状态。

    ControlledStateComponent.ApplyKnockDown¶

    关键逻辑:

    1. 取 knockDownComponent。
    2. 调用 KnockDownComponent.BeforeKnockDown(source, fromAirborne)。
    3. 成功后:
    4. 如果是从浮空转击倒，会走 AirborneComponent.KnockDown(duration + source.knockDownTimeAddition, faceDir)。
    5. 否则切到 ControlledState.KnockDown 并由 KnockDownComponent.ApplyKnockDown 接管。
    6. 会触发 AbilitySystem.Event 中的击倒相关事件。

    ControlledStateComponent.ApplyBlowOff¶

    关键逻辑:

    1. 取 blowOffComponent。
    2. 调用 EnemyBlowOffComponent.BeforeBlowOff(source, isPureBlowOff)。
    3. 成功后切换 ControlledState 到 BlowOff。
    4. 调用 EnemyBlowOffComponent.ApplyBlowOff(source, dir, distance, height, totalTime, isPureBlowOff)。
    5. 触发事件，并同样会处理死亡延迟释放。

    BlowOffEnemy / Crush / Fracture¶

    类定义¶

    字段:

    字段 含义
    attackerTargetSettings 攻击方
    targetSettings 被击飞目标
    blowOffDistance 水平击飞距离
    distanceRandomRange 距离随机范围
    overwriteHeight 是否覆盖高度
    blowOffHeight 击飞高度
    directionSettings 方向计算
    totalTime 总击飞时间
    isExtra 是否额外控制
    deadOption 死亡目标处理

    CrushAction.Data 继承 BlowOffEnemyAction.Data，额外:

    字段 含义
    damageMultiplier 粉碎伤害倍率
    immobilizedTime 硬直时间

    FractureAction.Data 继承 BlowOffEnemyAction.Data，额外:

    字段 含义
    immobilizedTime 硬直时间

    执行链路¶

    方法 状态
    BlowOffEnemyAction.ExecuteInternal 已确认
    BlowOffEnemyAction.DoBlowOff 已确认
    BlowOffEnemyAction.Process 已确认

    DoBlowOff 的关键调用:

    1. AbilitySystem.BeforeControlledStateApplied(...)
    2. AbilityActionUtils.GetDirection(...)
    3. 从黑板读取 totalTime
    4. ControlledStateComponent.ApplyBlowOff(...)
    5. AbilitySystem.AfterControlledStateApplied(...)

    因此 BlowOffEnemyAction 系列是“事件包裹 + 方向/数值计算 + 受控状态组件”的组合。

    配置示例¶

    AKEDataMining/data/Json/BuffData/buff_common_blowoff.json:

    {
 "$type": "Beyond.Gameplay.Core.BlowOffEnemyAction+Data, Gameplay.Beyond",
 "blowOffDistance": { "value": 4.0 },
 "distanceRandomRange": { "value": 1.0 },
 "overwriteHeight": false,
 "blowOffHeight": { "value": 0.0 },
 "directionSettings": { "directionType": "SourceToTarget" },
 "totalTime": { "value": 0.0 },
 "isExtra": false,
 "deadOption": "AllValid"
}

    totalTime = 0 时一般表示使用底层组件/配置默认时间。distanceRandomRange 会让击飞距离产生随机浮动。

    BlowOffCharacter¶

    类定义¶

    字段:

    字段 含义
    attackerTargetSettings 攻击者
    targetSettings 被击飞角色
    verticalSpeed 垂直速度
    horizontalSpeed 水平速度
    blownPriority 击飞优先级
    preventSamePriorityReplacement 同优先级是否阻止替换
    directionSettings 击飞方向
    directionAngleOffset 方向角偏移
    turnTargetToAttacker 是否转向攻击者
    overrideSuperArmorLimit 覆盖霸体判定

    底层逻辑¶

    方法 状态
    BlowOffCharacterAction.ExecuteInternal 已确认
    CharacterBlowOffComponent.ApplyBlowOff 已确认

    CharacterBlowOffComponent.ApplyBlowOff 的关键逻辑:

    1. 触发 OnBeforeCharacterTakeBlowOff 类事件。
    2. 使用 CheckImmuneForCurrentSuperArmor 检查霸体免疫，overrideSuperArmorLimit <= 0 时使用默认值。
    3. 比较 blownPriority 和当前击飞优先级。
    4. 如果同优先级且 preventSamePriorityReplacement 为真，则拒绝替换。
    5. 规范化方向。方向接近零时使用默认前向。
    6. 如果 turnTargetToAttacker 为真，停止旋转并调用 BaseController.RawSetDirXZ(-direction)。
    7. 调用 MoveInput.Blown(horizontalSpeed, verticalSpeed, direction, attackerEntity)。
    8. 触发 OnAfterCharacterTakeBlowOff 类事件。
    9. 记录角色受击。

    这条链路和敌人控制不同，不切 ControlledStateComponent，而是直接驱动角色移动输入系统。

    配置示例¶

    AKEDataMining/data/Json/BuffData/buff_common_blowoffcharacter.json:

    {
 "$type": "Beyond.Gameplay.Core.BlowOffCharacterAction+Data, Gameplay.Beyond",
 "verticalSpeed": { "value": 6.0 },
 "horizontalSpeed": { "value": 6.0 },
 "blownPriority": { "value": 1 },
 "preventSamePriorityReplacement": false,
 "directionSettings": { "directionType": "SourceToTarget" },
 "turnTargetToAttacker": true,
 "overrideSuperArmorLimit": -1
}

    PullAction¶

    类定义¶

    字段:

    字段 含义
    targetSettings 拉拽目标
    pullSpeed 基础拉拽速度
    isInfinity 是否无限持续
    pullDuration 拉拽持续时间
    stopTolerance 到达容差
    unmovable 拉拽期间是否不可移动
    finishPullByAction 是否随 Action 结束停止
    destination 拉拽目标点
    pullTargetType Enemy / Character
    attenuationBySuperArmor 是否按霸体衰减
    attenuationValues 霸体值到衰减倍率列表

    底层逻辑¶

    方法 状态
    PullAction.ExecuteInternal 已确认
    PullAction.OnEnd 已确认
    PullComponent.ApplyPull 已确认
    PullComponent._DoPull 已确认
    PullRequest.GetMoveDelta 已确认
    PullRequest._CalcSpeed 已确认

    PullComponent.ApplyPull:

    1. 分配递增 requestId。
    2. 创建 PullRequest(requestId, owner, speed, requestType, destination, direction, stopTolerance, unmovable, duration, infinity, attenuationValues)。
    3. 加入 m_pullRequests。
    4. 检查/添加 Pull 标签。
    5. 触发 Pull 开始类事件。
    6. 返回 requestId，用于 finishPullByAction 时在 OnEnd 结束请求。

    PullComponent._DoPull:

    1. 若没有请求或目标不能移动则跳过。
    2. 取第一个 PullRequest。
    3. 调用 PullRequest.GetMoveDelta(deltaTime)。
    4. 将位移 delta 交给实体移动组件执行。

    PullRequest.GetMoveDelta:

    1. 如果距离小于 stopTolerance，请求结束。
    2. 调用 _CalcSpeed 得到实际速度。
    3. 返回本帧移动 delta。

    PullRequest._CalcSpeed:

    1. 默认使用 pullSpeed。
    2. 如果配置了衰减表，会取目标当前霸体值。
    3. 遍历 PullAttenuationValue，找到满足阈值的 attenuationFactor。
    4. 实际速度 = pullSpeed * attenuationFactor。

    配置示例¶

    AKEDataMining/data/Json/BuffData/buff_chr_0013_aglina_normal_skill.json:

    {
 "$type": "Beyond.Gameplay.Core.PullAction+Data, Gameplay.Beyond",
 "pullSpeed": 6.0,
 "isInfinity": false,
 "pullDuration": 3.1,
 "stopTolerance": 1.5,
 "unmovable": true,
 "finishPullByAction": true,
 "pullTargetType": "Enemy",
 "attenuationBySuperArmor": true,
 "attenuationValues": [
 { "superArmorValue": { "value": 20 }, "attenuationFactor": 0.5 },
 { "superArmorValue": { "value": 30 }, "attenuationFactor": 0.0 }
 ]
}

    含义: 霸体越高，拉拽越弱；到 30 这一档时拉拽速度变成 0。

    InterruptAction¶

    类定义¶

    字段:

    字段 含义
    attacker 打断来源
    defender 被打断目标
    overrideSuperArmorLimit 覆盖霸体阈值
    immobilizedTime 打断后硬直

    执行链路¶

    方法 状态
    InterruptAction.ExecuteInternal 已确认
    AbilitySystem.OnInterruptAction 已确认

    InterruptAction.ExecuteInternal:

    1. 取第一个 attacker。
    2. 取 defender 目标列表。
    3. 遍历 defender。
    4. 对每个目标调用 AbilitySystem.OnInterruptAction(attacker, immobilizedTime, overrideSuperArmorLimit)。

    AbilitySystem.OnInterruptAction:

    1. overrideSuperArmorLimit <= 0 时使用默认值。
    2. 调用 CheckImmuneForCurrentSuperArmor，若霸体免疫则返回 false。
    3. 若当前有技能，调用 Skill.Interrupt(reason=5, context)。
    4. 触发 OnInterruptActionSuccess 类事件。
    5. 如果配置了 immobilizedTime，添加 Buff:
    buff_common_immobilized_interrup

    并传入黑板参数:

    duration = immobilizedTime

    配置示例¶

    AKEDataMining/data/Json/BuffData/buff_common_mini_poise_break.json:

    {
 "$type": "Beyond.Gameplay.Core.InterruptAction+Data, Gameplay.Beyond",
 "attacker": { "targetSource": "Source" },
 "defender": { "targetSource": "Owner" },
 "overrideSuperArmorLimit": 100,
 "immobilizedTime": 4.0
}

    AddDynamicCcs / InheritCCS 重新分类¶

    AddDynamicCcsAction¶

    字段大量指向相机:

    • CinemachineBlendDefinition.Style
    • blendInTime
    • blendOutTime
    • additiveHorizontalAngle
    • additiveVerticalValue
    • additiveZoomScale
    • minZoom / maxZoom
    • fovCurve
    • screenYCurve
    • shoulderOffset
    • lookAtOffset

    因此这里的 CCS 是 Camera Control State。

    InheritCCSAction¶

    方法 状态
    InheritCCSAction.ExecuteInternal 已确认
    InheritCCSAction.OnEnd 已确认

    执行逻辑:

    1. 根据 ccsOwner 找到目标 AbilitySystem。
    2. 调用 AbilitySystem.TryGetInheritCCS(ccsKey, ..., out InheritCCSData)。
    3. 从 LevelCameraController.CameraControlStateMachine.GetOuterControlState(id) 取相机控制状态。
    4. 如果 overrideBlendOut，覆盖 CameraControlState 的 blend out 时间。
    5. OnEnd 时调用 AbilitySystem.RemoveInheritCCS(id) 和 GameAction.RemoveCameraControlState(m_ccs)。

    结论: AddDynamicCcsAction / InheritCCSAction 应移动到相机/镜头分类。

    建议更新分类¶

    建议将“击飞与控制”拆成以下子类:

    子类 Action
    敌人受控状态 AirborneAction, LaunchUpwardAction, KnockDownAction, TakeDownAction, BlowOffEnemyAction, CrushAction, FractureAction
    角色击飞 BlowOffCharacterAction
    物理位移控制 PullAction, PushBackAction, BlowOffAction
    技能打断 InterruptAction, InterruptCurSkillAction
    关键词减速 SlowAction
    特殊窗口/标记 EliteBackSwingBeHit
    应移出本类 AddDynamicCcsAction, InheritCCSAction

    其中 BlowOffAction 和 PushBackAction 都属于位移控制，但 BlowOffEnemyAction 系列属于敌人 ControlledState，BlowOffCharacterAction 属于角色 MoveInput 击飞，两者底层完全不同。
