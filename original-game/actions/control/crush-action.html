    CrushAction / BlowOffEnemyAction / FractureAction¶

    类继承结构¶

    AbilityAction<T>

    BlowOffEnemyAction — 父类¶

    Data 字段 (extends AbilityActionData)¶

    字段 类型 偏移 说明
    attackerTargetSettings TargetSettings 0x20 攻击者目标筛选
    targetSettings TargetSettings 0x28 被击飞目标筛选
    blowOffDistance BlackboardDouble 0x30 击飞距离
    distanceRandomRange BlackboardDouble 0x38 距离随机范围
    overwriteHeight bool 0x40 是否覆盖高度
    blowOffHeight BlackboardDouble 0x48 击飞高度
    directionSettings DirectionSettings 0x50 方向设置
    totalTime BlackboardDouble 0x58 总时间
    isExtra bool 0x60 是否额外击飞段
    deadOption ControlledStateDeadOption 0x64 死亡判定选项

    执行流程¶

    ExecuteInternal(TargetHandleView) → 已确认:

    1. 获取 attackerTargetSettings → 调用 GetFirstTarget() 获取攻击源（attacker）
    2. 类型检查攻击源是否为 enemy
    3. 获取 targetSettings → 调用 GetTargets_Dispose() 获取目标列表
    4. 遍历每个目标：
    5. Lock 目标获取 AbilitySystem
    6. 检查 deadOption：
    7. AllValid(0) → 通过
    8. OnlyAlive(1) → 检查目标是否 alive（0x4E type check），不是 alive 则跳过
    9. 解析 blowOffDistance（含 random range 随机化）
    10. 解析 blowOffHeight

    11. 调用 this.Process(inputTargetHandle, target, part, blowOffSource, distance, height) Process(...) → 已确认（虚方法，可被子类重写）:

    12. 默认行为：以 type=Crush(3)、damageMultiplier=1.0 调用 DoBlowOff()

    DoBlowOff(...) → 已确认:

    1. 调用 target.BeforeControlledStateApplied(source, part, type, isExtra, actionEnvironment, damageMultiplier)
    2. 获取 target.entity._controlledStateCom
    3. 通过 AbilityActionUtils::GetDirection() 计算方向向量
    4. 解析 data.totalTime 通过 blackboard
    5. 调用 controlledStateCom.ApplyBlowOff(source, direction, distance, height, totalTime)
    6. 调用 target.AfterControlledStateApplied(source, part, type, isExtra, actionEnvironment)

    CrushAction — 击碎（压碎）¶

    C# 类名: Beyond.Gameplay.Core.CrushAction

    Data 字段（继承 BlowOffEnemyAction.Data，额外字段）¶

    字段 类型 偏移 说明
    damageMultiplier BlackboardDouble 0x68 伤害倍率（通过 blackboard 解析）
    immobilizedTime float 0x70 定身时间（秒）

    Process 重写¶

    完整流程：

    flowchart TD
    Start["CrushAction 处理单个目标"]
    Interrupt["用 immobilizedTime 强制中断目标动作"]
    IsPart{"目标是敌人部位系统？"}
    Count["读取 buff_physical_no_guard 层数"]
    NoCount["按 0 层处理"]
    HasGuard{"层数 > 0？"}
    AddGuard["添加 buff_physical_no_guard"]
    Record["记录 PhysicalNoGuard"]
    Multiplier["从黑板解析 damageMultiplier"]
    BlowOff["进入 DoBlowOff，类型为 Crush"]
    End["完成 Crush 施加"]

    Start --> Interrupt --> IsPart
    IsPart -- "是" --> Count --> HasGuard
    IsPart -- "否" --> NoCount --> HasGuard
    HasGuard -- "否" --> AddGuard --> Record --> Multiplier
    HasGuard -- "是" --> Multiplier
    Multiplier --> BlowOff --> End

    关键逻辑要点¶

    1. 强制中断: overrideSuperArmorLimit = -1 意味着无视所有超级护甲等级，必定中断目标当前动作。中断后目标进入 immobilizedTime 的定身状态。
    2. 免疫机制: buff_physical_no_guard 是一个保护性 buff。如果目标已经拥有此 buff（通过之前的 Crush/Fracture 添加或其他来源），则不会重复添加，也不会再次记录事件，但仍会正常执行击飞和伤害。
    3. 物理无保护标记: 添加 buff_physical_no_guard 的目的是在伤害计算阶段标记此目标"已失去物理保护"，允许后续的物理穿透计算生效。
    4. 伤害倍率: 与基类不同，CrushAction 不总是用 1.0 的伤害倍率，而是通过 blackboard 动态解析 damageMultiplier，允许配置不同的击碎伤害倍率。

    FractureAction — 破碎（部位破坏）¶

    C# 类名: Beyond.Gameplay.Core.FractureAction

    Data 字段（继承 BlowOffEnemyAction.Data，额外字段）¶

    字段 类型 偏移 说明
    immobilizedTime float 0x68 定身时间（秒）

    与 CrushAction 的区别¶

    • 无 damageMultiplier: FractureAction 不包含伤害倍率字段，使用默认 1.0
    • 无 buff_physical_no_guard 添加: FractureAction 不添加免疫 buff
    • 其余流程类似：中断目标 → 计算方向 → 调用 DoBlowOff(type=Fracture(2))

    物理异常 Buff 映射¶

    GetPhysicalInflictionBuff(type) 返回对应类型的 buff:

    PhysicalInflictionType 对应 Buff
    Airborne (0) buff_physical_airborne
    KnockDown (1) buff_physical_knockdown
    Fracture (2) buff_physical_fracture
    Crush (3) buff_physical_crush

    AddBuffOption 详情¶

    在添加 buff_physical_no_guard 时，会传递 AddBuffOption:

    • ownerInfo 包含一个 Blackboard.DataPair(key="dmg_multiplier", value=damageMultiplier)
    • 这意味着物理无保护 buff 会携带伤害倍率信息
