    AbilitySystem.ShieldController¶

    更新时间：2026-06-11

    归属¶

    ShieldController 是 AbilitySystem 的嵌套 SubFeature。它持有 shield instance priority queue，负责增删护盾并把伤害吸收委托给每个 ShieldInst.TryAbsorbDamage。

    字段¶

    字段 offset 说明
    m_shieldInsts 0x20 PriorityQueue<ObjectPtr<ShieldInst>>，按 priority 管理 shield instance。

    方法¶

    方法 状态 状态 结论
    get_remainingShieldValue() 已确认 已定位 返回当前剩余 shield value；细节待展开。
    OnInit() 已确认 已定位 初始化 sub feature。
    Start() 已确认 已定位 sub feature start。
    Tick(allScaledTime, timeManagerDeltaTime, unscaledDeltaTime) 已确认 已定位 shield tick；细节待展开。
    Finish() 已确认 已定位 sub feature finish。
    Release() 已确认 已定位 sub feature release。
    AbsorbDamage(DamageType, double, BATTLE_DAMAGE_DETAIL, out double, out EffectActionCfg) 已确认 已确认 初始 afterValue = inputValue、overrideHitEffect = null；若 queue 非空，从 count - 1 倒序遍历，lock shield 后调用 ShieldInst.TryAbsorbDamage，并用上一轮 afterValue 作为下一轮输入。
    ShieldInst._RefreshConsume() 已确认 已确认 TryAbsorbDamage 消耗后调用；根据剩余值/次数刷新 consumed 状态，并在配置要求时进入 buff finish path。
    ShieldInst.get_infinityValue() 已确认 已确认 TryAbsorbDamage 中用于判断无限盾值分支。
    ShieldInst.get_m_buff() 已确认 已确认 TryAbsorbDamage 中用于取得 source buff，消耗后可走 Buff.MarkFinish。

    调用入口¶

    • AbilitySystem.AbsorbDamage(...) 对外层 damage path 暴露吸收能力，内部转入 ShieldController.AbsorbDamage。
    • buff/action 生成 ShieldInst 后调用 ShieldController.AddShield。
    • shield handle、buff finish 或消耗完毕时调用 ShieldController.RemoveShield 或 ShieldInst 内部 finish path。

    生命周期¶

    1. AbilitySystem 初始化时创建 ShieldController。
    2. Buff/action 创建 ShieldInst，加入 m_shieldInsts priority queue。
    3. 受到伤害时，ShieldController 初始化输出值，然后倒序遍历 queue，逐个调用 ShieldInst.TryAbsorbDamage。
    4. ShieldInst.TryAbsorbDamage 根据 damage type、剩余值、剩余次数等更新 shield，并触发 shield event / buff finish 副作用。
    5. shield 被 handle 移除或消耗完毕时从 queue 移除。

    已确认行为¶

    • ShieldController 不直接计算所有吸收细节；具体吸收、消耗刷新、event、buff finish 由 ShieldInst.TryAbsorbDamage 承担。
    • AbsorbDamage 会把 afterValue 初始化为输入伤害，overrideHitEffect 初始化为空。
    • TryAbsorbDamage 计算 absorbed / afterValue 后会更新剩余 shield value（约 [rdi+0x30]）和 absorb count（约 [rdi+0x3C]），再调用 _RefreshConsume。
    • TryAbsorbDamage 会向 BATTLE_DAMAGE_DETAIL 写入 DAMAGE_ABSORB_INFO，并通过 NeedTriggerEvent / TriggerEvent 触发 shield block 相关 AbilitySystem event（观察到事件值 0x94）。
    • 如果 shield 消耗且配置要求移除来源 buff，会调用 Buff.MarkFinish(... FinishReason 100 ...)。
