    AbilitySystem¶

    更新时间：2026-06-15

    字段¶

    字段 offset 说明
    m_attributes 0x148 属性系统。
    m_hp 0x150 HP synced value。
    m_ap 0x168 AP。
    m_ultimateSp 0x16C 终结技能量。
    m_onDieAction 0x170 死亡回调 MultiAction<AbilitySystem>。
    m_timeManager 0x1F0 TimeManager 引用。
    m_skillDataBundle 0x228 当前技能配置 bundle。
    m_skills 0x230 所有 skill runtime。
    m_activeSkillMap 0x238 当前可用 skillId 到 Skill 的映射。
    m_isCastingSkillStart 0x240 cast start 状态。
    _curNormalAttackList 0x288 当前普攻链表。
    _curNormalSkill 0x298 当前战技。
    _curComboSkill 0x2A0 当前连携技。
    _curUltimateSkill 0x2B0 当前终结技。
    _dodgeSkill 0x2C8 闪避技能。
    alreadySendDieOp 0x2D3 BattleManager.SendEntityDie 发送死亡 op 后置位。
    _curSkill 0x338 当前技能。
    m_curProcessingSkill 0x340 当前处理中的技能。
    m_buffContainer 0x358 buff container。
    m_comboController 0x3A0 嵌套 AbilitySystem.ComboController 子功能。
    m_mappingHandles 0x400 command mapping handle 列表。
    m_timedMarkers 0x3D8 timed marker 列表；详见 ../buff-globalbuff/timed-marker.md。
    m_serverCheckDeath 0x458 enemy 死亡由 server 校验时，本地 _Die 发送 op 后返回。
    m_damageModifiers 等 0x460+ damage/heal/poise modifier。
    m_poise / m_poiseController 0x4C0 / 0x4C8 poise。
    m_subFeatures 0x4F0 sub feature 列表，包含 ComboController。

    方法¶

    方法 状态 状态 结论
    _InitSkills() 已确认 已确认 读取 m_skillDataBundle，写当前技能槽字段，按 allNormalAttackId -> allActiveSkillId -> allPassiveSkillId 顺序调用 _CreateSkill；normal/active skill allowFrameSplitInit=true，passive skill allowFrameSplitInit=false；passive 创建后加入 m_defaultPassiveSkills。
    TryCastSkill(SkillCastInputView, options) 已确认 已定位 overload 入口，最终进入 string skillId overload。
    _FrameSplitInitSkills() 已确认 已确认 若 m_pendingInitSkills 非空且 BattleManager.CanSplitFrameInitSkill() 为 true，则初始化第一个 pending skill 并 RemoveAt(0)。
    DetachSkill(String skillId) 已确认 已确认 空 skillId 返回 false；从 m_skills 尾部向前查找 skillId；若 _curSkill 为空立即 _DetachSkillInternal，否则把 detach action 加入 m_modifySkillListAction，cast end 后执行；函数找到与否之外主路径返回 true。
    _DetachSkillInternal(Skill skill) 已确认 已确认 从 m_skills 和 m_pendingInitSkills 移除；当 skill.m_data.castType == Active(0) 时从 m_activeSkillMap 移除；随后调用 Skill.Disable() 和 Skill.Remove()。
    TryGetSkill(String, out Skill) 已确认 已确认 out 参数先置 null；顺序扫描 m_skills，比较 Skill.skillId，命中后写 out 并返回 true；不查询 m_activeSkillMap。
    SetHpDirectly(double targetHp, bool skipModifier) 已确认 已确认 主路径先做对象有效性检查；skipModifier=false 时计算 targetHp - currentHp，正数构造 Modifier.NewHeal(godAbility,this,abs(delta))，否则构造 Modifier.NewDamage(godAbility,this,abs(delta), Real, None, Hidden)，随后 Modifier.Apply() 并创建 independent entity value op；skipModifier=true 时进入直接 HP 写入/上报路径，不走 heal/damage modifier。
    CharInfo.TryGetServerSkillData(UInt64 instanceId, out SkillData) 已确认 已确认 若 skill data 未初始化则 InitSkillData(false)；随后从 this+0x110 的 Dictionary<ulong, SkillData> 用 instanceId TryGetValue。
    CanCastSkill(SkillCastInputView, skipAngle, skipDistance) 已确认 已定位 overload 入口。
    RefreshNextSkillRequest(cmdType) 已确认 已定位 thin/bridge 方法，转到 m_comboController.RefreshNextSkillRequest 语义，细节在 ComboController 文档。
    AddMappingModifier(...) 已确认 已定位 AbilitySystem 对 ComboController mapping modifier 的 façade。
    ComboControllerOnSkillCastStart(skillId) 已确认 已定位 AbilitySystem 对 ComboController OnSkillCastStart 的 façade。
    OnSkillExclusiveTimeExecuted(skillId) 已确认 已定位 AbilitySystem 对 ComboController exclusive callback 的 façade。
    OnHpZero() 已确认 已确认 若 NeedTriggerEvent(OnOwnerHpZero=1)，构造 EventContext.eventSource=this 并 TriggerEvent；随后调用 _Die()。
    _Die() 已确认 已确认 按 owner Entity.objectType 分支：Character/Enemy 调 BattleManager.SendEntityDie(..., flush=true)；Enemy 在 m_serverCheckDeath 为 true 时返回；Projectile/AbilityEntity 调 SendEntityDie(..., flush=false)；Interactive 直接返回；其余继续 Entity.Die()。
    OnDie() 已确认 已确认 Character owner 时遍历当前 squad slot，向每个成员 AbilitySystem 触发 OnSquadMemberDie(67)，context.eventSource 为死亡者；随后 m_onDieAction.Invoke(this)，再 Clear()，最后调用 BaseComponent.OnDie()。

    调用入口¶

    • Beyond.Gameplay.Core.Entity.abilityCom 持有实体的 AbilitySystem；AbilitySystem 不是独立全局系统，而是附着在统一 Entity 上的 TickComponent 战斗 owner。
    • Player command / AI / action 会以 skillId 或 command request 进入 AbilitySystem.TryCastSkill。
    • AbilitySystem.ComboController 负责把 command 映射为 next skill request。
    • Skill 的 cast/interrupt/end 由 AbilitySystem 统一调度。
    • AbilitySystem.m_buffContainer 持有 Buff.BuffContainer；实体 buff、stacking group、buff tick/finish/dispel/ignite 入口应从这里进入。

    生命周期¶

    1. 初始化时创建 Skill 列表和 active skill map，并创建 ComboController sub feature。
    2. 初始化时创建 m_buffContainer，实体 buff runtime 由该 container 管理。
    3. _InitSkills 从 SkillDataBundle 初始化当前技能槽，并创建 normal/active/passive skill runtime。
    4. Player command 或 action 请求 cast 时，进入 AbilitySystem.TryCastSkill。
    5. TryCastSkill 在 active map 命中后调用 _InitSkillIfNot(skillId)，保证待释放 skill 已初始化并离开 pending-init 队列。
    6. CanCastSkill / CheckCanInterruptCurSkill 决定当前技能是否可被新技能替换。
    7. 若可 cast 且当前有技能，TryCastSkill 调用旧技能 Interrupt(CastNextSkill, context)。
    8. cast 过程中请求 detach skill 时，DetachSkill 会把实际 detach 延迟到当前 cast end 后执行。
    9. PreLateTick / tick 路径推进当前 skill、buff、action、timed marker 等。
    10. HP 归零进入 OnHpZero；OnOwnerHpZero event 先于 _Die。
    11. _Die 按 Entity.objectType 发送死亡 op / 继续或跳过本地 Entity.Die。
    12. Entity.Die 经 ComponentContainer.OnEntityDie 触发组件死亡；AbilitySystem 组件的 OnDie 负责 squad member death event 和 m_onDieAction 回调。

    连携技 UI / pending window 路径：

    1. BattleManager.CastPendingComboSkill(charIndex) 解析 slot ability 和 pending ComboSkillCastInfo。
    2. BattleManager._TryCastComboSkill(ability, comboCastInfo) 取 comboCastInfo+0x10/+0x18 的 input/trigger target handle view，并以 onCastEnd=null 调 AbilitySystem.TryCastComboSkill。
    3. AbilitySystem.TryCastComboSkill 读取 _curComboSkill，保存目标 handle，最终调用 PlayerController.CastSkill(..., immediatelyActivate=true, triggerAutoLock=true)。

    4. 真实 cast/interrupt/cost/CD 仍回到通用 AbilitySystem.TryCastSkill / Skill.DoCast 生命周期；不应给连携技写独立打断规则。

    5. 若 objectType == 8，直接返回 false。

    6. 调 _InitSkillIfNot(skillId)，再从 m_activeSkillMap[skillId] 取 newSkill。
    7. 调 CheckCanInterruptCurSkill(newSkill)；失败返回 false。
    8. 从 CastSkillOptions nullable 中解析 skipDistance / skipAngle，调 newSkill.CanCastToTarget(inputTarget, skipDistance, skipAngle)；失败返回 false。
    9. 调 newSkill.TrySwitchToAddBuff(inputTarget)；若成功，直接返回 true。
    10. 设置 m_isCastingSkillStart = true。
    11. 写 _curSkill = newSkill。这一步发生在 Skill.DoCast 之前。
    12. 对 newSkill.ability 调 Ability.AssignSkillCastInfo(skillCastInfo)。
    13. 调 newSkill.SaveTarget(inputTarget, options)。
    14. 取 newSkill.targetHandle，调 AbilitySystem.BeforeCastStart(newSkill, castTarget)。
    15. 对角色 objectType 的部分非闪避/非步进技能，若技能 ability 的某标志未开启，则调 ClearOffsetModifier(clearOffsetSkillId = true)。
    16. 调 newSkill.DoCast(onCastEndDelegate, onExclusiveTimeCallback, options)。
    17. 若 afterCastStartCallback 非空，立即调用 afterCastStartCallback(newSkill)。
    18. 调 newSkill.OnTick(0, 0)。
    19. 设置 m_isCastingSkillStart = false，返回 true。

    _OnCastEnd 顺序¶

    1. _OnCastEnd 检查 NeedTriggerEvent(OnSkillEnd=31)。
    2. 若需要触发事件，从当前 _curSkill 构造 CastSkillContext.AssignFromSkill(_curSkill)，调用 TriggerEvent(OnSkillEnd, context)。
    3. 若当前 _curSkill.skillType == 7，发送预定义全局事件，payload 是 owner entity 与 skill。
    4. 清 _curSkill。

    CancelCurrentSkillCastEndCallback 状态 已确认 只把 m_invokeCastEndCallback 设为 false，因此它只屏蔽外部 callback，不屏蔽 OnSkillEnd、_curSkill 清理或 _OnCastEnd 的其他副作用。

    已确认：

    1. _InitSkills 先写 _curBreakingAttackList、_curNormalAttackList、_curNormalSkill、_curComboSkill、_curUltimateSkill、_plungingAttackStart、_plungingAttackEnd、_dodgeSkill，再清 _curExtraPassiveSkill / _curSkill。
    2. _InitSkills 遍历 allNormalAttackId，用 allowFrameSplitInit=true 创建 skill；plunging start/end 设置 source 7，enabled breaking normal attack 设置 source 8。
    3. _InitSkills 遍历 allActiveSkillId，用 allowFrameSplitInit=true 创建 skill；normal skill type 2，dodge 5，combo 6，ultimate 7。
    4. _InitSkills 清 m_defaultPassiveSkills 后遍历 allPassiveSkillId，用 skillType/source -1、allowFrameSplitInit=false 创建 skill；创建成功后加入 m_defaultPassiveSkills。
    5. _CreateSkill 先查 m_activeSkillMap.TryGetValue(skillId)；重复 active skill 时记录 “当前技能已经存在” 并返回已有 skill。
    6. _CreateSkill 创建 SkillData，角色 owner 分支优先走 CharInfo.TryGetActiveSkillData / TryGetServerSkillData，失败才用 SkillUtil.CreateSkillDataWithCharInfo。
    7. _CreateSkill 调 Skill.Create(...) 后把 skill 加入 m_skills；只有 skillData.castType == Active(0) 时加入 m_activeSkillMap。
    8. Skill.Create 调 _OnCreate 后，如果 allowFrameSplitInit && skillData.castType != Passive(1)，调用 AbilitySystem.AddSkillToFrameSplitInitList(skill)；否则立即调用 Skill._Init()。
    9. _InitSkillIfNot 读取 m_activeSkillMap[skillId]。skill 存在且内部 init flag 未置位时调用 Skill.Init()，随后从 m_pendingInitSkills 移除该 skill。
    10. _FrameSplitInitSkills 每次只处理 m_pendingInitSkills[0]；只有 BattleManager.CanSplitFrameInitSkill() 返回 true 时才初始化并 RemoveAt(0)。
    11. DetachSkill 空 skillId 直接返回 false；否则从 m_skills 尾部向前找第一个 matching skillId。若 _curSkill 不存在，立即 detach；若存在当前技能，则把 detach 操作写入 m_modifySkillListAction，避免 cast 过程中修改 skill list。
    12. _DetachSkillInternal 会移除 m_skills、m_pendingInitSkills、当 skill data castType == Active(0) 时移除 active map，然后调用 Skill.Disable() 和 Skill.Remove()。
    13. SetHpDirectly(targetHp, skipModifier=false) 并不是直接写 m_hp：它根据目标 HP 和当前 HP 差值构造 heal/damage modifier，source 为 god entity 的 AbilitySystem，target 为当前 AbilitySystem，再走 Modifier.Apply()。只有 skipModifier=true 分支进入直接写值/上报语义。
    14. CharInfo.TryGetServerSkillData 的 server skill 数据保存在 CharInfo+0x110 字典，缺初始化时会先调用 InitSkillData(false)。

    GameEvents.CastSkill 消费者¶

    已确认原游戏全局事件名：

    已确认监听者：

    • Beyond.Gameplay.AI.GroupAISystem.OnInit 状态 已确认：
    • EventManager.get_gameEvents()
    • 创建 EventDelegateForStruct<EventData<Skill>>
    • handler 指向 GroupAISystem._RefreshLastPlayerCastSkill
    • GroupAISystem.OnRelease 状态 已确认：
    • 使用相同 handler 和 GameEvents.CastSkill 执行 UnBind。
    • GroupAISystem._RefreshLastPlayerCastSkill 状态 已确认：
    • 读取 payload skill。
    • 检查 skill.owner.objectType == 8。
    • 检查 skill.skillType == NormalAttack(1)；Skill.skillType 字段 offset 为 0xD8。
    • 检查 owner entity 的 CharacterAIComponent.isCenter == true。
    • 满足条件后写 m_lastPlayerCastSkillTime = TimeManager.time。

    已确认行为¶

    • 通用打断规则由 AbilitySystem.CheckCanInterruptCurSkill 集中处理，不应为连携技写独立显式打断规则。
    • TryCastComboSkill 不是独立打断实现；它解析当前 _curComboSkill 和 pending target，然后通过 player/skill cast path 进入通用 cast 生命周期。
    • allowed-next skill pack 优先于 priority，可以直接允许下一技能。
    • priority 比较是 newSkill.priority > curSkill.priority，相等不通过 priority 分支。
    • 如果 priority 不够，最终看当前技能 canInterrupt。
    • TryCastSkill 会在 Skill.DoCast 前写 _curSkill = newSkill，因此 cost、tag、cast-start 期间的 side effect 应能观察到新当前技能。
    • TryCastSkill 在 active map 命中后会执行 _InitSkillIfNot(skillId)。
    • _CreateSkill 会把 passive skill 保留在 m_skills，但不会加入 m_activeSkillMap。
    • TryGetSkill(String, out Skill) 状态 已确认 不查询 m_activeSkillMap；它把 out 参数置空后，从 index 0 开始顺序扫描 m_skills offset 0x230，比较 Skill.skillId offset 0xE8，命中后写 out 参数并返回 true。
    • Skill.Create 的 allowFrameSplitInit 只对非 passive skill 生效；passive skill 立即 _Init()。
    • TryCastSkill 在 DoCast 后立即调用 afterCastStartCallback(newSkill)，然后执行一次 Skill.OnTick(0,0)，再派发 GameEvents.CastSkill。
    • OnSkillEnd 由 AbilitySystem._OnCastEnd 触发；事件触发时 _curSkill 仍未清空，事件后再清 _curSkill，外部 callback 在 _OnCastEnd 之后调用。
    • CancelCurrentSkillCastEndCallback 不取消 _OnCastEnd，只取消外部 cast-end callback。
    • Skill.OnTick(0,0) 不是空转；它会进入 TimelineActionProcessor._TickInternal(0,0)，执行 startTime/frame 0 timeline actions。
    • DetachSkill 在当前技能存在时延迟修改 skill list，避免 cast 栈内修改 m_skills / m_activeSkillMap。
    • AbilitySystem.CompareTo 委托 owner Entity.CompareTo。
