    AbilitySystem.TacticalItem¶

    更新时间：2026-06-17

    归属¶

    TacticalItem 是 AbilitySystem 里的可复用战术物品 runtime。它持有装备数据、owner、条件检查器、等待施放/施放/冷却 timer，并响应入战状态、主控切换和 ability event。

    字段¶

    字段 offset 说明
    m_data 0x10 ItemEquipData。
    m_owner 0x28 owner AbilitySystem。
    m_conditionChecker 0x30 使用条件检查器。
    m_castInput 0x38 target slot / 是否对主控释放。
    m_isWaitingToCast 0x48 等待施放状态。
    m_waitToCastTimer 0x50 等待施放 timer。
    m_isCasting 0x58 正在施放状态。
    m_castTimer 0x60 施放 timer。
    m_castAudio 0x68 施放音频。
    m_holdingScope 0x6C holding scope。
    m_isCoolingDown 0x70 冷却状态。
    m_cooldownTimer 0x78 冷却 timer。
    m_isGuardAICloseToTarget 0x80 guard AI close-to-target 状态。
    m_guardAICloseToTargetTimer 0x88 guard AI close-to-target timer。
    instanceUid 0x90 reusable object uid。

    嵌套类型¶

    类型 字段/值 说明
    CastInput targetSlot, isToMainCharCast 延迟施放使用的输入。
    ConditionChecker itemId, useTargetType, condType, damageType, compareType, value, tag 战术物品条件检查器。
    ConditionChecker.DamageType Physical=0, Energy=1, Any=2 条件使用的伤害类型分类。
    ConditionChecker.CompareType LT/LE/GT/GE/EQ/NE 条件数值比较类型。

    方法¶

    方法 状态 状态 结论
    Init(ItemEquipData, AbilitySystem) 已确认 已确认 被 TacticalItemController.RefreshTacticalItem 调用，用装备数据初始化 runtime 并绑定 owner。
    Release() 已确认 已定位 release reusable runtime。
    OnTick(float deltaTime) 已确认 已确认 被 TacticalItemController.Tick 以 unscaledDeltaTime 调用；按 casting / cooldown / waiting / guard AI close-to-target 状态推进 timer。
    StopUse() 已确认 已确认 停止使用：casting 时释放 Inventory hold、停止 cast audio，然后清 cast input、waiting/casting 标记并 reset wait/cast timer。
    _TryCast(Squad.Slot, bool) 已确认 已确认 保存 CastInput；ItemEquipData.condType == OnDie(4) 时读取 GameplayMiscSetting.tacticalItemOnDieDelayCastTime (+0xB0) 重置 wait timer；非 owner 目标走一次零周期 wait timer；目标是 owner 自己时直接 _Cast。
    _Cast() 已确认 已确认 校验 squad slot、目标 alive/state、target cast cd，设置 casting/timer，持有 inventory tactical item，启动 target-cd timer，必要时触发 guard AI close-to-target。condType == OnDie 分支跳过普通 alive gate，允许对死亡目标进入 cast。若 target cast CD gate 失败，native 回到 _OnEventTrigger / CheckAndTryToUse 重试，而不是静默失败。
    _OnCastFinish() 已确认 已确认 释放 held item/audio；根据 target 选择调用 BattleManager.UseTacticalItem(sourceSlot,targetSlot,isToMainCharCast)，成功后启动 cooldown timer，否则重新 CheckAndTryToUse。
    ConditionChecker.Check(...) 已确认 已确认 若 useTargetType == Target 先查 BattleManager.CheckTargetIsNotInTacticalItemCastCd；condType == OnDie 走死亡/复活列表分支；否则转 _CheckAliveTarget。
    ConditionChecker._CheckAliveTarget(...) 已确认 已确认 condType switch：HP%、USP%、未持有 buff、受伤比例、死亡、法术异常/tag；详见下文。
    ConditionChecker._CompareValue(...) 已确认 已确认 LT/LE/GT/GE/EQ/NE，比较时使用 epsilon 容差。

    调用入口¶

    • TacticalItemController.RefreshTacticalItem 查找 ItemEquipData 后创建/初始化 TacticalItem。
    • TacticalItemController.Tick 调用 TacticalItem.OnTick(deltaTime)。
    • TacticalItemController.CheckAndTryToUseTacticalItem 转发到 TacticalItem.CheckAndTryToUse。

    生命周期¶

    1. controller refresh 时载入 item equip data 并 Init。
    2. 入战、主控切换或 ability event 触发条件检查。
    3. 条件满足后进入等待施放或施放状态。
    4. OnTick 推进 wait/cast/cooldown timers。
    5. cast 完成后 _OnCastFinish 更新 cooldown 或 cleanup。
    6. release/recycle 清理 runtime。

    已确认行为¶

    • TacticalItem 不是简单 buff 应用；它有 condition checker、wait/cast/cooldown 三组 timer 和事件触发入口。
    • OnInFightChanged(true) 和 OnMainCharacterChange() 都是自动检查入口；前者只在入战时调用 CheckAndTryToUse()，后者主控切换时直接调用。
    • TacticalItemController.Tick 直接调用 TacticalItem.OnTick，说明 timer 推进由 TacticalItem 自身负责。
    • RefreshTacticalItem 会查 ItemEquipData 并调用 TacticalItem.Init(data, owner)。
    • _OnEventTrigger 主流程：
    • 查询当前 game mode，若 BaseGameMode.IsItemForbidden(itemId) 为 true 则返回。
    • 战斗中或 autoCheckNotInFight 分支通过后，解析 owner 所在 squad slot，先检查 BattleManager.CheckTacticalItemRemainingUseCount(slotIndex)。
    • useTargetType == Target 时还会检查 BattleManager.CheckTargetIsNotInTacticalItemCastCd(targetSlot.index)。
    • 检查 m_isCoolingDown、m_isCasting、m_isWaitingToCast 三个状态，任一为 true 都不触发新 cast。
    • 读取当前 scope 下 InventorySystem.GetAvailableTacticalItemCount(scope,itemId,userInstId)，库存为 0 时返回。
    • owner 模型不可见时返回。
    • useTargetType == Self / Target / 特殊 main-char target 分支都会先调用 ConditionChecker.Check，满足后调用 _TryCast。
    • ConditionChecker._CheckAliveTarget 分支：
    • CheckHpPercent：currentHp / maxHp 与 m_value 按 m_compareType 比较。
    • CheckUspPercent：ultimateSp / maxUltimateSp 与 m_value 按 m_compareType 比较。
    • CheckNotHasBuff：读取 UseItemTable[itemId].useActions，遍历 action 中 buffBBData.buffId，目标 buff container 任一已存在则失败；全部不存在则成功。
    • OnTakeDamageOverPercent：要求 EventContext 可转 TakeDamageContext；按 m_damageType 过滤物理/能量/任意伤害，再用伤害值占 max HP 比例与 m_value 比较。
    • OnDie：要求目标 entity/runtime 已死亡，同时 CharBagSystem.GetCharInfo(charInstId).isDead == true，并且 BattleManager.inReviveCharInstList 不包含该 charInstId；该分支防止同一死亡目标被多个 revive tactical item 重复处理。
    • _CompareValue 的 compareType：
    • LT(0)：lhs < rhs - epsilon
    • LE(1)：lhs <= rhs + epsilon
    • GT(2)：lhs > rhs + epsilon
    • GE(3)：lhs >= rhs - epsilon
    • EQ(4)：abs(lhs - rhs) <= epsilon
    • NE(5)：abs(lhs - rhs) > epsilon
    • _TryCast 不直接扣库存或应用效果；它只决定立刻 _Cast 还是进入 wait timer。condType == OnDie 读取 GameplayMiscSetting + 0xB0 的 tacticalItemOnDieDelayCastTime，构造默认值为 2.0f；非 owner 目标使用零周期 wait timer；owner 自身目标直接 _Cast。
    • _OnCastFinish 成功调用 BattleManager.UseTacticalItem 后才启动 m_cooldownTimer，说明 cooldown 是成功使用后的本体 cd，不是 _TryCast 时预扣。
    • OnTick 的 native 顺序：
    • 若 m_isCasting == true，tick m_castTimer；timer 完成后调用 _OnCastFinish()。
    • 若 m_isCoolingDown == true，tick m_cooldownTimer；完成后清 m_isCoolingDown 并重新调用 _OnEventTrigger(default EventContext)，也就是 cooldown 结束会立刻重试自动使用条件。
    • 若 m_isWaitingToCast == true，tick m_waitToCastTimer；完成后调用 _Cast()。
    • 若 m_isGuardAICloseToTarget == true，tick m_guardAICloseToTargetTimer；完成后清标记并触发 guard AI close-to-target 相关 AI event。
    • StopUse 的 native 行为：
    • 有 m_castAudio 时调用 audio stop helper，并传 fade/stop 参数 100。
    • 将 m_castInput 重置为空，清 m_isWaitingToCast 和 m_isCasting。
    • reset m_waitToCastTimer 与 m_castTimer，并清 m_castAudio。
    • 数量、目标 cast cd、实际库存扣除和 BattleRecorder.RecordUseTacticalItem 都归 BattleManager / InventorySystem，不归 TacticalItem。
