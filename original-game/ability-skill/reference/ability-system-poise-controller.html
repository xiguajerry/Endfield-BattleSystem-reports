    AbilitySystem.PoiseController¶

    更新时间：2026-06-17

    归属¶

    PoiseController 是 AbilitySystem 的嵌套 SubFeature。原游戏把 poise 值、poise recover/broken timer、破韧期间 buff/tag，以及被攻击时的破韧相关副作用放在该子功能中，而不是放在独立 EntityCharacter / EntityEnemy 运行时类里。

    字段¶

    字段 offset 说明
    m_poiseRecoverTimer 0x20 poise recover timer。
    m_poiseBrokenEndTimer 0x28 poise broken end timer。
    m_poiseKnotBreakImmobilizeTimer 0x30 poise knot break immobilize timer。
    m_buffsDuringPoiseExist 0x38 poise 存在期间附加/维持的 buff 列表。
    m_buffsDuringZeroPoise 0x40 poise 为 0 期间附加/维持的 buff 列表。
    m_poiseBreakingAttackedBuff 0x48 breaking attacked buff。
    m_poiseBrokenTagHandle 0x58 poise broken tag handle。
    m_poiseKnotBreakTagHandle 0x68 poise knot break tag handle。
    m_needClearPoiseBrokenStartTrigger 0x78 poise broken start trigger 清理标记。
    m_poiseKnotPctList 0x80 poise knot 百分比列表。
    m_poiseKnotBuffList 0x88 poise knot buff id 列表。
    m_nextKnotIndex 0x90 下一 knot index。
    m_poiseGuardTime 0x94 poise guard time nullable。

    方法¶

    方法 状态 状态 结论
    OnInit() 已确认 已确认 将当前 controller 写入 owner AbilitySystem.m_poiseController。
    Start() 已确认 已定位 sub feature start。
    Tick(allScaledTime, timeManagerDeltaTime, unscaledDeltaTime) 已确认 已确认 m_poiseRecoverTimer.periodTime > eps 时用 timeManagerDeltaTime 推进 recover timer；owner entity 有 PredefinedTag.StopPoiseRecover (Status/StopPoiseRecover) 时跳过 recover timer update；timer ready 后调用 ResetPoise(false)。m_needClearPoiseBrokenStartTrigger && passedTime > 0.1 时清视觉 trigger 标记。recover timer 无效且 m_poiseGuardTime.HasValue 时累加 guard time。m_poiseBrokenEndTimer / m_poiseKnotBreakImmobilizeTimer ready 后 MarkInvalid 并分别 RemoveTag。
    Finish() 已确认 已定位 sub feature finish。
    Release() 已确认 已定位 sub feature release。
    ModifyTransferredPoise(ref Modifier, bool triggerTakeDamageEvent) 已确认 已定位 transferred poise 修改入口，细节待展开。
    _RemoveBuffsDuringPoiseExist() 已确认 已确认 汇编确认读取 this+0x38 的 m_buffsDuringPoiseExist，用 CollectionExtensions.SafeCount 遍历，ObjectPtr<Buff>.Lock() 后调用 owner AbilitySystem.FinishBuff(buff, ServerActionParams)，随后 List.Clear()。
    ResetPoise(bool isInit) 已确认 已确认 重置 poise；m_poiseRecoverTimer.MarkInvalid()；若 AbilitySystemData.poiseBrokenEndTime > eps，m_poiseBrokenEndTimer.Reset(poiseBrokenEndTime,true)，否则移除 m_poiseBrokenTagHandle；重新应用 buffDuringPoiseExist；清理 breaking attack / zero-poise buff；清理 knot immobilize timer/tag；若 m_poiseGuardTime.HasValue，按 enemy rank/subRank 选择 SkillSetting.PoiseGuardConfig：rank 0 -> mob，rank 1 -> boss，rank 2 且 EnemyTemplateData.subRank == 2 -> elite-two，rank 2 其它值 -> elite-one；用 elapsed guard time 评估 curve，结果 < 1.0 时添加 buff_common_poise_guard 并写入 blackboard duration / poiseTakenScalar，随后清空 m_poiseGuardTime；触发 recover event/统计/LevelEvent/server op。
    _RemovePoiseBreakBuff() 已确认 已确认 汇编确认读取 this+0x40 的 m_buffsDuringZeroPoise，用 CollectionExtensions.SafeCount 遍历，ObjectPtr<Buff>.Lock() 后调用 owner AbilitySystem.FinishBuff(buff, ServerActionParams)，随后 List.Clear()。
    OnAttacked(AbilitySystem attacker) 已确认 已确认 AbilitySystem._OnAttacked callsite 已确认 传入 modifier source。native 路径要求 attacker 不是 owner、m_poiseGuardTime 为空、recover timer 未 valid、owner 有 poise，且 enemy part 未委托 main-body poise；满足时写 m_poiseGuardTime = 0.0f。
    get_inPoiseRecovery() 已确认 已确认 若 m_poiseRecoverTimer.periodTime > 1e-5，返回 !m_poiseRecoverTimer.isReady；否则 false。
    get_poiseRecoveryProgress() 已确认 已确认 若 m_poiseRecoverTimer.periodTime > 1e-5，返回 m_poiseRecoverTimer.progress；否则 0.0。

    调用入口¶

    • AbilitySystem 持有 m_poise 和 m_poiseController。
    • damage/poise modifier 链路进入 PoiseController.ModifyPoise。
    • ModifyPoise 在达到 zero poise 条件时进入 OnPoiseZero。
    • OnPoiseZero 通过 GameWorld.timeDilationManager 调用 StartPoiseBreakTimeStop(ownerAbilitySystem)；这是 PoiseBreak TimeDilation 的原游戏上游触发链。
    • timer/tick 由 AbilitySystem sub feature tick 驱动。

    生命周期¶

    1. AbilitySystem 初始化时创建并初始化 PoiseController。
    2. 受到 poise damage 时，通过 modifier 进入 ModifyPoise。
    3. poise 到 0 时触发 OnPoiseZero，处理 event、buff/tag、timer 和统计等副作用。
    4. Tick 推进 recover/broken/knot timers，并在到期时清理 tag/buff 或恢复状态。
    5. ResetPoise 可由初始化、AI response 或 gameplay 逻辑触发，重置 poise 与相关副作用。

    已确认行为¶

    • 原游戏 poise 不只是一个数值字段；它和 timer、tag、buff、entity/level event、AI bark、统计系统绑定。
    • ModifyPoise 使用 AbilitySystem.Modifier.finalDelta，不是直接传 double。
    • ModifyPoise 调用链包含 OnPoiseZero，poise 归零副作用集中在 controller 中。
    • OnPoiseZero 会触发 TimeDilationManager.StartPoiseBreakTimeStop；manager 再按敌人模板 rank 选择 PoiseBreakConfig.normal/boss/elite。
    • poiseBreakEffects 在 OnPoiseZero 中只进入 EffectManager.CreateEffectByCfg，当前证据表明它属于视觉 effect 创建路径，不参与 TimeDilation tick/data flow。
    • Tick 使用 timeManagerDeltaTime 推进 poise recover timer；这意味着 owner/global time scale 为 0 时 recover 不应继续流逝。
    • Tick 在 owner entity 有 PredefinedTag.StopPoiseRecover (Status/StopPoiseRecover) 时不会推进 recover timer。
    • Tick 会处理 broken-end/knot timer 到期后的 PeriodicTimer.MarkInvalid 和 MultiTagHandle.RemoveTag。对应 tag 已由 predefine table 确认为 PoiseBroken -> Status/Immobilized/PoiseBroken，knot break immobilize 使用 Immobilized -> Status/Immobilized。
    • OnPoiseZero 通过 PeriodicTimer.Reset(sumPoiseRecTime, true) 启动 recover timer；sumPoiseRecTime 是原游戏 owner AbilitySystem 的恢复时长入口。
    • ResetPoise 不是单纯把 poise 填满；它会触达 event/timer/buff/tag 路径。
    • ModifyPoise 在 poise damage 路径按 m_poiseKnotPctList 推进 m_nextKnotIndex；跨过 knot 后触发 AbilitySystem.Event.OnPoiseKnotBreak，并调用 owner AbilitySystem AddBuff(owner, owner, knotBuffId, true)。
    • OnPoiseZero 会添加两个固定 buff：buff_common_poise_break_damage_taken_scale 存入 m_buffsDuringZeroPoise，buff_common_poise_can_be_breaking_attacked 存入 m_poiseBreakingAttackedBuff。
    • _RemovePoiseBreakBuff / _RemoveBuffsDuringPoiseExist 的主清理路径由汇编确认：遍历 tracked buff 指针、Lock 后调用 owner AbilitySystem.FinishBuff，最后清空列表。
    • OnPoiseZero 汇编确认 AbilitySystemData.buffDuringZeroPoise 非空时逐项调用 AbilitySystem.AddBuff(owner, BuffInput)，且只有返回 ObjectPtr valid 的 buff 才追加到 m_buffsDuringZeroPoise。
    • AbilitySystem._OnAttacked 会在本地受击链路调用 PoiseController.OnAttacked。该入口启动 m_poiseGuardTime，后续 Tick 在 recover timer 无效时累计 elapsed guard seconds。
    • ResetPoise(false) 会消费 m_poiseGuardTime：从 SkillSetting.poiseGuardConfigMob/EliteOne/EliteTwo/Boss 选择配置，评估 curve.Evaluate(elapsed)，若 scalar < 1.0 则添加 buff_common_poise_guard。该 buff 的 mined BuffData 使用 blackboard duration 控制 Limited lifetime，并用 poiseTakenScalar 作为 PoiseDamageTakenScalar 的 FinalMultiplier。
