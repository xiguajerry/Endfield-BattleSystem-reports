    AbilitySystem.ComboController¶

    更新时间：2026-06-11

    归属¶

    ComboController 是 AbilitySystem 的嵌套 SubFeature，不是 PlayerController 的子类。它负责 command 到 skill 的 runtime mapping、next skill request、allowed next skill pack、offset mapping 和 change skill mapping。

    字段¶

    字段 offset 说明
    m_baseCommandMapping 0x20 BattleCommandType -> MappingModifier 基础映射。
    m_runtimeCommandMapping 0x28 BattleCommandType -> PriorityQueue<MappingModifier> runtime 覆盖映射。
    m_realCommandMapping 0x30 当前实际生效映射。
    m_nextSkillRequest 0x38 NextSkillRequest?，缓存下一技能请求。
    m_allowedNextSkillPacks 0xA0 allowed-next 技能集合，优先放行打断。
    m_offsetModifierPack 0xA8 OffsetModifierPack?，dash/skill/jump 触发的 offset mapping。
    m_offsetTargetSkillId 0xE8 offset 目标 skill id。
    m_changeSkillHandles 0xF0 change skill handle 列表。

    嵌套类型¶

    类型 字段/值 说明
    BattleCommandType Attack=0, Dash=1, Jump=2, NormalSkill=3, ComboSkill=4, Count=5 原游戏 command mapping 枚举。
    Priority Default=0, Mode=10, Offset=15, Skill=20, Buff=30 mapping modifier 优先级，不是 skill interrupt priority。
    OffsetTriggerType Dash=0, Skill=1, Jump=2 offset mapping 触发来源。
    MappingModifier skillId, instId, priority, cacheTime runtime mapping 条目。
    NextSkillRequest nextSkillId, remainingTime, allowToCastNextSkill, modifierInstId, skillCastInputData? command 缓存请求。
    OffsetModifierPack handle, duration, triggerType, triggerSkillId, reduceDuration, skillCasted offset mapping 生命周期。
    AllowedNextSkillPack skillList, instanceUid 允许下一技能列表。

    方法¶

    方法 状态 状态 结论
    OnInit() 已确认 documented 设置 owner m_comboController，枚举 skillDataBundle.defaultCmdMapping，用 default cache time 建 Priority.Default base mapping，然后刷新 real mapping。
    AddMappingModifier(commandType, skillId, priority, clearSkillRequestOnRemove, overrideCacheTime, clearOffsetTargetSkillIdOnRemove) 已确认 documented 创建 runtime MappingModifier；cache time 用 override 或 BattleCommandMappingConfig.GetDefaultCacheTime；加入 command priority queue；刷新 real mapping；返回带清理标志的 handle。
    RemoveMappingModifier(cmdType, instId) 已确认 documented 在 runtime queue 中按 instId 查找并移除 modifier，然后刷新 real mapping；不直接处理 handle 的 clear request / clear offset flag。
    _UpdateRealEffectedMapping() 已确认 documented 遍历 BattleCommandType 0..4；runtime queue 有元素则取 queue item 0，否则回落到 base mapping；写入 m_realCommandMapping。
    _GetCacheTime(cmdType, modifier) 已确认 documented 默认返回 modifier.cacheTime；仅 Attack 在 dash / perfect dodge 状态读取 BattleCommandMappingConfig.attackCacheTimeInDash / attackCacheTimeInPerfectDodge。
    BattleCommandMappingConfig.GetDefaultCacheTime(cmdType) 已确认 documented defaultCacheTimeMap.GetValueOrDefault(cmdType, 0f)。
    MappingModifier.CompareTo(other) 已确认 documented 比较 priority 并取负，priority 数值越大越靠前；未看到 instId/sequence tie-break。
    ChangeSkillHandle.ctor(owner, curSkillId, revertSkillId, duration, slot, inheritCdProgress, handle) 已确认 documented 保存 owner/current/revert/slot/inheritCdProgress/handle；duration 同时写入初始和当前 duration；isValid=true。
    ChangeSkillHandle.Tick(deltaTime) 已确认 documented 仅当 isValid && duration > 0.00001 时递减 duration；递减后 duration <= 0.00001 调 Revert()。
    ChangeSkillHandle.Revert() 已确认 documented 若 invalid 直接返回；按 slot 检查当前 _curNormalSkill / _curComboSkill 是否已等于 revert id；未等于且 inheritCdProgress 为 true 时，将当前临时技能 cdProgress 复制到 revert skill；恢复当前 slot，发送对应全局事件；调用 mapping handle RemoveModifier()；isValid=false，duration=-1。

    调用入口¶

    • AbilitySystem.RefreshNextSkillRequest 转发到 ComboController.RefreshNextSkillRequest。
    • AbilitySystem.AddMappingModifier 转发到 ComboController.AddMappingModifier。
    • AbilitySystem.ComboControllerOnSkillCastStart 转发到 ComboController.OnSkillCastStart。
    • AbilitySystem.OnSkillExclusiveTimeExecuted 转发到 ComboController.OnSkillExclusiveTimeExecuted。
    • AbilitySystem.CheckCanInterruptCurSkill 读取 ComboController.allowedNextSkillPacks。

    生命周期¶

    1. 初始化时建立 base command mapping。
    2. runtime action、mode、buff、skill 等通过 AddMappingModifier 或 ChangeSkillMapping 叠加 mapping。
    3. 玩家命令进入后，RefreshNextSkillRequest(cmdType) 根据实际 mapping 产生 NextSkillRequest。
    4. Tick 每帧对 pending request 重新调用 _AllowNextSkill(rawSkillId)，写回 allowToCastNextSkill，并递减 remainingTime；过期时若存在 SkillCastInputData 会先 Dispose，再清空 request。
    5. cast start 时 OnSkillCastStart(skillId) 更新 offset target，并标记 offset pack 已 cast。
    6. request 成功、过期、modifier removal 或 ClearSkillRequest 会清理 next request。

    已确认行为¶

    • OnInit 将 owner.m_comboController 指向当前 controller；随后读取 owner._data.skillDataBundle.defaultCmdMapping，对每个 (BattleCommandType, skillId) 从 GameInstance.dataManager.battleCommandMappingConfig.defaultCacheTimeMap 取默认 cache time，创建 Priority.Default 的 MappingModifier 并写入 m_baseCommandMapping；最后调用 _UpdateRealEffectedMapping。
    • BattleCommandMappingConfig.GetDefaultCacheTime 是 defaultCacheTimeMap.GetValueOrDefault(cmdType, 0f)。
    • AddMappingModifier 的 cache time 选择顺序为：
    • overrideCacheTime.HasValue 时直接使用 override 值。
    • 否则调用 BattleCommandMappingConfig.GetDefaultCacheTime(commandType)。
    • AddMappingModifier 构造 MappingModifier(skillId, cacheTime, priority) 后放入 m_runtimeCommandMapping[commandType] 的 priority queue；该 command 首次出现时会先创建 List<MappingModifier>，放入新 modifier，再用该 list 构造 PriorityQueue<MappingModifier>。添加后立即 _UpdateRealEffectedMapping，返回 MappingModifier.Handle(instId, commandType, clearSkillRequestOnRemove, clearOffsetTargetSkillIdOnRemove, owner)。
    • RemoveMappingModifier 只负责从 m_runtimeCommandMapping[cmdType] 中倒序查找 matching instId 的 modifier 并移除，然后 _UpdateRealEffectedMapping。clearSkillRequestOnRemove / clearOffsetTargetSkillIdOnRemove 是 MappingModifier.Handle.RemoveModifier 或上层 handle lifecycle 的职责，不在该方法本体内处理。
    • _UpdateRealEffectedMapping 清空/重建 m_realCommandMapping，遍历 BattleCommandType 0..4：
    • 若 m_runtimeCommandMapping[cmd] 存在且 queue 非空，取 queue item 0 作为 real mapping。
    • 若 runtime queue 不存在或为空，回落到 m_baseCommandMapping[cmd]。
    • 无 base 时该 command 不写入 real map。
    • PlayerCommandConfig.CmdType 第一层分支在 RefreshNextSkillRequest 中按 cmdType - 1 跳转：
    • Jump=1 -> 读取 BattleCommandType.Jump=2 的 real mapping。
    • Attack=2 -> 读取 BattleCommandType.Attack=0 的 real mapping。
    • Skill=3 -> 读取 PlayerCommandController.GetCommand<SkillCommand>(CmdType=3)，再按当前 AbilitySystem 的 ObjectPtr 从 SkillCommand.m_entity2InputData 取 SkillCastInputData。
    • Dash=4 -> 读取 BattleCommandType.Dash=1 的 real mapping。
    • 其他 command type 走 default/fail path。
    • Skill command 分支中，SkillCastInputView.skillId 会与 owner AbilitySystem._curNormalSkill 和 _curComboSkill 比较：
    • 匹配 _curNormalSkill 时，读取 BattleCommandType.NormalSkill=3 的 real mapping，然后用 _curNormalSkill 更新 modifier skill id。
    • 匹配 _curComboSkill 时，读取 BattleCommandType.ComboSkill=4 的 real mapping，然后用 _curComboSkill 更新 modifier skill id。
    • 两者都不匹配时，直接用 input view 的 skillId 构造默认 MappingModifier。
    • RefreshNextSkillRequest 生成 request 时会先 dispose 旧 request 内的 SkillCastInputData，然后写入新的 nullable NextSkillRequest。
    • 新 request 初始 allowToCastNextSkill = false；实际放行由 Tick 每帧调用 _AllowNextSkill(rawSkillId) 写回，或者由 hasNextSkillRequestPrediction 实时调用 _AllowNextSkill 得到预测结果。
    • _GetCacheTime(cmdType, modifier) 默认返回 modifier.cacheTime；只对 PlayerCommandConfig.CmdType.Attack 有特殊分支：
    • owner movement component isInDash 时返回 BattleCommandMappingConfig.attackCacheTimeInDash。
    • owner ability inPerfectDodge 时返回 BattleCommandMappingConfig.attackCacheTimeInPerfectDodge。
    • 其他状态仍返回 modifier.cacheTime。
    • nextSkillId / hasNextSkillRequest 都要求 allowToCastNextSkill == true；rawNextSkillId 只要求 request 存在。
    • Tick 使用 pending request 的 remainingTime 做过期控制；到期会 dispose request 内 SkillCastInputData 并清空 nullable。
    • Tick 对 offset pack 的倒计时受 reduceDuration 控制：reduceDuration == false 时不递减 duration；置 true 后才递减，到期调用 MappingModifier.Handle.RemoveModifier() 并清空 offset nullable。
    • CreateOffsetMappingModifier 的目标 skill 不读取当前 Attack runtime mapping：m_offsetTargetSkillId 非空则用它，否则取当前 normal attack list 第 0 段。
    • CreateOffsetMappingModifier(Dash, "") 初始 reduceDuration = true，因此 Dash offset 从创建后就开始倒计时。
    • CreateOffsetMappingModifier(Skill, triggerSkillId) 和 CreateOffsetMappingModifier(Jump, "") 初始 reduceDuration = false；前者等 OnSkillExclusiveTimeExecuted(triggerSkillId)，后者等 OnJumpOffsetLanded()。
    • CreateOffsetMappingModifier(Jump, "") 若已有 Jump pack 且 skillCasted == false，会直接返回，不重复创建；若已 cast 过 skill，则允许重建。
    • RefreshOffsetMappingModifier 不创建 pack，也不把 reduceDuration 改成 true；它只是把 duration 刷成 _GetOffsetCachedTime(existing.triggerType)。
    • 入口先遍历 m_changeSkillHandles，对同 SkillSlot 的旧 handle 调 Revert() 并从 list 中移除。
    • SkillSlot.NormalSkill=0：从 _curNormalSkill 对应 Skill 读取 CD progress，写入 owner _curNormalSkill = targetSkillId，目标 skill ChangeSkillType(2) 并 Enable()，若 inheritCdProgress 为 true 则 SetCdProgress(progress)，随后 AddMappingModifier(BattleCommandType.NormalSkill, targetSkillId, Priority.Skill, clearSkillRequestOnRemove=true, overrideCacheTime, clearOffsetTargetSkillIdOnRemove=false)。
    • SkillSlot.ComboSkill=1：同理处理 _curComboSkill，目标 skill ChangeSkillType(6)，command 为 BattleCommandType.ComboSkill。
    • 添加 mapping 后创建 AbilitySystem.ChangeSkillHandle(owner, curSkillId, revertSkillId, duration, slot, inheritCdProgress, handle) 并加入 m_changeSkillHandles。
    • ChangeSkillHandle 构造函数保存 owner、当前 skill id、revert skill id、duration、slot、inherit cd flag 和 mapping handle；duration 写入初始和当前字段，并置 isValid=true。
    • ChangeSkillHandle.Tick(deltaTime) 不处理 mapping 本身，只在 handle 有效且 duration 大于 0.00001 时递减；递减到 epsilon 后调用 Revert()。
    • ChangeSkillHandle.Revert() 分 NormalSkill / ComboSkill 两条分支：
    • 若当前 slot 已经等于 m_revertSkillId，跳过 CD copy 和 slot 写入，但仍调用 mapping handle remove，并将 handle 标记 invalid。
    • 若当前 slot 不等于 revert，且 m_inheritCdProgress == true，从当前临时技能读取 Skill.cdProgress，写入 revert skill SetCdProgress(progress)。
    • 恢复 owner _curNormalSkill 或 _curComboSkill 到 m_revertSkillId。
    • NormalSkill 分支发送 EventManager.SendGlobal<ulong, string>，参数为 owner entity serverId 和 revert skill id；ComboSkill 分支发送 EventManager.SendGlobal<AbilitySystem, string>，参数为 owner AbilitySystem 和 revert skill id。
    • 最后调用 MappingModifier.Handle.RemoveModifier()，置 isValid=false、duration=-1。
    • allowed-next 并不直接执行 cast，它只作为 AbilitySystem.CheckCanInterruptCurSkill 的优先放行条件。
    • _AllowNextSkill 不另写打断规则，而是回到 AbilitySystem.CheckCanInterruptCurSkill。
    • OnSkillCastStart 和 OnSkillExclusiveTimeExecuted 对 offset pack 的两个标记不同：
    • cast start：skillCasted = true。
    • exclusive executed 且 trigger skill 匹配：reduceDuration = true。
