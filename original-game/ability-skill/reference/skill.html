    Skill¶

    更新时间：2026-06-11

    字段¶

    字段 offset 说明
    m_data 0x18 SkillData，含 castData、cost、duration、cooldown 等配置入口。
    m_owner 0x20 所属 AbilitySystem。
    m_ability 0x30 技能 cast 后驱动 timeline/action 的 Ability。
    m_cooldownTimer 0x48 MultiPeriodicTimer，负责技能 CD。
    m_durationTimer 0x50 PeriodicTimer，负责 cast duration/exclusive 进度。
    m_onCastEndCallbackOnce 0x58 cast end 一次性回调。
    m_appliedCost 0x60 cost 是否已经应用。
    m_executedExclusiveCallback 0x61 exclusive callback 是否已经执行。
    m_refreshedOffsetSkillId 0x62 是否已刷新 offset skill id。
    m_pauseTimer 0x70 pause 计数。
    m_hasReachedAtbThreshold 0x8C ATB threshold 是否已满足。
    m_markCanInterrupt 0x8D runtime 标记可打断。
    m_markCanDash 0x8E runtime 标记可 dash。
    m_buffsDuringSkill 0x90 技能期间创建或绑定的 buff 列表。
    m_castStartTime 0xA8 cast 起始 frame。
    _targetHandle 0xC0 目标句柄。
    _blackboard 0xC8 skill action blackboard。
    _skillType 0xD8 SkillType。
    _skillSource 0xDC BATTLE_SKILL_SOURCE。
    _isEnabled 0xE0 是否启用。
    _isPaused 0xE1 是否暂停。
    m_skillId 0xE8 skill id。

    方法¶

    方法 状态 状态 结论
    MarkCanInterrupt() 已确认 已定位 设置 runtime 可打断状态，待细拆。

    调用入口¶

    • AbilitySystem.TryCastSkill(String, TargetHandleView, ...) 已确认：
    • 检查/中断旧技能。
    • 调用 Skill.DoCast 启动新技能。
    • AbilitySystem.PreLateTick：
    • 调用 Skill.OnTick 和必要的 interrupt/finish 路径。
    • Skill.Interrupt：
    • 直接进入 Skill.CastEnd(Interrupted, ...)。

    生命周期¶

    1. AbilitySystem 选择 active skill 并执行 cast 前检查。
    2. Skill.DoCast 建立 cast 状态，重置 duration timer，立即用 ResetOne(cd,true) 启动一次 cooldown charge，记录 cast，初始化必要 buff/action。
    3. Skill.OnTick 推进 duration、ability action、CD recovery、exclusive callback、cost 应用点和 offset refresh。
    4. Skill.Interrupt 或 duration end 进入 Skill.CastEnd。

    已确认行为¶

    • 技能资源检查不在 PlayerCommandController，而在 Skill.CheckCost / _ApplyCost。
    • costType == 0 使用 owner ultimateSp，costType == 1 使用 BattleManager ATB。
    • _GetAtbCost 会叠加 owner atbCostAddition，并 clamp 到非负。
    • Skill.get_cooldown 会按技能类型应用 CD 属性：NormalSkill 加 NormalSkillCooldownAddition；ComboSkill 乘 ComboSkillCooldownScalar 后加 ComboSkillCooldownFinalAddition；其他非 Ultimate 分支除以 SkillCooldownScalar；Ultimate 返回 base。
    • ComboSkillCooldownRecoveryScalar 不改变 Skill.get_cooldown 的 duration，而是在 Skill.OnTick 更新 m_cooldownTimer 时乘到 CD 恢复 delta。
    • CD timer 在 Skill.DoCast 入口即启动；cost 应用不是 cast 入口立即执行，Skill.OnTick 会在技能进度达到 startCdFrame 对应时间点时调用 _ApplyCost(true)，startCdFrame=0 会在 cast-start zero tick 内提交。
    • 如果技能在 startCdFrame / 30 - epsilon 前结束，Skill.CastEnd 调 m_cooldownTimer.ResetOne(cooldown,false)，只释放当前未 ready charge，不重置所有 charge。
    • Skill.CheckCd 在没有 ready charge 时仍允许 maxPassedTime < startCdFrame / 30 的窗口，和 cast-start CD 启动配套。
    • AbilitySystem.TryCastSkill 在 Skill.DoCast 后立即调用 Skill.OnTick(0,0)；这会经由 TimelineActionProcessor._TickInternal(0,0) 在 cast-start 调用栈内执行 startTime/frame 0 的 timeline actions。
    • Skill.Create 中 frame-split init 不用于 passive skill；passive skill 创建后立即 _Init()，active skill 可进入 AbilitySystem.m_pendingInitSkills。
    • Skill.CastEnd 中 m_buffsDuringSkill 的 finish 顺序在 cast-end callback 之后、BattleRecorder.RecordEndSkill 之前；finish reason 使用 Buff.FinishReason.Other=100。
    • Skill.CastEnd 调用的 cast-end callback 进入 AbilitySystem._OnCastEnd；OnSkillEnd 的 ability event owner 是 AbilitySystem，不是 Skill 本体。
    • Skill.Interrupt(Dash) 会在 CastEnd(Interrupted) 前调用 AbilitySystem.CreateComboOffset(OffsetTriggerType.Dash, "")。
