    AbilitySystem.TacticalItemController¶

    更新时间：2026-06-16

    归属¶

    TacticalItemController 是 AbilitySystem 的嵌套 SubFeature。它持有当前 TacticalItem，负责 refresh、tick、使用/停止使用、对自身使用物品效果和 item buff 应用。

    字段¶

    字段 offset 说明
    _tacticalItem 0x20 当前 TacticalItem。
    m_isPlayingTacItemToSelfEffect 0x28 是否正在播放对自身使用物品效果。
    m_tacItemToSelfEffectTimer 0x30 对自身效果 timer。
    m_toSelfUseItemData 0x38 对自身使用的 item data。

    方法¶

    方法 状态 状态 结论
    OnInit() 已确认 已定位 初始化 sub feature。
    Start() 已确认 已定位 sub feature start。
    Tick(allScaledTime, timeManagerDeltaTime, unscaledDeltaTime) 已确认 已确认 以 unscaledDeltaTime 调用 TacticalItem.OnTick；若正在播放对自身使用效果，则 timer 完成后 ApplyUseItem 并触发 healed bark。
    Finish() 已确认 已定位 sub feature finish。
    Release() 已确认 已定位 sub feature release。
    OnAllocateInternal() 已确认 已定位 reusable allocate。
    OnRecycleInternal() 已确认 已定位 reusable recycle。
    RefreshTacticalItem() 已确认 已确认 查询 ItemEquipData，必要时比较 item id，随后调用 TacticalItem.Init(data, owner)。
    StopUseTacticalItem() 已确认 已确认 薄转发；若 _tacticalItem 存在则调用 TacticalItem.StopUse()。
    SetAsItemUseTarget(ItemUseData, AbilitySystem) 已确认 已确认 设置对某 AbilitySystem 使用 item 的目标；按目标/owner 类型创建 effect、self-effect timer 或直接 ApplyUseItem。
    ApplyUseItem(ItemUseData) 已确认 已确认 由 Tick/self effect 完成后调用；遍历当前 squad slot，转发每个角色的 TacticalItemController.AddItemBuff。
    AddItemBuff(ItemUseData, bool) 已确认 已确认 前置要求 owner entity objectType == Character(8) 且 _isPaused (Entity + 0x73) 为 false；随后遍历 ItemUseData.useActions，处理 ApplyBuffSingle(2) / ApplyBuffAll(3) 并给目标加 buff。

    调用入口¶

    • AbilitySystem sub feature lifecycle 调用 OnInit/Start/Tick/Finish/Release。
    • 装备或状态变化时调用 RefreshTacticalItem。
    • 输入、AI 或 event path 调用 CheckAndTryToUseTacticalItem / StopUseTacticalItem。
    • 对自身使用效果在 Tick 中推进，最终进入 ApplyUseItem。

    生命周期¶

    1. AbilitySystem 初始化时创建 TacticalItemController。
    2. RefreshTacticalItem 根据装备数据创建或刷新当前 TacticalItem。
    3. 每帧 Tick 推进 TacticalItem 和对自身效果 timer。
    4. 使用入口通过 controller 转发到 TacticalItem runtime。
    5. 对自身或目标使用成功后，ApplyUseItem 和 AddItemBuff 应用 item buff。
    6. release/recycle 清理 runtime 和 effect 状态。

    已确认行为¶

    • Controller 持有单个 _tacticalItem，不是每次使用都临时查表。
    • Tick 调用 TacticalItem.OnTick，说明 TacticalItem 自己推进 wait/cast/cooldown timer。
    • RefreshTacticalItem 查 item equip data 并调用 TacticalItem.Init(data, owner)。
    • ApplyUseItem 按当前 squad 倒序遍历 slot：
    • 取 slot character 的 AbilitySystem；
    • 取该 AbilitySystem 的 m_tacticalItemController；
    • isSingleBuffActive = targetAbility == this.owner；
    • 调用目标 controller 的 AddItemBuff(useItemData, isSingleBuffActive)。
    • AddItemBuff 前置检查：
    • owner entity 的 object type 必须为 ObjectType.Character(8)；
    • owner entity _isPaused 字段（offset 0x73）必须为 false，否则直接返回且不应用 item buff。
    • AddItemBuff 对 use action 的处理：
    • useType == ApplyBuffSingle(2) 时，只有 isSingleBuffActive == true 才给当前 owner 加 buff；
    • useType == ApplyBuffAll(3) 时给当前 owner 加 buff；
    • buffId 来自 UseActionBundle.buffBBData.buffId，blackboard 来自 buffBBData.blackboard 并经 Blackboard.Convert；
    • source 和 target 都传当前 owner AbilitySystem，再调用 AbilitySystem.AddBuff(source,buffId,blackboard)。
    • BattleManager.UseTacticalItem 成功扣库存后会记录 BattleRecorder.RecordUseTacticalItem；Controller 不直接记录战斗日志。
    • SetAsItemUseTarget 分支：
    • owner entity object type 为 Character(8) 且 _isPaused == false 时才走 tactical item effect / item buff 逻辑；否则直接或跳过。
    • 当 sourceAbility 非空且不同于当前 owner 时，读取 GameplayMiscSetting.tacticalItemToGuardEffects，为每个 effect cfg 调 EffectManager.CreateEffectByCfg(cfg, sourceAbility, targetAbility, ...)，并给 EffectInstance 注册 loaded callback。
    • loaded callback 已确认 在 effect load 成功且 GameObject 上存在 EffectParabolaMove 时，调用 EffectParabolaMove.Init(effectInstance,startEntity,endEntity,callback)；抛物线完成 callback 为 display class 中的 b__1。
    • callback b__1（确认状态 已确认）执行 ApplyUseItem(useItemData)，随后触发 BarkSystem.Bark(Battle_Healed, ...)。
    • 当目标就是当前 owner 时，读取 self-effect list 创建 effect，随后把 useItemData 写入 m_toSelfUseItemData、置 m_isPlayingTacItemToSelfEffect = true 并 reset m_tacItemToSelfEffectTimer(waitFirstPeriod=true)；真正 ApplyUseItem 发生在 controller Tick 的 self-effect timer 完成分支。
    • 其他不需要 effect 延迟的情况直接 ApplyUseItem(useItemData)。
    • GameplayMiscSetting.tacticalItemToSelfDelay 位于 offset 0xD8，GameplayMiscSetting.ctor 状态 已确认 在 已确认 写入默认值 1.0f；该值用于 self-effect timer period。
